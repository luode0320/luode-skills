# 需求暂停触发条件

## 用途

用于判断当前需求是否因为信息不足而必须暂停。

## 必须暂停

- 连目标都无法准确复述。
- 连输入输出都说不清。
- 缺少核心业务规则，继续写只会靠猜。
- 缺少关键依赖前提，继续推进会走错方向。
- 多个合理方向都说得通，但当前还没做选择。
- 当前需求文档里仍存在关键 `TBD/TODO/待补`、重大歧义或内部矛盾。

## 可继续但要标记风险

- 个别非核心字段尚未最终命名。
- 少量文案、展示细节暂未完全定稿。
- 不影响主流程的边缘场景还需后补。

## 原则

- 只拦关键缺口，不滥用暂停。
- 如果缺口会导致返工风险明显上升，应优先暂停。
- 如果未收敛方向会让不同实现路径都显得合理，也应优先暂停。
- 暂停结论也要更新到同一份 `ment/` 需求文档中，明确阻断项和恢复推进条件。
