# 功能验证范围

## 用途

用于界定“当前改动本身”应该验证什么，不应该验证什么，并明确验证结论最后如何落在新的测试目录结构中。

## 本轮应验证

- 当前需求新增或修改的主要功能路径。
- 与当前变更直接相关的输入输出结果。
- 当前接口、页面、交互链路的预期行为。
- 与当前变更直接相关的关键异常分支或失败提示。

## 本轮不主动扩张

- 与当前改动无关的历史功能兼容性。
- 全系统上下游联通性和环境一致性。
- 全量回归范围和公共模块扩散影响。
- 测试资源位置、命名和文档组织问题。

## 留痕要求

- 功能验证主结论统一写入 `test/yyyy-MM-DD_HHmmss/测试任务中文简介/README.md`。
- 详细步骤、样例、截图和执行证据放入同一时间戳根目录下的 ASCII 真实代码路径镜像目录。
- 中文说明目录不承载额外测试代码和明细附件。

## 检查清单

- 验证对象是否已经说清。
- 当前验收标准是否可落到具体结果。
- 关键主路径和必要边界路径是否已覆盖。
- 当前结论是否足以回答“这次改动做对了吗”。
- 当前结论与详细证据是否分别落在 README 和 ASCII 镜像路径中。
