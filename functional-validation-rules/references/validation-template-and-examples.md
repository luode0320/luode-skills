# 功能验证结论模板与样例

## 结论模板

- 验证对象：
- 关联需求 / 变更：
- 验证范围：
- 执行环境：
- 验证步骤摘要：
- 实际结果：
- 未通过项 / 风险项：
- 结论：通过 / 驳回 / 待确认
- 下一步流转建议：

## 正例

- 对新增接口验证请求参数、返回结果和错误分支，并给出是否满足当前接口契约的结论。
- 对页面交互验证关键操作路径、提示文案和状态变化，并说明是否满足当前需求描述。
- 对修改后的业务逻辑验证核心输入输出和关键边界条件，并记录未通过项和回改建议。

## 反例

- 只写“已测通过”，没有验证范围、步骤和结论依据。
- 本应验证当前需求，却扩成全量系统回归。
- 把联调失败直接判成当前功能不通过，但没有区分环境问题和实现问题。
- 把测试资源目录问题、文档缺失问题当成功能失败结论。
