# 功能验证边界

## 用途

用于区分功能验证与测试策略、联调验证、回归验证、测试资源管理之间的职责。

## 属于功能验证域

- 当前需求是否实现正确。
- 当前接口、页面、输入输出是否符合本次预期。
- 当前改动引入的行为是否满足当前验收标准。
- 功能验证结论如何写入中文 README，并指向详细执行证据。

## 不属于功能验证域

- 决定先测哪些场景、覆盖优先级怎么排。
- 测试资源怎么存放、怎么命名、文档怎么写。
- 上下游系统是否打通、环境是否一致、协议字段是否对齐。
- 旧功能是否被本次修改带坏、是否需要扩大回归范围。

## 回流规则

- 功能未通过且问题在实现本身，回流编码域或 Bug 域。
- 功能未通过但需求理解存在歧义，回流需求域。
- 功能现象无法闭环，且更像环境或链路问题，转交 `integration-debugging-rules`。
- 当前修改已通过功能验证，但仍需确认兼容性时，进入 `test-regression-rules`。
- 如果结论和证据落点不符合时间戳根目录 + 中文 README + ASCII 镜像路径结构，回流测试资源管理类 skill。
