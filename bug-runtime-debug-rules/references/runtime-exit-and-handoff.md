# 运行时诊断退出与交接

## 用途

用于判断运行时诊断何时结束，以及如何把结果交接回 Bug 域后续阶段。

## 结束条件

- 已观察到足够证据，能够排除主要竞争假设。
- 已锁定异常状态、异常顺序或异常位置。
- 已确认当前手段无法再带来新证据，应停止继续试错。

## 结束后流转

- 如果已形成稳定根因，回流到 `bug-root-cause-rules` 固化结论。
- 如果根因已经足够明确且可以开始方案讨论，进入 `bug-fix-proposal-rules`。
- 如果仍然证据不足，应明确卡点，而不是假装已定位成功。

## 特别提醒

- 运行时诊断的价值在于证据，不在于保留调试动作本身。
- 临时 debug 代码在进入测试或交付前必须有明确处理结论。
