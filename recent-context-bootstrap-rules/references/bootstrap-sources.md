# 近 3 天上下文扫描来源

## 默认时间范围

- 默认只看最近 3 天。
- 优先理解为“从当前时刻向前 72 小时内”的材料。
- 如果用户明确给出日期范围，服从用户范围。

## 默认扫描来源

按下面顺序扫描：

1. `ment/`
2. `test/`
3. `bug/`
4. `doc/`
5. 最近 3 天的 git 提交

## 扫描原则

- 目录不存在时直接跳过，不报假错误。
- 优先看最近修改时间更近、文件名更像当前任务的材料。
- 不要一上来深读所有文件，先找最近、最相关的候选。
- git 先看提交标题；只有标题不足以判断时，再看必要的改动线索。
- 如果最近 3 天材料很多，优先保留和当前问题最相关的 3 到 10 条。

## 结果要求

- 必须明确写出扫描了哪些来源，哪些来源不存在，哪些来源没有发现相关内容。
- 不要把“目录不存在”写成“项目没有这类历史”。
- 不要把旧材料混进最近 3 天摘要里。
