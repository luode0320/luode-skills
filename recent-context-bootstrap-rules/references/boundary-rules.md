# 边界规则

## 与 `history-recall-rules` 的边界

- `recent-context-bootstrap-rules` 是新会话启动时的轻量预热。
- `history-recall-rules` 是用户明确要求回忆过去方案、过去修复、过去决策时的深度历史回溯。
- 如果只是想先知道最近几天发生了什么，优先走这里。
- 如果已经明确要查“以前怎么做过”，转交 `history-recall-rules`。

## 与 `project-timeline-rules` 的边界

- 这里不负责长期项目历史。
- 这里只看最近 3 天，不做项目阶段演进报告。
- 如果用户要的是项目完整历程、关键决策时间线，转交 `project-timeline-rules`。

## 与 `project-design-doc-rules` 的边界

- 这里最多只把根目录 `项目设计.md` 及其同类文档当作弱参考源，帮助快速理解项目总览。
- 这里不负责判断项目设计文档是否过期，不负责同步、合并或补建 `项目设计.md`。
- 如果用户明确要求“分析整个项目并输出 / 更新项目设计文档”“同步架构说明”“生成根目录项目设计.md”，转交 `project-design-doc-rules`。

## 与主域 skill 的边界

- 这里不负责需求分析、Bug 定位、编码实现、测试验证。
- 这里只负责在进入主域之前，先补一个“最近上下文压缩包”。
- 预热结束后必须让路给真正的主域 skill。
- 不允许把过期设计文档直接当成当前实现结论。

## 不应触发的场景

- 用户只是闲聊、问通用知识、问工具怎么用。
- 当前会话里已经有足够前情，不缺上下文。
- 当前任务与本仓库项目无关。
