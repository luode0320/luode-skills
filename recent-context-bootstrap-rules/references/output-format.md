# 输出格式

建议输出结构保持短而稳定：

## 1. 最近 3 天相关背景

- 只保留与当前任务直接相关的最近活动。
- 优先写需求、Bug、测试、文档、提交中最关键的几条。

## 2. 对当前任务最有价值的前置信息

- 写“现在继续做这件事最应该先知道什么”。
- 不要把材料原文大段搬出来。

## 3. 来源

- 列出主要来源目录和关键文件。
- 必要时补一两条最近 git 提交标题。
- 如果引用了根目录项目设计类文档，单独标注“弱参考来源”，不要和最近 3 天材料混写成同一层级事实。

## 4. 当前仍缺什么

- 如果最近摘要仍不足以继续，应明确指出缺什么，而不是假装上下文已经完整。

## 压缩要求

- 摘要要短，默认控制在一个简短区块内。
- 重点是“让后续主域更快进入状态”，不是把近 3 天材料全文复述一遍。
