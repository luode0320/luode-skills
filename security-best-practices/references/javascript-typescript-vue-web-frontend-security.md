# Vue 前端安全规则

## 适用范围

适用于 Vue 3 及 TypeScript/JavaScript 客户端应用。

## 默认工作方式

- Vue 模板默认会做一定转义，但 `v-html`、动态组件、动态 URL 仍然是高风险点
- 客户端只能做体验控制，不能替代服务端授权

## 核心规则

### VUE-CONFIG-001：客户端配置默认公开

- 打进前端构建产物的配置默认可见
- 密钥、凭证、数据库连接信息不能进入客户端配置

### VUE-XSS-001：谨慎使用 `v-html`

- 不可信内容不能直接绑定到 `v-html`
- 富文本必须先清洗再渲染

### VUE-XSS-002：不要把用户输入编译成模板

- 不要动态编译来自外部的模板字符串
- 动态组件名、动态模板来源要走白名单

### VUE-URL-001：动态链接、资源地址、跳转目标必须校验

- `href`、`src`、跳转目标都要限制协议和域名
- 危险协议默认禁止

### VUE-AUTH-001：认证状态和令牌处理要考虑 XSS

- 不要把高敏感 Token 长期放在浏览器可读存储
- Cookie 会话要明确 CSRF 防护策略

### VUE-AUTHZ-001：前端权限控制不是最终边界

- 路由守卫、按钮显隐只解决体验问题
- 对象级权限和关键操作权限必须由后端兜底

### VUE-3P-001：第三方脚本和插件最小化

- 外部脚本、富文本组件、上传组件、图表组件都要视为风险输入点
- 引入后要确认其更新节奏和安全记录

### VUE-POSTMSG-001：跨窗口消息必须校验来源

- `postMessage` 收发两侧都要显式校验

## 审计重点

- 是否存在 `v-html`、动态模板、动态组件来源不可信
- 是否把敏感配置或 Token 放进客户端
- 是否只在前端做权限判断
- 是否存在危险外链、跳转和第三方脚本
