# 图片红框重点关注规则

用于统一“用户给图后，AI 应该先看哪里、先说什么”。
核心原则：红框优先、结论前置、避免跑偏。

## Skill 作用与适用场景

- 用户上传截图、页面图片、设计稿、报错截图并要求分析。
- 用户在图中使用红框/红圈/红色线条标注重点区域。
- 用户希望根据图片做样式调整、布局修复、元素删减、视觉审查。
- 用户未额外说明重点时，自动把红框区域当成本轮优先关注对象。

## 默认提示词

当检测到用户提交图片或截图时，默认先使用以下提示词进行思考与输出：

```text
我会优先关注你图片中的红色框选/标注区域。该区域通常是你本轮希望我重点处理的位置（如样式不整齐、对齐问题、可删除元素、文案或交互异常）。我会先给出红框区域结论，再补充与其直接相关的非红框信息，避免分析跑偏。
```

## 自动触发信号

- 用户消息中包含图片、截图、设计稿等视觉素材。
- 用户提到“看图”“看截图”“图里这个地方”“红框里的问题”等表达。
- 任务与 UI/样式/布局/可视化结果相关，且输入包含图像证据。

## 默认执行流程

1. 先识别是否存在红色框选/标注区域，并定位重点对象。
2. 若存在多个红框，按从上到下、从左到右顺序逐个说明，并标明优先级。
3. 先输出红框区域的问题判断和处理建议，再补充与其直接相关的上下文信息。
4. 若图片中没有红框或红色标注，明确说明“未检测到红框”，再按全图进行常规分析。
5. 若任务涉及前端 UI/样式修改，先承接本 skill 的红框重点结论，再路由到 `frontend-design`（排版/对齐问题先经过 `web-design-guidelines` 审查）。

## 输出要求

- 输出时先写“红框重点结论”，再写“关联补充结论”。
- 不把主要篇幅用于无关区域，避免偏离用户意图。
- 当红框信息不足以判断时，要明确补充所需信息，而不是臆断。

## 权责边界与不负责事项

- 本 skill 只负责“图片重点区域识别与注意力排序”。
- 不代替 `frontend-design`、`web-design-guidelines`、`bug-*`、`requirement-*` 等主域 skill 的具体执行。
- 不负责图像编辑本身；若需要生成/编辑图片，应转交对应图像能力 skill。

## 执行通过 / 驳回标准

- 通过：能先聚焦红框区域，先给重点结论，再给关联补充，且不跑题。
- 驳回：忽略红框区域、先分析无关区域、或没有明确红框优先策略。
