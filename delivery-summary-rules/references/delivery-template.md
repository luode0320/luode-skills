# 交付总结模板

结论：{用一句普通中文说明本次交付达成的结果}；影响：{说明受影响的用户、角色或业务环节}；范围：{列出本次实际交付的内容}；非范围：{列出明确未随本次交付的内容}；变化：{说明交付后用户或业务可感知的变化}；完成标准：{说明交付完成和可交接的条件}；术语说明：无技术术语需要解释，或 {术语：它在本次交付中的作用}；验证状态：{说明已完成和未完成验证及依据}。

## 白话正文与附录

正文先说明这次交付带来的结果、影响、范围和非范围、可感知变化、完成标准、术语含义、验证结论和风险；H1 后单段必须完整包含八个固定字段。提交范围、命令、构建记录和证据链接放入文末连续附录。

1. 文档标题与保存文件名（北京时间 `yyyy-MM-DD_HHmmss+简短标题说明.md`）。
2. Git 提交范围（用户指定从哪个历史提交之后开始统计，到哪个提交结束）。
3. 任务目标。
4. 主要改动。
5. 影响范围。
6. 已完成验证。
7. 未覆盖项和已知风险。
8. 发布 / 交接注意事项。
9. 后续建议。

## 执行附录

- 提交范围、local 命令、构建和发布记录、验证样本、交接步骤、清理和回滚说明。

## 追踪附录

- 来源对象、稳定标识、提交或版本定位、测试和审查证据链接、风险与决策记录。
