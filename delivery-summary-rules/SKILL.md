# 交付总结规则

只在需要形成一份可交接、可复盘、可追溯的交付说明或发布总结文档时使用这个 skill。
如果当前还没有稳定实现结论、测试结论或发布结论，请先完成相邻环节；如果只是做上线动作，请转回团队发布流程处理。

## Skill 作用与适用场景

- 把本次交付的变更、验证、风险和遗留事项收成一份可读说明。
- 以用户指定的 Git 提交范围为边界，整理本次需要总结的提交信息。
- 将总结文档统一保存到项目根目录 `doc/` 下的 Markdown 文件中。
- 降低后续交接、复盘和维护成本。
- 让交付域不只停留在发布动作，还能形成信息闭环。
- 避免“代码已经合了，但没人知道到底改了什么”。

## 自动触发信号

- 任务已经准备交付，需要说明变更内容、验证结论和风险。
- 用户明确要求给出交付说明、更新摘要或后续建议。
- 用户明确提出“请帮我生成交付文档”“发布文档”“发布总结文档”“准备上线总结文档”。
- 需要把实施结果交给测试、产品、运维或下一位维护者。
- 需要留下一份可供后续快速定位的交付记录。

## 进入后先做什么

1. 先确认用户是否明确同意“现在开始发布总结”；未确认前不进入下一步。
2. 确认用户已经给出“从哪个历史提交之后开始统计”的 Git 起始提交；如果没有给，必须先让用户确认。
3. 再确认需求、实现、测试和发布结论已经基本收口。
4. 提炼本次交付的目标、主要改动和影响范围。
5. 汇总已验证项、未验证项、风险点和遗留事项。
6. 确认输出文件保存到项目根目录 `doc/`，文件名使用北京时间 `yyyy-MM-DD_HHmmss+简短标题说明.md`。
7. 形成结构化交付摘要和后续建议。

## 默认执行流程

1. 默认先确认用户是否明确同意开始发布总结；如果没有明确确认，先暂停。
2. 确认用户已经给出本次 Git 汇总的起始提交；如果没有给，先暂停并向用户确认。
3. 默认再读 `references/delivery-template.md`，按统一结构输出交付说明。
4. 如需继续展开，再读 `references/delivery-risk-and-debt.md`，整理风险、遗留和待补项。
5. 需要对照边界或正反例时，再读 `references/delivery-examples.md`，对照交付总结正反例。
6. 输出交付总结、Git 提交范围摘要、验证摘要、风险与遗留说明。
7. 将总结文档保存到项目根目录 `doc/` 下；若 `doc/` 不存在，应创建后再保存。
8. 交付总结完成后，可用于提交需求方、测试方、发布方或后续维护者；若总结中暴露新缺口，应回流相邻阶段修正。

## 权责边界与不负责事项

- 只负责交付说明，不代替代码评审、测试执行或上线动作。
- 不替用户猜测本次交付应统计哪个 Git 提交范围。
- 不把所有实现细节写成冗长日志，重点是让接收方知道要点。
- 不隐去已知风险和未覆盖项。
- 不在结论尚未收敛时伪造“已完全完成”的交付口径。

## 需要暂停并确认的条件

- 实现、测试或发布结论仍明显不稳定。
- 关键验证结果、遗留问题或风险项还没汇总。
- 交付对象和需要的说明深度不明确。
- 团队试图用交付说明掩盖实际未完成项。
- 用户还没有明确确认“是否开始发布总结”。
- 用户没有给出本次需要汇总的 Git 起始提交，无法判断哪些提交属于本次总结范围。

## 执行通过 / 驳回标准

- 通过：交付说明能清楚说明目标、主要改动、Git 提交范围、验证结论、风险、遗留事项和后续建议，并已保存到项目根目录 `doc/` 下的规范命名 Markdown 文件中，适合接收方快速使用。
- 驳回：交付说明只写空泛总结，缺少 Git 提交范围、验证、风险和遗留说明；或未得到用户确认就直接开始；或未按规则保存到 `doc/` 目录。

## 执行结果归档要求

- 将交付总结统一保存到项目根目录 `doc/` 下的 Markdown 文件中，不再使用 `delivery/`、`analysis/` 或其他并行目录。
- 文件名使用北京时间命名格式：`yyyy-MM-DD_HHmmss+简短标题说明.md`。
- 归档内容至少包含 Git 提交范围、变更摘要、验证结论、风险项、遗留事项和建议动作。
- 如果后续又有补交付或补验证，应继续在 `doc/` 目录下新增增量总结文件，并沿用同样命名规则。

## references 读取规则

- 默认先读 `references/delivery-template.md`。
- 只有在 梳理交付中的风险与遗留事项 时，再读 `references/delivery-risk-and-debt.md`。
- 只有在 对照样例或判断总结是否足够可交接 时，再读 `references/delivery-examples.md`。
