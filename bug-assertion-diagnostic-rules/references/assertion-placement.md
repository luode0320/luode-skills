# 断言放置原则

- 放在状态刚进入关键区间前后，便于收缩发生范围。
- 断言内容要围绕不变量，不要重复已有正式校验。
- 失败时最好带上最小必要上下文，便于回溯。
- 同一链路避免堆叠过多断言，防止噪音和副作用。
- 当前 Bug 的断言记录统一归档到项目根目录 `bug/` 下对应的 Bug 根目录中，命名格式参考 `artifact-storage-rules` 的时间戳模板，使用 `yyyy-MM-DD_HHmmss_问题中文简介`。
- 诊断断言属于调试改动，问题定位或修改完成后默认删除。
- 如果后续需要正式保护逻辑，应删除当前诊断断言，再按正式校验规则重写，不直接保留当前断言。
