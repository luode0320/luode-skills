# 缓存策略规则

## 用途

用于约束缓存值不值得加、key 怎么设计、何时失效。

## 基本原则

- 先确认是否真的需要缓存，再决定放在哪一层。
- 缓存 key 必须稳定、可解释、与租户、用户态或权限态边界一致。
- 失效策略必须明确，不能只写入不失效，也不能指望“自然过期”兜底所有场景。
- 缓存命中收益要能说明，不靠想象添加额外复杂度。

## 风险提示

- 含用户态、权限态、动态敏感数据的结果缓存要特别谨慎。
- 写后读一致性要求高的场景，缓存策略必须解释清楚。
- 缓存不能替代正确的 schema、query 和接口实现。
- 引入缓存后必须考虑击穿、雪崩、穿透和绕过条件。
