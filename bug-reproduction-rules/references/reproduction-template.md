# 复现模板

结论：{用一句普通中文说明问题是否复现}；影响：{说明复现确认会影响的用户、角色或业务环节}；范围：{说明本次会验证的触发场景}；非范围：{说明本次不验证的场景}；变化：{说明复现时用户或业务可观察到的变化}；完成标准：{说明复现结论成立的可观察条件}；术语说明：无技术术语需要解释，或 {术语：它在本次问题中的作用}；验证状态：{说明复现频率、稳定性和当前证据}。

## 白话正文与附录

先说明问题是否复现、影响、范围和非范围、可观察变化、如何判断修复有效、术语含义和复现进展；H1 后单段必须完整包含八个固定字段。环境、账号、数据、精确步骤和日志统一写入文末连续附录。

1. 环境与版本。
2. 账号、权限、关键数据。
3. 前置状态。
4. 操作步骤。
5. 实际结果。
6. 期望结果。
7. 复现频率与稳定性。

## 执行附录

- local 环境与版本、账号和权限、关键数据、前置状态、精确步骤、命令、样本、日志、清理和回滚。

## 追踪附录

- 问题来源、稳定标识、复现证据、关联需求或修复任务、验证记录和结论变更。
