# 本地材料时间线工作流

## 适用条件

当没有真实持久记忆后端时，改用本地材料拼接时间线。

## 建议材料来源

1. `ment/` 中的需求、验收和变更文档
2. Bug 相关分析与验证文档
3. 交付摘要、评审结论和根目录分析文档
4. Git 提交历史、标签、版本节奏和关键提交说明

## 建议组织顺序

### 1. 先列出时间轴骨架

至少先列：

- 最早可找到的起点
- 中间重要阶段
- 最近阶段

### 2. 再填关键事件

例如：

- 新模块引入
- 架构调整
- 大版本切换
- 重大 Bug 修复
- 发布 / 回滚 / 灰度节点

### 3. 最后补结论

按时间线抽出：

- 项目演进主线
- 重要决策
- 常见问题模式
- 后续维护者需要注意的历史包袱

## 输出要求

- 必须明确说明这是“基于本地材料和 Git 历史整理”的时间线。
- 如果某段历史材料不足，应显式标出空白阶段。
- 如果用户只需要短版本，优先输出骨架摘要，不必强行扩写长报告。
