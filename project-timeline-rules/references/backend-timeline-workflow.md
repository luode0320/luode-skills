# 真实记忆后端时间线工作流

## 核心目标

从真实持久记忆里拿到项目长期历史，再按时间顺序组织成阶段报告。

## 推荐步骤

### 1. 确认项目名

- 优先使用用户明确指定的项目名。
- 如果当前目录是 Git worktree，优先确认主仓库名，不要误用 worktree 目录名。

### 2. 获取完整或分段时间线

如果记忆后端支持完整时间线拉取，可直接按项目获取全量或分段时间线。

如果材料很大，优先按阶段或按主题分段处理，不要盲目一次读完。

### 3. 先评估体量

至少先判断：

- 时间范围有多长
- 记录数量大概有多少
- 是否需要先给用户一个短摘要，再决定是否展开长报告

### 4. 组织报告结构

建议至少覆盖：

- 项目起点与初始目标
- 关键架构演进
- 重要突破与转折点
- 重大问题与调试攻坚阶段
- 技术债积累与偿还
- 对后续维护最重要的结论

## 输出要求

- 明确写出使用的项目范围和时间范围。
- 明确哪些部分来自真实记忆记录。
- 如果存在历史缺口，必须直说，不要补写成连续完整故事。
