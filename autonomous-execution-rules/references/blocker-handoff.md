# 阻断状态收口与事实交接

本文件只定义真实 `blocked/manual_handoff` 的运行时收口和事实交接，不负责面向用户渲染阻断区块。

### 进入条件

- 已满足当前运行时对 `blocked` 或 `manual_handoff` 的真实定义。
- `limited`、`not_applicable`、普通失败、可继续修复的问题或尚未达到阻断阈值的候选，不得升级为真实阻断。

### 收口顺序

1. 若运行环境存在 goal / plan / task 等显式状态机制，先真实调用对应工具或协议完成状态收口。
2. 确认状态工具调用成功；不得用“任务已阻断”的文字代替真实收口。
3. 按 `artifact-delivery-gate-rules/references/task-blocker-closure-contract.md` 交接结构化事实：状态、阶段、依据与证据、已尝试与停止条件、影响、解决计划、恢复验证和重入点。
4. 将事实交给 `reasoning-summary-structure-rules`，由其唯一渲染最终用户状态。

### 无显式状态机制

只有运行环境确实不存在状态收口机制时，才允许退化为普通最小文案收口；不得伪造不存在的工具调用。

### 禁止

- 状态未真实收口前宣称“已阻断”。
- 自行生成与最终总结 Owner 重复的阻断区块、后续内容或解决计划。
- 把可继续处理、验证不适用或普通受限状态包装成阻断。
