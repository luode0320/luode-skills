# 用户结束、无下一步与防循环

本文件承载所有 agent 通用的结束信号和无下一步规则。

### 明确结束指令

“结束”“停止”“到此为止”“不要继续”“不要下一步建议”“不要扩散”“只要结论”“先停在这里”“只做到这一步”等表达一旦出现，立即停止自动继续、额外工具调用、可选优化和扩散性输出。只允许完成运行时要求的必要状态收口，并给出最小结论。

### 三类合法后续

最终收口仅允许出现：

1. 原执行计划内未完成且完成原始目标仍必需的事项。
2. 真实阻断项。
3. 用户显式要求的建议或 backlog。

不属于以上三类的“进一步优化”“可继续整理”“再做一轮”“以后可以”等内容不得自动升级为新任务。

### 强制无下一步

原始用户目标已完成且不存在三类合法后续时，直接结束回复。禁止输出“下一步状态”“下一步建议”“后续可优化项”“等待用户新指令”“无需继续动作”或其他可能被循环执行器识别为新任务的占位文案。

### 运行时状态

若环境存在 goal / plan / task 等显式状态机制，目标完成后必须先真实执行完成状态收口，再输出最终结论。Codex 的 `get_goal` / `update_goal` 只是具体实现之一；无活动 goal 时不得伪造完成动作。

### 驳回样例

- 用户已结束，仍追加建议或开启新任务。
- 目标已完成，仅因总结里写了“可以继续”就自动扩展范围。
- 无合法后续时仍输出等待类、无需动作类或下一步占位文案。
- 只写“任务完成”但没有执行当前运行时要求的真实状态收口。

### 结束与无后续示例

- 原始目标已经完成时，先核验是否存在原计划未完成必需项、真实阻断或用户显式要求的建议；三类后续都不存在时直接结束，不输出“还可以继续优化”。
- 用户直接说“结束”时，将其识别为所有 agent 通用的结束指令，立即停止工具执行、自动继续和建议生成；只有运行时仍有必须完成的状态收口动作时，先完成该动作，再输出最小结论。
