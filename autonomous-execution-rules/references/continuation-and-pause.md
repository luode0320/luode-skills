# 继续、暂停与高风险边界

本文件承载连续推进节奏和关键节点；不得扩展 `SKILL.md` 的自动触发范围。

### 允许自动继续

- 当前仅完成阶段性子步骤，整体目标尚未闭环。
- 下一步属于原执行计划内未完成的必需项，且仍是完成原始用户目标所必需。
- 下一步可在当前上下文、授权和安全边界内执行。
- 中间进度后仍可继续实现、真实测试、必要修正、审查、验收或收口。

### 最小任务与周期节奏

- 默认节奏是“当前周期 -> 当前最小任务实现 -> 当前最小任务真实测试 -> 当前最小任务审查 -> 当前最小任务验收 -> 下一个最小任务 -> 当前周期收口 -> 下一周期”。
- 当前最小任务未闭环不得进入下一个任务；当前周期未收口不得跳到下一周期。
- 禁止先完成多个任务的实现，再统一测试、审查或验收。
- 当前最小任务进入 `in_progress`、完成为 `completed` 或下一个任务保持 `pending` 时，先由 `task-plan-rehydration-rules` 原子更新 `PROJECT_CURRENT.md`，再调用 `update_plan`；磁盘写入失败不得刷新 UI，工具失败不得回滚已持久化状态。
- Desktop 或宿主中断后恢复到 `in_progress` 步骤时，先核验磁盘、测试和外部状态；未知或非幂等写操作不得自动重放。任务投影恢复不改变 `confirmed/unknown/revoked` 执行许可。
- 可自行消解的不确定项采用最小可逆决策继续；不能自行消解且会改变目标、接口或风险时暂停。

### 必须暂停

- 首次进入编码且执行许可不是 `confirmed`。
- 开工指令缺少有效计划，或采纳的建议尚未转为计划。
- 上下文压缩后编码许可为 `unknown` / `revoked`。
- 需要新增超出已确认范围的模块、依赖、数据结构或业务逻辑。
- 删除或覆盖大量文件、破坏性数据操作、生产动作、密钥或权限变更等高风险或不可逆操作。
- 外部付费、跨系统联动或预计成本显著增加。
- 信息冲突无法靠现有证据收敛，继续执行有较高做错概率。
- 任务准备进入 Git 动作但当前轮无 Git 授权；若用户当前轮已明确 Git 动作，直接交给 `git-collaboration-rules`。
- 命中用户结束、无合法后续或真实阻断条件。

### 关键节点提问

只说明当前阶段结果、节点类型和 2-3 个互斥选项；推荐项置首。不得用开放式“你想让我接下来做什么”替代可执行决策。

固定格式：

```text
我已完成 <当前阶段结果>。当前进入关键节点：<节点类型>。
建议选项：
1) <推荐选项>（推荐）
2) <备选选项>
3) <保守选项>
```

连续执行示例：当阶段性证据分析完成且下一步仍属于原计划必需项时，不停在“如果你要，我可以继续”的可选话术；应直接推进到下一项必需动作。只有进入提交授权、高风险或越界节点时，才使用上述格式暂停。

### 安全与审批

自主执行不能绕过系统安全限制、权限审批、local 环境边界、外部副作用确认或高风险防护。任何更高优先级安全规则要求暂停时，本 Skill 必须停止继续。
