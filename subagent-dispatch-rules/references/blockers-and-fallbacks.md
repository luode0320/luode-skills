# 阻断与回退规则

## 阻断信号

- 用户明确禁止委派 subagent。
- 子任务写集重叠且无法拆分。
- 子任务结果与主路径证据冲突。
- 子任务涉及高风险不可逆操作。

## 回退策略

1. 立即停止新增委派。
2. 主 agent 接管关键阻塞步骤，本地推进主路径。
3. 对已返回的 subagent 结果做“可用 / 需复核 / 丢弃”分级。
4. 记录冲突原因与后续动作，避免重复踩坑。
5. 主 agent 必须输出一条可见回退说明，明确“本次不委派的原因”与“已改为本地执行”。

## 结果冲突裁决

- 与主路径事实冲突时，以主路径可验证证据为准。
- 无法裁决时，暂停并向用户汇报冲突点与选项。

## 最低安全线

- 不绕过权限、审批和安全限制。
- 不在未知风险下并行执行破坏性动作。
- 不把“委派失败”伪装成“任务完成”。
