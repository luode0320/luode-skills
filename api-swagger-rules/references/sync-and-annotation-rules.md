# Swagger / OpenAPI 同步与注解规则

## 同步原则

- 只要接口路径、tag、请求 DTO、响应 DTO、错误响应或鉴权要求发生变化，就必须同步 Swagger/OpenAPI。
- 不允许“接口已经改完，文档后面再补”长期积压。
- 如果项目采用注解/注释生成文档，注解/注释应视为接口契约的一部分，而不是普通说明性注释。

## 与接口主干 skill 的同步关系

### 与 `api-endpoint-rules`

- 路径、方法、tag、摘要、弃用标记必须同步。
- 新增接口后，Swagger/OpenAPI 中必须可定位到该接口。

### 与 `api-request-rules`

- 请求字段名、必填/可选、基础格式约束、枚举值、示例应同步。
- 不允许代码里已经切到 JSON body，但 Swagger/OpenAPI 仍保留旧 query/path 参数描述。

### 与 `api-response-rules`

- 成功响应、错误响应、分页结构、兼容字段和版本字段应同步。
- 不允许代码里已经统一返回结构，但 Swagger/OpenAPI 仍展示旧裸对象。

## 最小同步检查项

- tag 是否正确。
- 摘要是否表达真实动作。
- 请求模型是否与当前字段一致。
- 成功响应模型是否与当前响应包装一致。
- 错误响应是否覆盖主要失败路径。
- 鉴权说明是否遗漏。
- 示例是否仍能用于联调调试。

## 弃用与隐藏

- 已废弃接口要显式标记为 deprecated。
- 内部接口、仅运维接口、临时排障接口应默认隐藏或限制暴露。
- 不要把实验性接口直接当成稳定公开接口对外展示。
