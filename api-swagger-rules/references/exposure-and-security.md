# Swagger / OpenAPI 暴露与安全规则

## 环境开放基线

- 开发环境：默认允许访问 Swagger/OpenAPI 页面和调试入口。
- 测试环境：默认允许联调使用，但应避免匿名公网暴露。
- 生产环境：默认关闭；如确需开放，必须受控。

## 生产环境受控开放的最低要求

- 环境开关。
- 登录鉴权或统一认证。
- IP 白名单、内网限制或网关层保护。
- 明确哪些分组可见、哪些接口禁止调试。

## 不允许的情况

- 生产环境直接裸开 Swagger UI。
- 生产环境把内部管理接口、调试接口、临时排障接口直接暴露给所有访问者。
- 没有任何权限控制就允许在线调试高风险接口。

## 调试入口使用边界

- Swagger/OpenAPI 主要用于开发、测试和联调效率提升。
- 它可以帮助快速构造参数、查看响应和确认契约，但不能代替正式功能验证与回归验证。
- 如果调试入口已经影响安全边界，应优先收紧入口，而不是继续扩张功能。
