# PROJECT_STYLE.md 模板

把这份文件作为项目根目录的唯一代码风格记忆文件。

## 目的

- 只保留一份权威代码风格来源。
- 只写明确、可直接复用的代码样例。
- 让后续写代码时复用同一套风格表述。

## 建议章节

1. `核心风格`
2. `命名与注释`
3. `方法与函数`
4. `结构体与类`
5. `变量与常量`
6. `异步与并发`
7. `日志与错误处理`
8. `接口与边界`
9. `工具调用`
10. `循环与控制流`
11. `废弃写法`
12. `变更记录`

## 条目格式

每条风格样例尽量用简短的块或表格表示。

```md
### 方法返回空值
- 别名: 空结果返回, empty return
- 类型: 方法风格
- 示例: `return nil, nil`
- 说明: 当查询无结果且业务允许空返回时，直接返回空值，不额外包一层错误。
- 来源: `UserService.FindOne()`
- 适用范围: 查询方法
- 更新时间: 2026-06-26
- 状态: 启用
```

## 合并检查清单

- 先读当前主文档，再开始编辑。
- 只补充对话、代码或文档里已经明确的风格样例。
- 把别名规范到一个主样例下。
- 去重时保留最清晰、最权威的写法。
- 旧写法过时后，直接在原位置替换。
- 已废弃内容直接标记 `status: deprecated`。
- 保持文档紧凑、易扫描。

## 什么算风格

- 项目偏好的命名方式。
- 项目偏好的注释写法。
- 方法、类、结构体、变量的常见组织方式。
- 异步、日志、错误处理、接口、工具调用、循环等常见写法。
- 值得长期复用的项目习惯。

## 什么不该写进来

- 未确认的猜测。
- 临时讨论笔记。
- 重复的原始聊天记录。
- 另建一份承载同类长期风格的日志文件。
