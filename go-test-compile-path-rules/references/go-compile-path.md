# Go 测试编译路径与禁放处理

## 1. 源码目录禁放

- `internal/`、`common/`、`api/`、`cmd/` 等真实源码目录中不得出现 `*_test.go`。
- 只要发现源码目录 `*_test.go`，默认判定为违规落点，而不是“白盒特例”。

## 2. ASCII 可编译路径

- 所有会被 `go test ./...` 扫描到的测试路径都必须保持 ASCII。
- 中文只能出现在说明目录中，不能出现在 Go 可编译测试包路径中。
- 即使测试文件位于 `test/` 根目录下，只要镜像路径包含中文，仍判定为不通过。

## 3. 白盒同包诉求处理

- 需要访问未导出符号时，不允许在源码目录同包落地测试文件。
- 统一处理方式是改 seam：导出接口、依赖注入点、测试钩子或其他可外部测试入口。
- seam 到位后，再把测试文件放回 `test/<时间戳>/` 的 ASCII 镜像路径。

## 4. 联动关系

- 若问题本质是“新测试任务目录还没建”，先转交 `test-task-root-layout-rules`。
- 若问题同时包含散落资产迁移，联动 `test-scattered-asset-location-rules`。
- 本 skill 只处理 Go 路径、禁放和 seam，不替代测试程序编写本身。

## 5. 最小检查清单

- 当前 `*_test.go` 是否仍位于源码目录。
- 当前测试包路径是否全 ASCII。
- 当前白盒诉求是否已经转换成 seam 方案。
- 相关测试资产是否已经回到 `test/<时间戳>/` 的正确镜像路径。
