# 审核流程（main 时间锚点之后的当前分支提交）

## 输入参数

- 无外部输入参数。
- 被审核项目目录固定为当前执行目录（即当前项目根目录）。

## 关键流程

1. 确认当前项目根目录
- `project_path=$(pwd)`
- 后续所有命令在当前目录执行，不再接收外部项目路径参数。

2. 获取当前分支
- `current_branch=$(git rev-parse --abbrev-ref HEAD)`

3. 读取 main 分支最近一条提交时间
- 先校验 `main` 是否存在：`git rev-parse --verify main`
- 读取时间戳：`main_latest_ts=$(git log -1 --format=%ct main)`

4. 基于时间锚点筛选当前分支提交（时间正序）
- `commits=$(git rev-list --reverse --since=\"@${main_latest_ts}\" HEAD --not main)`
- 含义：只保留当前分支中尚未并入 `main`，且提交时间晚于 `main` 最新提交的记录。
- 若为空：输出“当前分支没有需要审查的新提交”并结束。

5. 逐 commit 审查
- 对每个 commit 单独构造提示词并执行审核。
- 每个 commit 的结果必须独立成段。

6. 汇总报告
- 合并所有 commit 审核结果为单份 Markdown。
- 写入项目根目录固定文件名（同名覆盖）。

## 问题判定逻辑

- 若任一 commit 在 `致命/严重/中等/建议` 任一级别出现非“无”内容，视为发现问题。
- 若输出缺少固定级别标题，也按“发现问题”处理（保守策略）。

## 明确不做

- 不拉取远端。
- 不读取或校验远端分支引用。
- 不做增量状态文件追踪。
- 不上报企业微信。
