# 项目设计文档同步策略

## 应更新的触发条件

出现以下变化时，应同步更新根目录 `项目设计.md`：

- 项目目录结构发生明显调整
- 核心模块边界或职责划分发生变化
- 关键主链路、业务主流程或系统交互关系发生变化
- 技术栈、关键依赖、部署形态或项目整体约束发生变化
- 原有设计文档与当前代码、当前有效文档明显偏移

## 可补建的条件

满足以下条件时，可补建根目录 `项目设计.md`：

- 当前项目根目录没有统一主入口 `项目设计.md`
- 已经完成一次足够可靠的整项目分析
- 能够基于真实代码、当前文档和最近变更提炼出稳定总览

## 不应机械补建的场景

- 只分析了一个局部模块，却试图生成整项目设计文档
- 当前项目还在快速搭建期，没有稳定模块和主链路
- 代码、文档、Git 变更三者冲突严重，尚未判断真实状态

## 收口原则

- 最终主入口统一为根目录 `项目设计.md`
- 历史别名文档可以保留，但不应继续与 `项目设计.md` 并行维护同一层级主说明
- 如果当前轮不适合直接迁移历史文档，至少要在结论中明确后续收口方向
