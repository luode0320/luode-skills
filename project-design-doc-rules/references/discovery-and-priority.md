# 项目设计文档发现与优先级

## 根目录候选文档

优先在仓库根目录查找以下文件：

1. `项目设计.md`
2. `架构设计.md`
3. `项目大纲.md`
4. `系统设计.md`
5. `项目说明.md`

规则：

- `项目设计.md` 是统一主入口。
- 其他文件默认视为历史同类文档或临时过渡入口。
- 如果多个文件并存，不代表它们都同样可信。

## 证据优先级

判断项目真实状态时，默认优先级如下：

1. 真实代码、目录结构和当前实际实现
2. 当前有效的需求、实施、Bug、测试和交付文档
3. 最近 Git 变更
4. 根目录项目设计类文档

补充原则：

- 根目录项目设计类文档属于弱参考源，不保证是最新。
- 只要设计文档和真实代码冲突，优先以真实代码与当前有效文档为准。
- 设计文档的价值在于快速建立项目总览，而不是覆盖实现事实。

## 输出要求

- 必须明确指出当前读取了哪份根目录设计文档。
- 必须明确标注“弱参考”或“当前同步较新”。
- 如果文档内容与代码明显偏移，必须直接指出偏移点，不得继续沿用旧说法。
