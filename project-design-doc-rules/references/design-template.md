# 项目设计文档模板

根目录 `项目设计.md` 建议至少包含以下章节：

## 1. 文档说明

- 文档用途
- 当前可信度说明
- 最近同步时间：`yyyy-MM-DD HH:mm:ss`
- 主要依据：代码 / 当前文档 / 最近 Git 变更 / 历史设计文档

## 2. 项目目标与范围

- 项目主要用途
- 当前主要服务对象或业务目标
- 当前不覆盖的范围

## 3. 技术栈与运行形态

- 后端 / 前端 / 数据库 / 中间件 / 外部依赖
- 关键框架与语言
- 部署或运行形态的概要说明

## 4. 目录与模块总览

- 根目录结构概要
- 核心模块及职责
- 关键公共层与共享能力

## 5. 核心链路

- 主要业务主流程
- 关键上下游交互
- 关键数据流或调用链

## 6. 当前约束与风险

- 设计上的当前限制
- 已知偏移点
- 待同步或待补齐项

## 写作要求

- 文档默认使用 UTF-8。
- 说明文字默认使用中文。
- 不追求面面俱到，优先写“当前仍然成立的总览事实”。
- 如果某部分仍不确定，应明确写“待确认”，不要伪造确定性。
