# 测试任务根目录与布局

## 1. 根目录约定

- 强制使用 `test/` 作为统一测试根目录。
- 不得使用 `tests/`、`qa/`、`verify/`、`tmp-test/` 等其他名称作为测试根目录。
- 每次新增测试任务，必须先在 `test/` 下创建独立时间戳根目录。

## 2. 时间戳根目录规则

- 时间戳根目录命名格式固定为 `yyyy-MM-DD_HHmmss`。
- 默认新测试任务必须创建当天时间戳目录。
- 若目标目录日期早于运行时当天日期，必须中止并询问用户是否明确复用旧目录。
- 未获明确确认前，禁止把新文件写入旧时间戳目录；旧目录仅可读取参考。
- 如用户明确复用旧目录，需在任务说明中写明复用原因和旧任务 ID。

## 3. 中文说明目录规则

- 每个时间戳根目录下必须创建一个中文说明目录。
- 中文说明目录默认只存放这一个 `README.md`。
- `README.md` 至少说明：测试目的、执行方式、依赖数据、覆盖范围、验证结论、真实测试资产路径。
- 中文说明目录只承载说明，不承载真实测试代码、脚本、mock、fixture、截图或报告产物。

## 4. ASCII 真实代码路径镜像规则

- 所有真实测试资产都放在同一个时间戳根目录下，并按真实代码路径镜像组织。
- 真实测试资产路径必须保持 ASCII。
- 若当前测试对象无法精确映射到单一源码文件，至少映射到最接近的源码模块路径。
- 中文说明目录与真实测试资产目录并列存在，不能互相替代。

## 5. 多轮测试与归档

- 同一需求需要多轮独立验证时，分别创建多个时间戳根目录。
- 每个时间戳根目录仅存放对应测试任务的脚本、数据、代码、说明和产物。
- 旧时间戳根目录建议按周或按月归档到 `test/archive/` 下。

## 6. 最小检查清单

- 当前测试任务是否位于 `test/` 根目录下。
- 是否已创建当天时间戳根目录，或已获得明确复用许可。
- 是否已创建中文说明目录与 `README.md`。
- 真实测试资产目录是否保持 ASCII，并按真实代码路径镜像组织。
- 是否错误地把真实测试资产写进中文说明目录。
