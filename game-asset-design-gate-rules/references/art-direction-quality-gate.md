# 商业级美术审稿闸门

用于把“质量要更高”拆成可执行、可复核、可回炉的标准。
当用户提供商业游戏截图、指出当前素材像低级 hero prop、或要求对标成品游戏观感时，必须读取本文件。

## 质量阶梯

### Tier 0: Blockout

- 只验证尺寸、位置、碰撞、图层关系。
- 允许简单几何图形、Pillow 草图、占位色块。
- 禁止作为正式素材交付。

### Tier 1: Low Hero Prop

- 能看出资产是什么。
- 有基本体块、阴影、结构和少量细节。
- 但外轮廓僵硬、线条均匀、负形贫乏、材质像平面色块。
- 只能作为中间稿，不能作为正式收口。

### Tier 2: Production Candidate

- 外轮廓好看，缩小后仍有记忆点。
- 线稿有粗细、压重、断续和手绘节奏。
- 主体有姿态或生长势，避免完全对称和机械重复。
- 材质层级清楚，能看出前后遮挡、亮暗、软硬或发光关系。
- 接地影和地图融合度合格，不像贴纸。

### Tier 3: Shippable

- 单体拿出来也能成立，放进地图也不突兀。
- 用户拿商业游戏截图对比时，主要差距不再是“像不像正式素材”，而只是风格偏好或精修程度。
- 具备 1-3 个高完成度焦点细节，并且这些细节服务玩法识别。
- 适合进入游戏正式资源链路。

## 参考差距拆解

看到参考图后，不要只提炼“卡通、俯视、发光、描边”这类宽泛词。
必须拆成以下审稿项：

| 审稿项 | 低级结果 | 高级结果 |
| --- | --- | --- |
| 外轮廓 | 椭圆、圆角矩形、规则多边形堆叠 | 不规则但受控，缩小后有独特剪影 |
| 线稿 | 均匀描边、每处一样粗 | 受光面轻、落地面重、遮挡处重，局部断线 |
| 姿态 | 正面摆拍、对称、僵硬 | 偏心、倾斜、外伸、枝条或结构有方向感 |
| 负形 | 大块实心，没有孔洞和缝隙 | 枝条、空洞、悬挂物、破损边形成可读空隙 |
| 材质 | 同色块填充，加几条线 | 层级、遮挡、边缘磨损、局部高光共同表达材质 |
| 焦点 | 到处平均点缀 | 1-3 个最亮/最细/最高对比细节形成记忆点 |
| 发光 | 随机亮点 | 发光体有载体、内芯、外晕和对附近结构的影响 |
| 接地 | 一团透明黑椭圆 | 形状跟主体受力点和枝干落点对应 |
| 地图融合 | 像贴纸浮在地上 | 明度、饱和度、阴影方向、边缘噪点与地图一致 |

## 硬失败条件

出现以下任一项，默认必须回炉：

- 看起来像程序几何图形或 PPT 图标。
- 主体由几个椭圆、圆形、规则多边形直接堆叠，缺少二次造型。
- 只有“结构齐全”，但没有手绘线稿节奏。
- 所有描边粗细一致，没有压重和轻重变化。
- 没有负形，轮廓像一坨实心块。
- 细节平均分布，没有视觉焦点。
- 发光点像随机贴上去的小圆点，没有载体和照亮关系。
- 阴影和主体落点无关，只是一团通用黑椭圆。
- 放进地图后明显像贴纸、贴片或廉价占位图。
- 用户已经指出“比参考差很多”，但下一版只做了微调，没有解决主要差距项。

## 高级树木 / 大型植被标准

正式树木、藤蔓、变异植物、发光树、菌树等默认必须满足：

- 树冠不是圆形堆叠，而是 3-5 个不规则冠层块面。
- 树干和枝条必须影响外轮廓，不能被树冠完全遮住。
- 至少有一条有方向性的主枝或主茎，让资产有姿态。
- 枝条、果实、晶体、叶洞或破损形成清晰负形。
- 发光元素必须有载体，例如果实、晶体、裂缝、符文、孢子囊。
- 接地影必须对应树根、枝条和主干落点。
- 颜色不超过主色、暗部、亮部、焦点色四组，但每组内部允许少量明度变化。

## 建筑 / 路障 / 残骸标准

正式建筑、棚架、路障、残骸、机器物件默认必须满足：

- 有清楚的主结构：屋顶/墙体/梁柱/支架/底座/机械核心至少两层。
- 结构必须能解释受力，不能只是几块板漂浮。
- 破损边、铆钉、绑带、裂缝、木纹、金属磨损至少选择两类做局部焦点。
- 顶面和侧面要有明度差，让俯视或 3/4 视角成立。
- 底部必须有接地压重和遮挡关系。
- 不允许用“一个多边形板 + 几个发光点”冒充建筑或路障。

## 自审评分

交付前必须按 10 分制自评：

- 外轮廓：0-2
- 线稿节奏：0-2
- 结构/姿态：0-2
- 材质/焦点：0-2
- 接地/地图融合：0-2

通过线：

- Tier 2 至少 7 分，且不能有 0 分项。
- Tier 3 至少 8.5 分，且线稿节奏、外轮廓、材质/焦点都必须至少 1.5 分。

如果低于目标阶梯，不要写“已完成”，必须写“当前为草稿 / 候选失败 / 需要回炉”，并指出下一轮优先修哪 3 项。

## 生成策略

优先策略：

1. 先写参考差距拆解。
2. 再写原创 brief。
3. 再生成或制作初稿。
4. 初稿后按自审评分判断是否回炉。
5. 回炉时只改最影响分数的 1-3 项，不做随机改动。

工具策略：

- 图像生成或人工绘制更适合 Tier 2 / Tier 3 原始图。
- 程序绘制更适合 blockout、图层合成、碰撞、尺寸验证和快速 layout。
- 若必须程序绘制正式资产，必须额外模拟手绘线稿、不规则轮廓、负形、遮挡和局部焦点；否则只能标记为 Tier 0 或 Tier 1。

## 输出要求

涉及高价值 2D 素材时，交付说明必须包含：

- 目标质量阶梯。
- 参考差距拆解。
- 自审评分。
- 是否达到目标阶梯。
- 若未达到，下一轮回炉优先级。
