# 项目风格一致性合同

正式 2D 素材必须像来自同一个游戏。
本合同用于防止角色、怪物、地图、道具、UI、特效各自使用不同画风，导致项目像多个素材包拼接。

## 核心原则

- 风格一致性优先于单个素材炫技。
- 单个素材更精致但不像当前项目，仍然是失败素材。
- 外部参考只能提升项目既有风格的完成度，不能替换项目风格。
- 同一阶段尽量只维护一个主视觉基线；确实需要分支风格时，必须说明分支边界和融合方式。

## 设计前必须锁定视觉基线

制作新素材前，先从项目已有正式素材和文档中提炼视觉基线。

至少记录：

- 视角：俯视、3/4、侧视、镜头高度、角色朝向。
- 比例：角色头身比、怪物体量、道具图标尺寸、地图 prop 尺度。
- 轮廓：外轮廓粗细、压重位置、是否有断线、是否非均匀描边。
- 色彩：主色族、焦点色、饱和度、明度范围、冷暖关系。
- 材质：树冠、金属、木材、皮肤、布料、晶体、能量的表达方式。
- 光照：主光方向、亮面位置、暗部面积、投影方向。
- 发光：是否使用发光点、内芯/外晕/照亮周边的规则。
- 细节密度：裂缝、高光、纹理、颗粒和小装饰的数量级。
- 噪点与边缘：是否低噪点、是否手绘抖动、边缘是否柔化。
- 玩法可读性：敌我、威胁、拾取、互动、背景的层级优先级。

如果项目已有素材质量不稳定，先建立“本轮临时风格基线”，并强制本轮全部新素材共享它。

## 风格一致性检查项

交付前必须检查：

| 检查项 | 通过标准 | 失败信号 |
| --- | --- | --- |
| 镜头 | 视角和现有资产一致 | 某些素材像侧视，某些像俯视 |
| 描边 | 粗细、压重、断线规则一致 | 一个像矢量图，一个像厚涂图 |
| 色彩 | 主色和焦点色属于同一色彩系统 | 单个素材突然高饱和或过暗过亮 |
| 材质 | 同类材质表达方式一致 | 同是金属，一个写实，一个卡通 |
| 光照 | 亮面和阴影方向一致 | 发光和阴影方向互相矛盾 |
| 细节密度 | 小尺寸下复杂度接近 | 地图很简，角色过写实，或反过来 |
| 边缘处理 | 抗锯齿、手绘抖动、轮廓节奏接近 | 某些像干净 AI 渲染，某些像手绘 |
| 地图融合 | props 像长在地图上 | props 像贴纸或另一个素材包 |
| UI/图标融合 | 图标和角色/道具同一视觉语言 | 图标像网页 icon 或别的游戏 |

## 受控差异

允许为了玩法识别做差异，但必须受控：

- 玩家可以比怪物更亮、更干净，但描边和光照规则仍要一致。
- 精英怪/Boss 可以更复杂，但材质语言和镜头不能跳变。
- 投射物和命中特效可以更高对比，但色彩应来自项目焦点色系统。
- UI 图标可以更简洁，但轮廓、焦点色和材质表达要和游戏物件有关联。
- 地图背景可以低对比，但地图 props 不能因此退化成另一个画风的贴纸。

## 参考图使用规则

当参考图比项目现有素材更成熟时：

1. 先提炼质量方法，而不是复制画风。
2. 保留项目镜头、配色系统、描边规则和世界观。
3. 只吸收参考图的线稿节奏、负形组织、材质分层、焦点控制和阴影/发光方法。
4. 如果参考图会导致项目风格整体迁移，必须把它作为“全项目视觉升级”单独确认，而不是在单个素材里偷偷切换。

## 硬失败条件

出现以下任一项，默认风格一致性失败：

- 新素材像来自另一个素材包。
- 新素材描边、阴影、色彩或材质明显不同于项目已有资产。
- 角色、怪物、地图、道具的镜头角度不一致。
- UI 图标像网页图标，战斗素材像手绘游戏，地图又像低饱和矢量图。
- 参考图风格被整体搬进项目，破坏已有视觉基线。
- 只有单体好看，放进实际地图或 UI 后突兀。

## 自审输出

涉及正式素材时，交付说明必须包含：

- 项目视觉基线摘要。
- 本素材继承了哪些基线。
- 本素材有哪些受控差异，以及为什么需要。
- 风格一致性风险。
- 风格一致性结论：通过 / 需要回炉 / 需要全项目视觉升级确认。
