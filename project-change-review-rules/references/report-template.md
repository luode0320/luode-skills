# 审查报告模板

```text
发现:
- [P1] path/to/file.ext:123 - 简短标题
  证据: 指出当前 diff 或源码行为。
  风险: 说明可能造成的缺陷、安全、回归或规则违规。
  建议: 给出最小可执行修复方向。
  关联 skill: <主 skill 名称；若仅为总审查发现可留空>

未发现阻断项:
- 未发现 P0/P1 时写本段；若存在阻断项则省略。

已覆盖:
- Diff 范围:
- 已读规则:
- 已联动 skill:
- 已执行命令:

未覆盖/剩余风险:
- 未运行测试:
- 未展开文件:
- 仍需确认:

下一步建议:
1. 优先处理 P0/P1。
2. 补齐验证或专门 skill 审查。
3. 复审修复后的 diff。
```

## 写法约束

- findings 永远放最前。
- 不用“整体不错”开头。
- 没有发现时直接写“未发现阻断项”，同时列出未覆盖风险。
- 建议必须能落地到文件、测试、skill 或复审动作。
