# 当前改动审查清单

## Diff 入口

- `git status --short`
- `git diff --stat`
- `git diff --name-only`
- `git diff`
- `git diff --cached --stat`
- `git diff --cached --name-only`
- 对未跟踪文本文件按入口、配置、迁移、脚本、测试、文档优先读取。

## 必查问题

- 边界：是否只改当前需求需要的文件；是否有无关格式化、重命名、生成物、锁文件漂移。
- 完整性：请求、响应、接口文档、配置、迁移、测试、日志、错误处理是否同步。
- 缺陷：空值、边界、并发、幂等、事务、缓存、时区、分页、排序、权限、异常路径。
- 安全：鉴权、越权、敏感日志、注入、路径穿越、密钥、调试入口。
- 复用：是否已有 util、组件、常量、枚举、框架能力可复用。
- 可读性：函数过长、嵌套过深、命名模糊、魔法值、补丁式分支、文件继续膨胀。
- 注释：关键改动是否解释“为什么”；中文和 UTF-8 是否正常；是否有过期或废注释。
- 日志：是否使用正式 logger；是否具备业务上下文、错误对象、trace/span、脱敏。
- Skill：是否漏触发对应 review skill；已触发 skill 是否有未执行的阻断规则。
- 验证：是否运行了与风险匹配的测试、构建、lint 或手工验证。

## 证据要求

- 每条发现都必须有文件、行号或 diff 片段依据。
- 不确定的问题标为“风险/疑问”，不要写成已确认缺陷。
- 没有执行的测试必须写入剩余风险。
