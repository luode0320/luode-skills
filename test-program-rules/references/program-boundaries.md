# 测试程序边界

## 用途

用于区分正式测试程序、临时调试代码、联调脚本、文档留痕和生产辅助代码的边界。

## 明确属于测试程序域

- 正式测试文件中的用例组织与断言入口
- mock server、stub、fake、fixture builder
- 测试环境初始化和清理脚本
- 为测试复用而沉淀的公共断言和公共构造器
- 放在 ASCII 真实代码路径镜像目录中的长期保留测试脚本

## 不属于测试程序域

- 中文说明目录中的 `README.md`
- 一次性排障脚本、临时打印日志、临时断言、手工定位代码
- 联调阶段为排查环境问题写的短期验证脚本
- 生产代码里的通用工具、业务服务、正式运行时逻辑
- 测试说明文档和测试报告

## 处理原则

- 临时排障代码不直接升级为正式测试程序，除非已经补齐职责、命名和归档。
- 测试辅助逻辑不得直接塞进生产 `utils`、`common`、`service` 目录。
- 正式测试程序、模拟程序和验证脚本必须跟随 `test-location-rules`，统一落在 `test/yyyy-MM-DD_HHmmss/<ASCII 真实代码路径镜像>/` 下。
- 中文说明目录只保留 `README.md`，不承载任何真实测试代码。
- 如果本质是目录归属错误，应回流 `test-location-rules` 或 `code-placement-review-rules`。
