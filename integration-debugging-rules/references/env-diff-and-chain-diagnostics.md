# 环境差异与链路诊断规则

## 用途

用于判断联调失败是环境差异、协议不一致还是链路断裂。

## 环境差异排查

- 比对环境变量、配置分层、网关配置、域名、证书、版本和依赖服务状态。
- 明确是否只有测试环境失败、只有预发失败或只有特定租户 / 特定账号失败。
- 不把“本地能跑”直接当作环境无差异的证据。

## 链路诊断

- 先确认请求是否发出，再确认是否到达，再确认是否被正确消费。
- 关注 Header 丢失、字段重命名、序列化差异、超时、重试和网关覆盖。
- trace 断链时要同时检查生成点、透传点、记录点和采集点。

## 检查清单

- 哪一跳开始出现差异
- 哪个环境独有问题
- 是协议不一致、配置差异、权限差异还是依赖未就绪
- 当前证据是否足以交给责任方修复
