# 联调边界与回流规则

## 用途

用于区分联调问题与功能验证问题、Header 规则问题、日志 trace 问题、单模块 Bug 诊断问题。

## 回流功能验证

- 当前改单点就不满足需求或验收标准
- 不依赖上下游也能稳定复现错误

## 回流 Bug 运行时诊断

- 问题明确卡在单个服务、单个模块、单个进程内
- 需要断点、变量观察、条件日志来定位本地运行时状态

## 回流 Header / 日志 trace

- Header 契约本身未定义清楚，或透传规则本身有争议
- trace / span 字段设计、日志字段设计本身需要重定

## 保留在联调层

- 上下游协议不一致
- 环境配置不一致
- 网关、代理、Header 透传、链路串联过程中出现断裂
- 多团队、多系统协作下需要统一证据和责任分流
