# 认证与安全通用基线

## 用途

用于统一认证、鉴权、输入校验、敏感信息和高风险入口控制的最小基线。

## 认证与鉴权

- 认证控制点要集中，避免每个接口各写一套零散逻辑。
- 对象级和租户级访问必须做所有权或租户边界校验。
- 前端只能做体验控制，不能承担最终授权。
- Cookie 会话、JWT、API Key、Session 的使用前提要明确，不混用成模糊身份体系。

## 输入校验

- 所有输入都视为不可信，包括 path、query、body、Header、Cookie、上传文件和外部系统返回值。
- 结构化校验优先于零散 `if` 判断。
- 不把认证信息、重置密钥、敏感令牌放进 URL 参数。

## 敏感信息

- 密钥、密码、私钥、访问令牌、带凭证连接串必须外置，不写入仓库、日志、报错或客户端 bundle。
- 用户、账单、权限、认证数据都要明确区分公开模型和内部模型。
- 错误、响应、日志和调试输出都不能泄露高敏感信息。

## 高风险入口

- 上传、下载、文件访问必须限制类型、大小、数量，并防止活动内容直接暴露。
- 外部请求必须限制目标、协议、重定向和超时，防止 SSRF。
- 跳转目标必须校验，不能直接信任用户提供的 URL。
- 浏览器自动带 Cookie 的写操作要考虑 CSRF；只用 Header Token 的场景通常不以 CSRF 为主风险。
