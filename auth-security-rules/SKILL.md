# 认证与安全规则

只在“身份怎么验、权限怎么控、输入怎么校验、敏感信息怎么保护、高风险入口怎么兜底”这个问题上使用这个 skill。
如果当前问题只是 Header 透传、请求参数建模、错误处理路径或日志记录点，请转交相邻 skill。

## Skill 作用与适用场景

- 统一认证、鉴权、对象级授权、输入校验、敏感信息处理和高风险入口防护基线。
- 覆盖登录、登出、会话、JWT、Cookie 会话、上传下载、外部请求、重定向和浏览器相关安全边界。
- 把你已经下载的中文 `security-best-practices` seeds 纳入本 skill 的框架细则来源。
- 防止只做表面权限控制、只做前端隐藏、只做零散输入判断或把敏感信息暴露给客户端、日志和响应。

## 自动触发信号

- 新增或修改认证、鉴权、登录、登出、会话、JWT、对象级授权、输入校验、敏感信息处理、上传下载安全、外部请求安全。
- 新增或修改登录接口、受保护路由、中间件、依赖注入鉴权、权限判断、角色判断、租户隔离校验。
- 新增或修改文件上传、文件下载、URL 跳转、外部 HTTP 请求、WebSocket、Cookie 认证或浏览器安全相关实现。
- 发现访问控制只在前端、输入未结构化校验、敏感数据泄露、外部请求无边界或上传下载存在活动内容风险时。

## 进入后先做什么

1. 先识别当前问题属于认证、鉴权、输入校验、敏感信息、上传下载、外部请求还是浏览器安全。
2. 判断当前实现位于服务端、客户端、网关边界还是浏览器边界。
3. 明确安全控制点应集中在哪一层，不零散复制到每个入口。
4. 先读通用基线，再根据语言 / 框架选择对应的已下载中文 seed 细则。

## 默认执行流程

1. 默认先读 `references/security-baseline.md`，确定认证、鉴权、输入、敏感信息和高风险入口的通用基线。
2. 如果需要区分与请求头、请求参数、错误处理、日志 trace 的边界，再读 `references/security-boundaries.md`。
3. 如果涉及具体语言 / 框架实现，再读 `references/framework-seed-selection.md`，按栈选择 `downloaded-seeds/security-best-practices/` 下对应中文 seed 文件。
4. 输出安全控制点、集中位置、禁止做法、风险说明和是否需要回流其他位点 skill。
5. 安全边界未稳定前，不把实现继续包装成普通接口或性能优化问题。

## 权责边界与不负责事项

- 只负责认证、鉴权、输入校验、敏感信息和高风险入口安全基线，不负责 Header 分类和透传契约，那属于 `request-header-rules`。
- 不负责请求参数模型落点，那属于 `api-request-rules`，但会要求“所有输入都必须校验”。
- 不负责错误处理路径与降级策略，那属于 `error-handling-rules`，但会要求错误不能泄露敏感信息。
- 不负责日志字段设计，那属于 `logging-trace-rules`，但会要求日志不能打印密钥、令牌和敏感数据。
- 不把“前端隐藏按钮”“只靠 CORS”“只靠客户端状态”当成最终授权边界。

## 需要暂停并确认的条件

- 当前实现把真正的权限控制放在前端或可伪造 Header 上。
- 当前 diff 中出现真实密钥、访问令牌、签名密钥、密码哈希明文或高敏感个人数据。
- 当前安全逻辑零散分布，准备一次性大改多层实现，但未先固定控制边界。
- 当前规则涉及 Cookie、JWT、外部请求、上传下载等高风险入口，但缺少运行环境和部署前提信息。

## 执行通过 / 驳回标准

- 通过：认证和授权控制点清楚，输入校验结构化，敏感信息未泄露，高风险入口有明确限制，且框架细则与通用基线一致。
- 驳回：安全控制分散且可绕过、输入未校验、敏感信息暴露、前端承担最终授权，或上传下载 / 外部请求等高风险入口缺少基本防护。

## 执行结果归档要求

- 将安全结论记录到评审记录、任务记录、安全说明或交付说明中。
- 归档内容至少包含安全场景、控制点位置、适用栈、主要风险、禁止做法和遗留问题。
- 如果使用了具体框架 seed 细则，需记录命中的 seed 文件和与当前实现相关的规则范围。

## references 读取规则

- 默认先读 `references/security-baseline.md`。
- 只有在职责边界不清时，再读 `references/security-boundaries.md`。
- 只有在进入具体语言 / 框架实现时，再读 `references/framework-seed-selection.md` 并打开相应的中文 seed 文件。
