# initial-discovery 条件路由

> 路由 marker：`initial-discovery`
> 物理 owner：`requirement-intake-rules`
> 原独立入口已迁移（已迁移，删除前仅作为冻结基线）

## 目的与适用范围

本路由负责把粗略 idea、老板式方向和可主动查证的资料缺口转成有证据来源的需求设计，再回到 `requirement-intake-rules` 的唯一主需求文档入口。它不创建第二份长期需求主文档，不代替边界、拆分、变更、验收标准或实施规划。

## 自动触发信号

- 用户只提出一句话 idea、粗略想法、老板式方向、新功能愿景或“想做某个能力”。
- 用户明确要求 agent 自己找资料、找数据、看代码、查上下游、查 GitHub、相关网站或官方 API 文档。
- 用户补充本地项目路径、URL、数据库名、服务名、文档路径、截图、资料包、GitHub 仓库或 API 文档入口。
- 当前需求信息缺失，但缺失项可能通过当前项目、数据库 schema、代码、上下游、历史资料或用户补充线索主动获得。
- 主入口准备向用户追问大量上下文前，应先进入本路由侦察。

## 核心保护规则

- 先侦察，再追问；一次只推进一个来自真实证据的关键问题。
- 只读优先；代码、文档、配置、schema、URL、本地项目和官方文档可以主动读取。
- 所有数据库、缓存、消息队列、HTTP/RPC 和服务连接只能来自 local 配置；test、staging、pre、release、prod/production 一律阻断。
- 证据优先；关键判断必须可追溯到真实文件、命令输出、schema、接口文档、URL 内容或用户明确补充。
- 无证据内容只能写成候选方案、待验证假设或待用户拍板问题。
- 官方文档、官网、自有仓库和自有站点优先；公共 GitHub、社区资料和第三方公开文档只能作为补充证据，不能单独定案。
- 已验证且后续可复用的路径、URL、数据库、表、服务、接口、搜索关键词和侦察经验必须交给 `project-memory-rules` 回写；不得记录敏感凭据或未验证猜测。
- 需求主文档真实落盘前，禁止进入实现、补丁式编码或实施规划。

## 进入后执行

1. 用一句话复述原始 idea 和当前理解，不扩写成确定需求。
2. 列出当前仓库、项目记忆、需求/实施/验收文档、代码、配置、数据库线索、上下游、关联项目、URL、GitHub 和官方 API 文档等侦察入口。
3. 先读长期记忆，再按只读方式寻找相似功能、相关表、接口、页面、配置和上下游调用。
4. 将证据分为已确认事实、可复用素材、补充参考、候选方案和仍需拍板的问题。
5. 形成 2-3 个可行方案；只有一个明显方案时说明其他方向不合适的证据。
6. 如需读取 local 真实环境，按测试域本地环境发现与隔离规则执行，不回退到非 local 环境。
7. 对每个重要结论标注来源，不用“应该”“大概”“看起来”替代证据。
8. 将可复用线索交给 `project-memory-rules`，再回到主入口生成或更新同一份需求主文档。
9. 已具备可开发条件时转主入口成文；仍存在侦察无法解决的关键缺口时进入 `gap-routing`，只保留真正阻断项。

## 暂停与拒绝

- idea 连目标对象都无法判断，无法确定侦察方向。
- 需要未授权私有仓库、非 local 环境、付费接口、登录态页面或敏感凭据。
- 多方案都成立且业务目标、成本、风险或优先级差异明显，需要用户拍板。
- 资料互相冲突且无法由官方文档、自有代码或当前项目事实裁决。
- 用户补充路径或 URL 无法访问，且项目内没有替代资料。

## 通过标准

通过：输出有来源的资料清单、项目现状、可用数据/代码/上下游、方案对比、推荐方案、风险和最少待确认问题；可复用线索已回写长期记忆；具备条件时已转回主入口落盘。

驳回：粗略 idea 直接退回用户补完整需求；未查项目就写大量猜测；关键结论无来源；使用非 local 环境；或需求主文档未落盘就进入实施/编码。

## 归档要求

- 侦察结论回写同一份需求主文档，不单独建立长期侦察文档。
- `project-memory-rules` 的长期记忆回写至少包含别名、类型、定义、来源、适用范围、更新时间和状态。
- 最终收口前按 `artifact-delivery-gate-rules` 核对资料、方案、待确认项、图片决策、图形和追踪是否真实落盘。

## 条件引用

- 默认读取 `references/initial-discovery-checklist.md`。
- 涉及证据等级、用户线索、外部来源和记忆回写时读取 `references/initial-discovery-evidence-and-memory.md`。
- 组织侦察输出时读取 `references/initial-discovery-output-template.md`。
- 判断需求域承接、合并边界和下游移交时读取 `references/initial-discovery-domain-routing.md`。
- 需求文档落地时继续读取 `artifact-delivery-gate-rules` 的普通语言与审查验收契约。
