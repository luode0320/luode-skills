# 项目知识来源、冲突与编码共享契约

本契约由 `project-memory-rules` 与 `project-style-rules` 共同引用，只定义两类项目知识资产共享的来源、冲突和写入底线；事实记忆与代码风格仍由各自 Skill 独立负责。

## 来源优先级

1. 当前项目代码与配置中的可验证事实。
2. 最近一次用户明确确认的对话口径。
3. 当前项目已有文档。
4. 旧记忆或旧风格内容。

低优先级来源不得覆盖高优先级证据。当前代码和用户最新明确口径冲突时，先判断用户是在声明新目标还是描述当前事实；无法裁决时不得写成已启用真相。

## 冲突处理

- 同一事实或写法只是更新时，覆盖原主条目并记录必要的变更原因，不创建同义竞争条目。
- 两种口径确需并存时，必须写明适用范围、上下文或生命周期状态。
- 证据不足时保持未写入、`conflicted` 或等价非启用状态，不得猜测补全。
- `PROJECT_MEMORY.md` 只承载稳定规则、关键决策和长期事实；`PROJECT_STYLE.md` 只承载当前项目可复用的代码风格样例，禁止互相替代 Owner。

## UTF-8 写入契约

- 新增或修改 `PROJECT_CURRENT.md`、`PROJECT_MEMORY.md`、`PROJECT_HISTORY.md`、`PROJECT_STYLE.md` 时必须显式使用 UTF-8。
- 写入后必须回读关键段落并检查差异，确认中文未乱码、换行未发生无关批量漂移。
- 不得使用依赖 shell 默认编码的写入方式；Windows 命令行场景继续遵守仓库级 UTF-8 规则。
