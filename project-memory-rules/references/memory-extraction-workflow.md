# 抽取与回写流程

## 目标

统一长期记忆的抽取、定位、合并和回写顺序，避免“直接改正文词条”的旧心智。

## 标准流程

1. 读取整个 `PROJECT_MEMORY.md`。
2. 定位底部 `## 机器索引区`。
3. 若机器索引区缺失，补齐最小骨架。
4. 从代码、项目文档和当前对话中提取候选事实。
5. 为候选事实补齐类型、作用域、证据、上下文和生命周期判断。
6. 按 `entity_id + 类型 + 作用域 + 别名 + 来源` 查找现有实体。
7. 优先更新原实体；只有未命中时才新增实体。
8. 同步补齐关系、证据和上下文索引。
9. 机器索引区更新完成后，再同步人类阅读区。
10. 若本次更新会影响后续理解，再更新 `变更记录`。

## 合并优先级

1. 当前项目代码
2. 最近明确对话
3. 已有项目文档
4. 旧记忆内容

## 阻断条件

- 事实来源不明确
- 证据缺失
- 身份无法稳定定位
- 两个高优先级来源互相冲突且无法裁决

出现阻断条件时，不得直接写入“启用事实”。
