# 关系类型定义

## 目标

统一 `relations` 的命名和语义，降低后续检索与推理歧义。

## 推荐关系类型

| 类型 | 含义 | 示例 |
| --- | --- | --- |
| `depends_on` | 依赖 | 报表指标依赖登录事件 |
| `derived_from` | 由...推导 | 活跃用户由登录事件推导 |
| `maps_to` | 映射到 | DTO 字段映射到表字段 |
| `alias_of` | 别名关系 | 登录用户是活跃用户别名 |
| `defined_in` | 定义于 | 规则定义于某方法或脚本 |
| `produced_by` | 产出于 | 指标由某任务生成 |
| `consumed_by` | 被...消费 | 缓存键被某模块消费 |
| `upstream_of` | 上游关系 | 事件是报表上游 |
| `downstream_of` | 下游关系 | 报表是事件下游 |
| `replaces` | 替代 | 新口径替代旧口径 |
| `conflicts_with` | 冲突 | 两种口径互相冲突 |

## 使用规则

1. 关系名优先体现稳定业务语义，而不是一次性描述。
2. 方向必须清晰，避免同一关系双向混用。
3. 关系若因口径切换失效，应同步更新状态。
