# 检索模式

## 目标

定义后续 agent 和 skill 读取项目长期记忆时应如何命中 `PROJECT_MEMORY.md`。

## 默认检索顺序

1. 先读取整份 `PROJECT_MEMORY.md`。
2. 优先在机器索引区按结构化键检索。
3. 再回看人类阅读区中的主题表达。

## 推荐命中方式

### 按实体检索

- 精确匹配 `entity_id`
- 匹配 `name`
- 匹配 `aliases`

### 按作用域检索

- `scope`
- `contexts`
- 相关模块或流程节点

### 按来源检索

- 代码路径
- 方法名
- 文档路径
- 对话确认

### 按状态检索

- `active`
- `deprecated`
- `stale`
- `conflicted`
- `retired`

### 按关系检索

- 上游 / 下游
- 映射关系
- 替代关系
- 冲突关系

## 检索原则

1. 需要稳定判断时优先看机器索引区。
2. 需要快速向用户解释时，再引用人类阅读区。
3. 机器区与正文区冲突时，优先回看证据来源并更新主文档，不直接凭旧正文下结论。
