# 检索模式

## 目标

定义后续 agent 和 skill 读取项目本地记忆时应如何按四件套启动，并在需要时命中 `PROJECT_MEMORY.md`。

## 默认检索顺序

1. 先读取项目目录父目录的当前平台规则文件。
2. 读取 `PROJECT_CURRENT.md`，获得当前目标、状态、待办、阻断和交接点。
3. 读取整份 `PROJECT_MEMORY.md`，获得稳定规则和关键决策。
4. 优先在机器索引区按结构化键检索。
5. 再回看人类阅读区中的主题表达。
6. 普通启动不读 `PROJECT_HISTORY.md`；历史追问、状态不足或真实卡点时，按日期、关键词或事件类型窄读。

## 推荐命中方式

### 按实体检索

- 精确匹配 `entity_id`
- 匹配 `name`
- 匹配 `aliases`

### 按作用域检索

- `scope`
- `contexts`
- 相关模块或流程节点

### 按来源检索

- 代码路径
- 方法名
- 文档路径
- 对话确认

### 按状态检索

- `active`
- `deprecated`
- `stale`
- `conflicted`
- `retired`

### 按关系检索

- 上游 / 下游
- 映射关系
- 替代关系
- 冲突关系

## 检索原则

1. 需要当前任务判断时优先看 `PROJECT_CURRENT.md`。
2. 需要稳定判断时看 `PROJECT_MEMORY.md` 的机器索引区。
3. 需要快速向用户解释时，再引用人类阅读区。
4. 机器区与正文区冲突时，优先回看证据来源并更新主文档，不直接凭旧正文下结论。
5. 历史文件只作为按需证据，不能覆盖当前代码、当前对话或当前状态。
