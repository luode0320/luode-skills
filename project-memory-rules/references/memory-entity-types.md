# 实体类型定义

## 目标

给 `entities` 提供统一类型集合，避免每次抽取长期记忆时随意命名。

## 推荐实体类型

| 类型 | 适用对象 | 示例 |
| --- | --- | --- |
| `术语` | 业务术语、口径名、概念名 | 活跃用户 |
| `指标` | 统计值、报表指标、计算结果 | 7 日留存率 |
| `规则` | 业务规则、校验规则、状态约束 | 提现风控规则 |
| `字段` | 表字段、DTO 字段、接口字段 | `user_id` |
| `表` | 数据库表、缓存表、索引表 | `user_login_log` |
| `缓存键` | Redis 键、缓存命名规则 | `user:profile:{id}` |
| `常量` | 稳定常量、枚举值、标志位 | `OrderStatusPaid` |
| `方法` | 值得长期记住职责的方法 | `UserService.CountActiveUsers()` |
| `模块` | 模块、包、目录、服务边界 | 用户服务 |
| `脚本` | 自动化脚本、迁移脚本、生成脚本 | `bootstrap_agents.sh` |
| `流程节点` | 业务流程、状态流转节点 | 审批通过 |
| `配置项` | 配置键、环境变量、开关 | `APP_ENV` |

## 选型规则

1. 优先使用最稳定、最能复用的类型。
2. 同一实体不要因为不同表述反复切换类型。
3. 若同一事实既可看作术语又可看作指标，优先选择后续复用价值更高的类型。
4. 类型不明确时，先回看代码与文档证据，再决定是否入库。
