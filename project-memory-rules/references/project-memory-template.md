# 项目记忆模板

把这份文件作为项目根目录的唯一长期记忆文件。

## 目的

- 只保留一份权威长期记忆来源。
- 只写明确、可直接使用的事实。
- 让后续对话和代码阅读复用同一套权威表述。

## 建议章节

1. `核心记忆`
2. `术语表`
3. `指标与口径`
4. `表与字段映射`
5. `缓存、变量与常量映射`
6. `方法与调用映射`
7. `计算规则`
8. `业务约束`
9. `废弃术语`
10. `变更记录`

## 条目格式

每条事实尽量用简短的块或表格表示。

```md
### 活跃用户
- 别名: 登录用户, active user
- 类型: 指标
- 定义: 按登录行为统计的活跃用户数
- 来源: `UserService.CountActiveUsers()`
- 适用范围: 日报
- 更新时间: 2026-06-26
- 状态: 启用
```

字段规则：

- 标题、章节名、条目标题和字段名必须使用中文。
- 英文只允许作为代码符号、文件名、数据库字段名、原始接口字段名或别名保留。
- 不要使用英文字段名作为记忆字段名。

## 合并检查清单

- 先读当前主文档，再开始编辑。
- 只补充对话、代码或文档里已经明确的事实。
- 把别名规范到一个主词条下。
- 去重时保留最清晰、最权威的表述。
- 旧说法过时后，直接在原位置替换。
- 已废弃内容直接标记 `状态: 废弃`。
- 保持文档紧凑、易扫描。

## 什么算记忆

- 反复被提到的指标定义。
- 从代码里读出来的字段含义。
- 具有稳定业务意义的缓存键或变量。
- 计算公式或业务规则。
- 值得长期记住职责的方法。

## 什么不该写进来

- 未确认的猜测。
- 临时讨论笔记。
- 重复的原始聊天记录。
- 另建一份承载同类长期记忆的日志文件。
