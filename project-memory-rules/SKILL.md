# 项目记忆规则

## 核心目标

维护项目根目录下唯一的长期记忆文档 `PROJECT_MEMORY.md`，把项目里反复出现、需要长期保留的事实统一沉淀到这一份主文档里。

## 适用场景

- 记录项目中的指标、参数、表字段、缓存键、变量、公式、常量、状态、方法映射或别名。
- 通过阅读代码归纳业务含义，并把结论写回记忆文档。
- 对话中出现新的明确事实后，立即补入主文档。
- 用户后续调整某个指标、口径、字段含义或业务规则时，立即回写原词条。
- 以后遇到同类问题时，先读取这份记忆文档再回答。

## 默认流程

1. 先读取当前的 `PROJECT_MEMORY.md`，再开始分析项目事实。
2. 再读取当前对话、相关代码和已有项目文档。
3. 只抽取明确、可直接使用的事实。
4. 先用条目标题、别名、来源、适用范围定位是否已有同一词条。
5. 若已有同一词条且事实被修订，直接更新原词条，不新增平行冲突条目。
6. 规范化名称，合并别名，去除重复项。
7. 必要时在 `变更记录` 追加一条简短更新说明。
8. 直接重写主记忆文档，让新事实立刻可用。
9. 后续对话优先使用更新后的主文档。

## 写入规则

- 只保留一个长期记忆文件：`PROJECT_MEMORY.md`。
- 不创建 `PROJECT_MEMORY.log.md` 或其他临时记忆文件。
- 不保留待确认区、草稿区或临时区。
- `PROJECT_MEMORY.md` 的标题、章节名、条目标题和字段名必须使用中文；英文只允许作为代码符号、文件名、原始字段名或别名保留。
- 不写猜测内容，事实不明确就不写入。
- 两条描述有重叠时，合并成一个主词条。
- 同一指标、字段或规则如果只是口径更新，必须回写原词条，不得额外新增一个同义冲突词条。
- 旧说法过时了，就直接替换为当前权威说法，并同步刷新 `更新时间`。
- 新旧事实确实需要并存时，必须按 `适用范围` 拆开；若旧事实已失效，则直接标记为 `状态: 废弃`。
- 发生口径切换、字段改名或规则修订时，若会影响后续理解，必须在 `变更记录` 中追加一条简短原因说明。
- 某条事实已废弃时，直接在主文档里标记为 `状态: 废弃`，不要迁移到别的文件。

## 来源优先级

- 优先当前项目代码。
- 其次优先最近一次明确的对话确认。
- 再次优先已有项目文档。
- 最后才参考旧记忆内容，而且只能在没有更新来源时使用。

## 记忆条目结构

每条内容尽量短、稳定、便于检索。必须使用中文字段名：

- `别名`
- `类型`
- `定义`
- `来源`
- `适用范围`
- `更新时间`
- `状态`

参考 [project-memory-template.md](references/project-memory-template.md) 获取可直接复制的主文档模板和合并检查清单。
