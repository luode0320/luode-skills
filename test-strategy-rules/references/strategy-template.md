# 测试策略模板

主摘要写入 `test/yyyy-MM-DD_HHmmss/<测试任务中文主题>/README.md`，建议至少包含：

1. 任务目标与改动范围。
2. 当前时间戳测试任务目录，或建议拆分出的多个时间戳任务目录。
3. 主要风险来源。
4. 计划采用的测试类型。
5. 优先级排序。
6. 环境与数据前提。
7. 待补测项与原因。
8. 建议流向的具体测试 skill。

## 反例提示

- 把策略摘要写到 `testing/`、`analysis/` 或仓库根目录。
- 在中文说明目录里继续创建多个策略 markdown，而不是直接把摘要放进 README。
- 已经明显是多轮独立测试，却不拆任务目录，只在一个 README 中持续追加。
