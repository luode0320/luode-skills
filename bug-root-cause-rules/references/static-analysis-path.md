# 静态定位路径

## 用途

用于在不进入运行时调试的前提下，沿代码和证据链逐步定位 Bug 根因。

## 默认路径

1. 从异常现象对应的入口开始。
2. 看错误日志、trace 或报错位置。
3. 顺着调用链往内收窄。
4. 对照关键状态、关键分支、关键数据转换。
5. 找到最接近异常产生的位置。

## 常用证据

- 报错堆栈
- 接口日志
- trace 链路
- 调用链关系
- 上下游输入输出差异

## 原则

- 先沿事实走，不先沿猜测走。
- 每收缩一步，都要能说明依据是什么。
- 每收缩一步的关键判断和证据来源，都应写回当前 Bug 根目录的 `README.md` 或同级根因记录文件。
