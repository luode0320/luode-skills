# 项目接口基线字段结构

本文件定义项目级接口测试基线的标准化字段，用于长期维护所有接口的定义、判定规则和测试记录，支持跨上线迭代复用。

## 基线核心字段（必填）

| 字段名 | 说明 | 示例 |
| --- | --- | --- |
| 接口标识 | 全局唯一的接口ID，建议使用模块名_接口名_方法 | order_create_POST |
| 接口名称 | 接口的中文业务名称 | 创建订单接口 |
| HTTP方法 | 标准HTTP方法 | POST / GET / PUT / DELETE / PATCH |
| 接口路径 | 接口的全路径 | /api/order/v2/create |
| 所属模块 | 接口所属的业务模块 | 订单模块 / 用户模块 / 支付模块 |
| 鉴权要求 | 接口是否需要鉴权，以及鉴权类型 | 需要Bearer Token / 不需要鉴权 / 需要签名 |
| 请求参数schema | 核心请求参数的JSON结构定义，必填字段、类型、说明 | `{"userId":"string(必填，用户ID)","amount":"number(必填，订单金额)","currency":"string(必填，币种)"}` |
| 响应结构摘要 | 核心响应字段的JSON结构定义，必填字段、类型、说明 | `{"httpStatus":"number","code":"number(业务返回码)","message":"string","data":{"orderId":"string","status":"string"}}` |
| 业务成功判定 | agent判断接口业务成功的规则 | code == 0 && data.status == "pending" && data.orderId != "" |
| 业务失败判定 | agent判断接口业务失败的规则 | code != 0 || (data.status != "pending" && data.status != "fail") |
| 依赖数据 | 测试需要的依赖数据说明 | 需要已认证用户token、有效币种配置 |
| 数据副作用 | 接口执行后对业务数据的影响 | 创建新订单记录、扣减用户可用额度 |
| 清理方式 | 测试后清理脏数据的方法 | 删除测试创建的orderId对应订单、回滚用户额度扣减 |
| 风险等级 | 接口的风险级别 | P0 / P1 / P2 |
| 是否上线必测 | 是否每次上线都必须测试该接口 | 是 / 否 |
| 最近测试时间 | 上次测试该接口的时间 | 2026-06-30 17:20:00 |
| 最近测试结论 | 上次测试的结论 | 通过 / 不通过 / 待确认 |

## 可选扩展字段
- 接口负责人：接口的业务或技术负责人
- 上线时间：接口首次上线时间
- 废弃标记：是否已废弃，废弃时间和替代接口
- 限流规则：接口的限流策略
- 幂等规则：接口是否幂等，幂等键说明
- 第三方依赖：接口依赖的外部服务名称

## 基线存储规则
- 每个项目统一存放在doc/5-tests/基线/interface-inventory.yaml，YAML格式，可被脚本直接读取
- 新增、修改、删除接口必须更新基线
- 每次上线测试后必须更新对应接口的最近测试时间和结论
- 基线禁止随意删除，废弃接口标记废弃标记即可
