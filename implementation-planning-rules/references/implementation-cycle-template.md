# 实施周期模板：可直接执行版

结论：本模板用于把一个实施周期收敛为可直接执行的闭环；影响：相关人员能看清本周期带来的结果和限制；范围：覆盖当前目标、任务顺序、测试、回滚和周期收口；非范围：不跨周期执行，也不擅自扩展到未列任务；变化：把面向业务的结果说明与执行、追踪细节分开；完成标准：当前任务依次完成实现、真实测试、审查和验收；术语说明：无技术术语需要解释；验证状态：执行前按本地真实验证结果确认。

## 白话正文与附录

H1 后的固定摘要已经说明本周期要带来的结果、影响、完成标准和验证限制。保留当前文档原有标题、层级和顺序；任务、文件、命令、测试、回滚和追踪字段归入文末连续的执行附录与追踪附录，正文开场不展示稳定 ID 或机器状态。审查和验收适用性统一写入 `review_acceptance_gates`，并保留 `appendix_policy: preserve_existing_or_one_terminal_appendix`。

## 周期图片资产决策与边界

- 图片资产决策：`需要` 或 `N/A + 原因 + 证据`。仅在 UI/原型、截图证据、视觉对比、真实产物、空间布局、外观基线或 Mermaid 无法准确表达时选择需要图片。
- Mermaid 边界：任务依赖、流程、时序、状态、数据关系和周期门禁仍必须使用 Mermaid；图片不能替代任务依赖图、流程图、时序图或其它既有门禁。
- 生成与引用：真实生成走 `imagegen`，目标为 `doc/data/images/<document_stem>.<asset-slug>-v<number>.<ext>`；正文只允许 `/` 分隔相对路径（例如 `../data/images/<document-stem>.login-state-v1.png`），禁止绝对/越界/远程/HTML/Base64/反斜杠路径及旧 `doc/data/<file>` 路径。
- 格式校验：PNG 用于 UI/截图/文字密集图，JPEG 用于照片，WebP 仅在渲染器兼容性有证据时允许；扩展名必须与文件签名一致。
