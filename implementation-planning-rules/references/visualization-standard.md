# 实施文档图形化标准

## 语义匹配

| 问题 | 必选图形 | 说明 |
| --- | --- | --- |
| 周期/任务依赖 | `flowchart` | 节点使用 `CYCLE-*` / `TASK-*` |
| 多角色调用 | `sequenceDiagram` | 角色、调用、返回和异常必须与接口契约一致 |
| 状态迁移 | `stateDiagram-v2` | 每个状态和迁移条件必须在规则表出现 |
| 数据关系 | `erDiagram` | 表、字段和约束必须与 SQL/数据契约一致 |
| 风险分支 | `flowchart` + 决策表 | 阻断边必须指向停止或回滚，不得静默跳过 |

## 数量与质量

- 实施总览至少包含系统/边界图、周期依赖图和端到端流程或时序图（Mermaid 图块总量 >= 3）；实施周期至少包含任务 DAG 和一个与本周期领域匹配的图（Mermaid 图块总量 >= 2）；两者都必须至少含 1 张流程图，其余为语义补图。机器门禁按 profile 的 `min_diagrams`（总量下限）与 `diagrams`（按类型下限）联合校验。
- 每张图前写目的、关联 ID 和正文回指；图后写异常/边界覆盖说明。
- 图形不能替代字段、表格、SQL、测试命令或回滚步骤；它们只是降低跨模型交接歧义。
- 节点 ID、任务 ID、周期 ID、状态名和正文术语必须一致；不得使用无法在正文定位的缩写。
- 提交前用 Mermaid 解析器或仓库提供的静态检查验证代码块闭合、首行类型和非法占位词；解析器不可用时必须记录为验证阻断。
## Markdown 图片资产边界

- Mermaid 继续负责流程、时序、状态、依赖和数据关系；图片只能补充 UI/原型、截图证据、视觉对比、真实产物、空间布局、外观基线或 Mermaid 无法准确表达的内容，不得替代既有 Mermaid 图。
- 需要图片时，计划必须写 `图片资产决策：需要`，并冻结生成输入、目标文件名、`IMG-*`、用途、关联 `REQ/RULE`、`AC`、`CYCLE`、`TASK`、引用章节和相对路径；不需要时写 `N/A + 原因 + 证据`。
- 图片统一生成到 `doc/data/images/`，Markdown 只允许从当前文档位置计算的 `/` 分隔相对路径；禁止绝对路径、反斜杠、`file://`、Base64、HTML `<img>`、外部热链、越界路径和 `doc/data/<file>` 旧路径。
- 每个图片引用必须有非空 alt，alt 中带 `IMG-*`；计划与实施周期应维护图片资产清单，记录版本、来源、敏感状态和版权状态。真实生成必须走 `imagegen`，失败时阻断，不得用程序绘图或占位图冒充。
