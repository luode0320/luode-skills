# 实施文档图形化标准

## 语义匹配

| 问题 | 必选图形 | 说明 |
| --- | --- | --- |
| 周期/任务依赖 | `flowchart` | 节点使用 `CYCLE-*` / `TASK-*` |
| 多角色调用 | `sequenceDiagram` | 角色、调用、返回和异常必须与接口契约一致 |
| 状态迁移 | `stateDiagram-v2` | 每个状态和迁移条件必须在规则表出现 |
| 数据关系 | `erDiagram` | 表、字段和约束必须与 SQL/数据契约一致 |
| 风险分支 | `flowchart` + 决策表 | 阻断边必须指向停止或回滚，不得静默跳过 |

## 数量与质量

- 实施总览至少包含系统/边界图、周期依赖图和端到端流程或时序图；实施周期至少包含任务 DAG 和一个与本周期领域匹配的图。
- 每张图前写目的、关联 ID 和正文回指；图后写异常/边界覆盖说明。
- 图形不能替代字段、表格、SQL、测试命令或回滚步骤；它们只是降低跨模型交接歧义。
- 节点 ID、任务 ID、周期 ID、状态名和正文术语必须一致；不得使用无法在正文定位的缩写。
- 提交前用 Mermaid 解析器或仓库提供的静态检查验证代码块闭合、首行类型和非法占位词；解析器不可用时必须记录为验证阻断。
