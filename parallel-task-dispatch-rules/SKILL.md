# 并行与子代理统一分发规则

## 目标

- 本 skill 是并行分类、子代理委派、真实启动、生命周期观测、结果回收和串行回退的唯一自动入口。
- 每轮只执行一次统一状态机，不再把“能否并行”和“是否真实启动”交给两个竞争入口重复判断。
- 自动触发、用户习惯、授权、安全、停止、回滚和可见输出协议保持不变；详细矩阵、模板和平台差异下沉到 references。

## 统一状态机

1. **识别边界**：列出主目标、当前关键路径、独立问题、证据来源、模块、文件集和依赖关系。
2. **分类任务**：判定为 `必须串行`、`条件并行` 或 `可并行`；分类规则见 `references/task-classification.md`。
3. **核算成本**：若子线程需要重复读取主 agent 已掌握的大段共享上下文、重复扫描同一批核心文件，或启动成本高于串行收益，回退主线程本地执行。
4. **冻结职责**：主 agent 保留关键阻塞步骤、统一决策、冲突裁决和最终收口；每个子线程必须有明确输入、输出、验收标准和停止条件。
5. **冻结写集**：代码或文档写任务必须声明互斥 write set；任意两个活跃线程不得写同一文件，边界未冻结时不得启动。
6. **检查能力与授权**：系统规则和工具元数据优先；再判断当前轮授权、项目级完全授权及用户明确禁止信号。项目级完全授权不能覆盖系统限制或用户当前轮禁止。
7. **生成启动计划**：记录逻辑中文名、任务、依赖、write set、收口负责人和批次；批量计划按需使用 `scripts/generate_subagent_plan.py`。
8. **真实启动**：只有真实调用当前平台提供的 subagent / multi-agent / thread 工具并取得返回记录，才能声明“已启动”或“已并行”。
9. **主路径继续**：启动 sidecar 后，主 agent 立即推进不重叠的本地工作；不得把下一关键阻塞步骤外包后原地空等。
10. **观测与回收**：收取完成、失败或放弃结果，核验输出和写集，由主 agent 处理冲突并裁决是否接纳。
11. **关闭释放**：结果已收回且不再使用时，立即调用当前平台真实 `close_agent` 或等价工具；不存在该工具时按平台契约记录等价回收，不伪造调用。
12. **输出状态**：最终对齐计划线程数、实际启动数、完成数、关闭数、未启动/未关闭原因和串行回退结果。

## 分类与委派硬规则

- 只读任务，或能按独立问题、证据来源、目录、模块、职责或互斥文件集窄切分的任务，优先评估并行。
- 彼此相关但能在冻结边界后独立推进的任务判定为条件并行。
- 单一根因、需求边界、接口契约、schema、架构方向、高风险删除/迁移/发布以及下一关键阻塞步骤必须由主 agent 串行裁决。
- 审查、项目分析、需求侦察、Bug 证据收集、测试策略和影响面盘点可拆 sidecar，但不得要求子线程重新通读主线程已收敛的大段上下文。
- 能拆分不等于值得拆分；上下文重复读取成本和启动成本未通过门槛时必须串行。
- 用户当前轮明确禁止子代理时不得启动；若原计划依赖并行才能推进，则按真实阻断或本地替代路径处理。
- 不得绕过系统安全限制、权限审批、高风险操作保护或平台工具元数据。

## 授权与平台优先级

授权判定顺序固定为：

1. 系统与开发者规则。
2. 当前平台真实工具元数据和能力。
3. 用户当前轮明确允许或禁止。
4. 项目级 standing authorization。
5. 默认本地串行回退。

- 本仓库声明的完全授权可满足“需要用户显式授权”的工具前提，但不能覆盖更高优先级限制。
- 当前项目没有完全授权、当前轮也没有授权且工具要求显式授权时，输出“工具授权不支持自动启动”，实际启动数为 0。
- 环境没有真实并行工具时，输出“环境不支持真实并行执行”，回退串行；禁止用线程分配文本冒充已启动。
- 平台差异和等价回收方式见 `references/delegation-decision-matrix.md` 与 `references/blockers-and-fallbacks.md`。

## 生命周期与并发上限

- 同时活跃且未回收的子 agent 数不得超过 5；批量计划必须分批，上一批全部完成并关闭后才能启动下一批。
- 启动后立即记录：逻辑中文名、平台昵称、agent id、线程、执行 skill、任务和当前活跃数。
- 完成后立即记录：结果摘要、接纳/驳回结论、完成状态、关闭结果和回收后活跃数。
- 平台昵称由工具返回；若工具不支持自定义昵称，不得把平台英文昵称误报为中文命名失败。
- 完成不等于关闭。结果已收回、已放弃、长时间无有效进展或用户取消时，必须及时关闭或执行平台等价回收。
- 不设固定分钟数作为唯一空闲标准；以“结果已收回且不再使用、已放弃、用户取消”等状态信号为准。

## 可见输出协议

进入实质执行前或首条等价命中证据中输出：

- `并行技能:<skill1,skill2,.../无>`
- 并行判定：`必须串行 / 条件并行 / 可并行`
- 线程边界与 write set
- 主线程收口负责人

真实启动与收口至少记录：

- `计划线程数:<N>`
- `实际启动数:<N>`
- `完成数:<N>`
- `关闭数:<N>`
- `回退原因:<无/具体原因>`

启动、完成公告格式见 `references/thread-template.md`；批量结构见 `references/launch-plan-schema.md`。只输出计划、公告模板或并行技能列表而没有真实工具调用，视为未启动。

## 失败、冲突与回退

- 写集重叠、边界不清、结果影响关键决策且无可验证标准、子线程证据冲突无法自动裁决时，暂停该候选并回退主 agent。
- 计划线程数与实际启动数不一致时立即记录差额和原因；不得把计划并行伪装成真实并行。
- 子代理启动或运行异常时联动 `execution-failure-learning-rules`，改变方法后再恢复；不得无变化重试。
- 任何候选失败只回退该候选，不回滚已独立验证通过的并行结果。
- 详细阻断与恢复模板见 `references/blockers-and-fallbacks.md`。

## 权责边界

- 本 skill 只负责“分类、拆分、授权检查、真实启动、观测、回收、关闭与回退”。
- 领域事实、业务决策、代码实现、真实测试、审查、验收和最终用户总结仍由对应 Owner Skill 负责。
- 主 agent 始终负责最终判断、结果整合、冲突裁决和状态真实性。

## 执行通过 / 驳回标准

- **通过**：统一状态机只运行一次；分类和成本判断有证据；主/子职责与写集清晰；系统和授权边界正确；允许启动时存在真实工具记录；计划线程数、实际启动数、完成数和关闭数可对齐；主路径未空等；失败已真实回退。
- **驳回**：重复触发两套分发判断；写集冲突仍启动；用户禁止或系统不允许时强行启动；只写计划却宣称已并行；关键主路径外包后空等；完成结果未回收关闭；数量不一致且无说明；平台不存在关闭工具却伪造调用。
