# 委派判定矩阵

## 判定顺序

1. 任一 skill 命中并进入执行阶段后，自动启动委派判定，不等待用户明确要求。
2. 先确认当前真实 subagent / multi-agent / thread 工具元数据是否允许自动启动；若工具要求用户显式授权，先检查当前轮授权和项目级完全授权声明。本仓库完全授权模式视为已授权。
3. 是否存在“明确禁止委派”信号。
4. 是否可切分为独立子任务。
5. 评估 subagent 是否需要为入门重复读取主 agent 已掌握的大段共享上下文或重复扫描同一批核心文件；若上下文重叠高且收益有限，回退本地优先。
6. 子任务是否阻塞主路径。
7. 是否存在写集冲突、高风险操作或环境不支持。
8. 检查当前活跃 subagent 数（已启动且未回收）是否已达到 `SKILL.md`「并发上限与空闲回收（强制）」规定的上限 5；已达上限只表示需要先排队，不等同于“不可委派”，待现有 subagent 回收释放槽位后仍应启动。
9. 满足条件且工具授权允许、且活跃数未超上限后必须委派；活跃数已达上限则先排队，否则本地执行。

## 默认策略

- 默认允许并推荐优先使用 subagent，但前提是已经通过上下文重复读取与启动成本门槛。
- 只有命中“明确禁止委派”或其他阻断条件时，才回退主 agent 本地执行。
- 用户没有主动提到 subagent 不属于通用阻断条件，不能作为泛化本地执行理由。
- 本仓库完全授权模式视为用户已给出项目级 standing authorization；若当前真实工具元数据要求显式授权，该声明满足授权条件。
- 只有当前项目没有完全授权声明且当前轮也没有授权时，才属于环境阻断，必须回退本地执行。
- 不要求子任务已经出现在固定 skill 映射表中；主 agent 必须基于任务结构自主识别可委派子任务。
- 项目分析、找 Bug、需求完善侦察、测试策略和影响面盘点等任务，默认先检查是否能拆出只读 sidecar 子任务并行执行。

## 可委派（满足全部）

- 未出现用户明确禁止委派信号。
- 子任务输入、输出和验收标准可写清。
- 子任务不阻塞主 agent 的下一步关键动作。
- 子任务写集与主路径写集不冲突，或天然只读。
- 当前环境支持 subagent 能力。
- 当前轮授权或项目级完全授权允许真实启动 subagent。
- 子任务能按独立问题、独立证据来源、独立资料源、独立目录、独立模块或独立文件集拆分。
- 子任务可以通过窄上下文委派消息限定范围，不需要 subagent 为了入门重复通读主 agent 已收敛的大段共享上下文。

## 不可委派（命中任一）

- 用户明确禁止 subagent / delegation / 并行代理。
- 任务边界无法定义，或依赖实时交互决策。
- 下一步就是主路径关键阻塞步骤。
- 涉及高风险不可逆操作且必须主 agent 统一控制。
- 当前环境不支持 subagent 能力。
- 当前工具元数据要求用户显式授权 subagent / delegation / parallel agent work，且当前轮授权与项目级完全授权均不存在。
- 单一根因、需求边界、接口契约、schema 或架构方向仍未收敛，且子任务会直接替代主 agent 的最终裁决。
- 子任务规模过小，主 agent 一次聚焦读取即可完成，subagent 启动与重复读码成本更高。

## 并发上限与排队（详见 SKILL.md）

- 完整的并发上限数值、空闲判定信号和回收要求以 `SKILL.md`「并发上限与空闲回收（强制）」章节为唯一权威定义，本文档不重复枚举，避免两处口径漂移。
- 本文档只强调一点：活跃数已达上限时，判定结果是“排队等待槽位”，不是“判定为不可委派”；两者的下游动作不同，不要混用。

## 主 agent 可见性要求

- 只要启动 subagent，主 agent 必须立即输出“已启动 + 执行 skill + 任务摘要”。
- subagent 完成后，主 agent 必须输出“已完成 + 执行 skill + 结果摘要”。

## 建议优先委派类型

- 注释补齐、注释位点检查、注释一致性对照。
- 静态审查类检查（格式、语法、规则核对）。
- 可并行的文档同步与清单更新。
- 项目分析中的目录结构、入口链路、模块职责、配置/部署面只读侦察。
- 需求完善中的代码现状、既有文档、外部官方资料、竞品/API 资料收集。
- Bug 分诊中的日志/报错、复现路径、近期改动、调用链和影响面收集。
- 测试策略、回归范围和风险清单的模块化只读分析。

## 建议本地执行类型

- 关键实现路径上的实时编码决策。
- 风险高、强耦合、难以拆分的重构。
- 需要连续上下文推演的主链路分析。
- 最终根因裁决、需求边界冻结、接口契约冻结、schema 决策和架构方向裁决。
- 需要重读主 agent 已收敛的大段共享上下文才能完成的小型侦察、小型审查或少文件核查。
