# 任务分类表

## 可并行

- 多目录 / 多模块项目结构盘点
- 独立资料源或官方文档检索
- 只读代码库侦察
- 代码规范检查
- 注释补充
- 格式清理
- import 清理
- 按独立文件集拆分的 lint 修复
- 按独立模块拆分的测试补充
- 按独立文档区域拆分的文档更新

## 条件并行

- 项目分析中的模块职责、入口链路、配置/部署面并行侦察
- 需求完善中的代码现状、文档现状、外部资料、竞品/API 资料并行侦察
- Bug 分诊中的日志分析、复现路径、近期改动、影响面并行收集
- 按模块拆分的功能开发
- 文件归属清晰前提下的 review 与 cleanup
- 接口冻结后的前后端并行工作
- 不修改同一批文件前提下的 Bug 分诊与验证

## 必须串行

- 单一问题的根因裁决
- 下游工作开始前的 schema 或契约变更
- 重写共享抽象的重构
- 最终会汇聚到同一批文件的工作
- 需求边界、契约、schema 或架构方向尚未冻结前的最终决策

## 自主识别原则

- 本表不是并行白名单；主 agent 必须基于当前任务结构自行判断是否存在可并行子任务。
- 只要能拆成独立问题、独立证据来源、独立文件集或独立模块职责，默认先尝试并行委派。
- 项目分析、找 Bug、需求完善侦察等任务即使没有固定映射 skill，也应先检查是否能拆分 sidecar 子任务；本仓库完全授权模式下，任务可切分且环境支持时应委派子 agent 并行收集证据。
- 需要统一裁决的主路径保持串行；旁路证据收集、影响面盘点、资料检索和只读审计可在边界清晰时条件并行。
