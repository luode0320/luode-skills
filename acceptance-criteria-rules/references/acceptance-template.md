# 验收标准模板

## 推荐结构

1. 场景名称：在哪个业务场景下验证。
2. 前置条件：依赖数据、角色、环境、权限、配置。
3. 输入动作：用户操作、接口调用、数据变化。
4. 预期结果：页面表现、返回结果、状态变化、日志或副作用。
5. 异常分支：失败时应如何表现。
6. 边界条件：空值、极值、重复提交、并发、历史数据。
7. 范围外说明：本次不负责什么。

## 归档要求

- 验收标准文档路径统一为 `ment/YYYY-MM-DD_验收中文说明.md`。
- 验收文档与需求文档分开维护，不要混写到 `ment/YYYY-MM-DD_HHmmss_<需求中文简介标题>.md`。
- 同一个需求只保留一份验收文档，后续补充和变更继续更新这同一份文档。
