# 项目四件套记忆闭环

## 1. 需求目标

建立项目本地四件套记忆机制，并与现有 Obsidian 知识流分层协作：

1. 父目录规则文件保存跨项目通用规则。
2. `PROJECT_CURRENT.md` 覆盖式保存当前任务交接状态。
3. `PROJECT_MEMORY.md` 保存稳定项目规则与关键决策。
4. `PROJECT_HISTORY.md` 只追加关键历史事件，默认不读取。

新项目或新线程必须先读取父目录公共规则，再读取 current 和 memory；文件缺失时先创建最小模板。Obsidian vault 继续只通过 CLI 选择性检索与沉淀。

## 2. 范围与边界

- 纳入：Obsidian skill、project-memory skill、自举规则与脚本、Codex/Claude 规则、四件套实例、字典和验证资产。
- 不纳入：业务代码、远程服务、外部记忆服务、vault 全量迁移和 Git 历史写入。
- `PROJECT_CURRENT.md` 最大 51,200 字节，超限必须阻断，不得静默截断。
- 不记录 secret、token、密码、私钥和连接串原值。

## 3. 兼容要求

- `PROJECT_MEMORY.md` 已有机器索引区继续保留，作为稳定记忆的可选内部结构。
- 现有 vault 笔记不批量改写；本次新决策在收口时通过 CLI 检索后追加到现有承接笔记。
