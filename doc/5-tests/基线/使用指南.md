# 上线接口测试门禁使用指南

## 首次使用步骤

1. 先确认被测项目使用 `config_local*`、`.env.local` 或 `.env.development` 配置；禁止连接 test、staging、pre、release 或 prod 配置。
2. 运行诊断，确认项目、local 配置和 adapter 可用：

```powershell
python -X utf8 project-release-test-rules/scripts/generate_release_test_plan.py doctor `
  --project-root . `
  --baseline-root doc/5-tests/基线
```

3. 执行全链路上线测试。缺省会自动发现接口；已有基线和参数文件可显式传入：

```powershell
python -X utf8 project-release-test-rules/scripts/generate_release_test_plan.py run `
  --project-root . `
  --baseline-root doc/5-tests/基线 `
  --output-dir doc/5-tests/2026-07-12_180240 `
  --inventory doc/5-tests/基线/interface-inventory.yaml `
  --parameter-sources doc/5-tests/基线/parameter-sources.yaml `
  --reusable-params doc/5-tests/基线/reusable-params.yaml
```

4. 需要只生成发现、参数和依赖计划而不发送请求时，增加 `--dry-run`。本地写接口和第三方副作用按 local 配置执行；极端破坏性操作由安全规则硬阻断。
5. `run` 的 JSON 结果、请求/响应脱敏证据、依赖追踪和门禁报告必须落在本轮 `doc/5-tests/<时间戳>/` 目录；`PENDING`、`BLOCKED`、`PARTIAL` 和 `FAIL` 均不得视为允许上线。

## 兼容旧入口

旧十个子命令仍可调用：`bootstrap-inventory`、`reconcile-inventory`、`generate-plan`、`init-release-test-task`、`init-baseline-assets`、`build-dependency-graph`、`validate-reusable-params`、`resolve-test-data`、`update-baseline-assets`、`sync-interface-contract-assets`。检测到新内核时会自动委派；新内核暂不可用时才执行原有资产逻辑。

## 判定规则

- P0 接口任一不通过、待确认、跳过或阻断：直接阻断上线，结论 `FAIL`
- P1 接口任一失败、阻断或待确认：结论 `PARTIAL`，需要人工确认；P1 的 `EXPECTED_FAIL` 必须有显式断言证据
- 所有必测入口通过且无待确认项：结论 `PASS`，允许上线

## 注意事项

- 所有测试资产必须落地到 `doc/5-tests/<时间戳>/` 根目录，遵循测试域归档规则
- 请求参数和响应结果必须输出为 JSON 字符串，脱敏敏感信息
- Agent 判定必须给出明确理由和 `failure_type`，禁止模糊结论；未知业务语义只能输出 `PENDING`
- 完整请求/响应内容归档到受限 ASCII 镜像目录，长期基线只保留脱敏值、hash、引用和 TTL
