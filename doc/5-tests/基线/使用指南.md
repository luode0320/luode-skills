# 上线接口测试门禁使用指南
## 首次使用步骤
1. 生成项目接口基线文件：扫描项目路由、controller、Swagger文档，按照interface-inventory-schema.md格式填写，保存为doc/5-tests/基线/interface-inventory.yaml
2. 每次上线前执行：
`ash
# 生成上线测试计划，--modules参数指定本次上线改动的模块列表
python project-release-test-rules/scripts/generate_release_test_plan.py \
  --inventory doc/5-tests/基线/interface-inventory.yaml \
  --modules "订单模块,用户模块" \
  --output doc/5-tests/2026-06-30_172140/release-test-plan.yaml
`
3. 按照测试计划执行接口测试，Agent自动判定每个接口是否通过
4. 生成测试报告，按照report-format要求输出，禁止使用Markdown表格
5. 得到门禁结论，作为上线审批依据

## 判定规则
- P0接口任一不通过/待确认/跳过：直接阻断上线，结论FAIL
- P1接口不超过2个且不影响核心功能，风险可控：结论PARTIAL，需要人工确认
- 所有P0/P1接口通过：结论PASS，允许上线

## 注意事项
- 所有测试资产必须落地到doc/5-tests/对应时间戳根目录，遵循测试域归档规则
- 请求参数和响应结果必须输出为JSON字符串，脱敏敏感信息
- Agent判定必须给出明确理由，禁止模糊结论
- 完整响应内容归档到ASCII镜像目录，主README只展示摘要
