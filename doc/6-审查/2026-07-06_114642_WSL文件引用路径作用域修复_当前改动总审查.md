## 背景

用户反馈：某会话最终输出里项目内文件路径仍是 `/home/luode/...` 裸路径，而不是 Windows 可打开的 `\\wsl.localhost\...`；`windows-wsl-execution-rules` 里明明已经写过这条规则。排查发现真实根因：`SKILL.md` 与 3 个 references 文件、以及 `PROJECT_MEMORY.md` 的"Windows / WSL 执行边界"词条，都把"面向用户的文件引用要用 `\\wsl.localhost\...`"这条规则表述成挂在"agent 在 Windows"分支下的从句／表格行；"agent 在 WSL"（推荐运行方式）对应段落完全没提这条规则，导致 agent 直接跑在 WSL 内时容易误判该规则不适用。
