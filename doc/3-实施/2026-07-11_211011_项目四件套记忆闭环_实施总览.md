# 需求实施总览：项目四件套记忆闭环

## 1. 当前计划最终方案简要说明

采用项目本地四件套承载启动上下文和项目事实，保留 Obsidian CLI 的跨项目选择性知识流。先冻结文件契约，再同步三个 skill 和自举脚本，最后刷新字典并完成临时仓库、UTF-8 和 CLI 验证。

## 2. 基本信息

- 对应需求文档：`doc/2-需求/2026-07-11_211011_项目四件套记忆闭环.md`
- 对应验收标准：`doc/7-验收/2026-07-11_211011_项目四件套记忆闭环_验收标准.md`
- 当前范围：规则资产、项目记忆初始化、自举、字典、测试、审查和验收。
- 明确不在范围：业务代码、外部记忆服务、vault 全量迁移、Git commit/push。
- 当前状态：实施中。
- 是否获得开始实施授权：是，用户已明确要求实现本计划。

## 3. 实施周期

1. 周期 01：四件套契约、需求和验收文档。
2. 周期 02：Obsidian、project-memory、bootstrap skill 和脚本同步。
3. 周期 03：仓库规则、项目记忆实例、字典和验证收口。

每个周期内按“实现 -> 真实测试 -> 审查 -> 验收”闭环后进入下一任务。

## 4. 代码与文档落点

```text

├── obsidian-knowledge-flow\              # 项目记忆与 vault 分层规则
├── project-memory-rules\                 # current/memory/history 更新规则

├── PROJECT_CURRENT.md                     # 当前交接状态
├── PROJECT_MEMORY.md                      # 稳定规则与关键决策
└── PROJECT_HISTORY.md                     # 追加式历史事件
```

## 5. 真实测试

- quick validator 校验受影响 skill。
- 临时本地仓库运行 bootstrap，验证缺失创建、幂等复跑、current 超限阻断和 history 追加。
- 运行 skill 字典生成脚本并检查生成物。
- 执行 `git diff --check`、UTF-8 回读和 Obsidian CLI `version`/`vaults verbose`/`search`/`read`。

## 6. 完成与停止条件

- 完成条件：四件套职责在规则、skill、模板、自举脚本和验证资产中一致，所有测试与审查通过。
- 停止条件：发现规则冲突、UTF-8 漂移、current 超限未阻断、bootstrap 破坏已有内容或 vault 根目录不一致时停止并记录阻断。
- 最大推进边界：不执行 Git 历史写入，不扩展到业务项目迁移或 vault 全量沉淀。
