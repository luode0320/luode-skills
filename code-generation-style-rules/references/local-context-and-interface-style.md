# 局部上下文与接口实现风格规则

## 适用范围

本 reference 用于处理两类必须优先跟随既有写法的场景：

- 当前文件、同目录或同模块已经形成高度统一的代码结构。
- 本轮新增代码需要实现项目中已经存在的接口。

它只约束当前必要改动范围，不要求借机清洗历史代码或统一整个项目。

## 局部统一风格判定

出现以下任一情况时，视为当前上下文存在局部稳定风格：

- 同一函数中连续出现两处及以上相同结构的调用、注册或赋值。
- 同一文件中同一职责的多个实现使用相同的函数组织、字段顺序、注释位置或错误处理方式。
- 同目录或同模块中存在重复出现且没有已知问题的同构实现。
- 用户明确给出一个要求复制其结构的代码片段。

判定后必须提取局部模板，至少记录：

- 调用或控制流结构。
- 语句顺序和分组方式。
- 空行、缩进、换行和注释位置。
- 字段、参数或注册项的排列方式。
- 错误处理、日志和返回方式。

## 局部模板执行规则

- 新增代码只替换目标名称、目标字段和完成当前需求所需的最小业务内容。
- 保持已有调用链、注册顺序、分组边界、空行、注释和排版方式。
- 不新增 helper、wrapper、factory、manager、临时变量、循环、日志、校验、抽象或其他无需求代码。
- 不把个人偏好、外部模板或“更优雅”的表达方式带入统一代码区段。
- 不因为局部风格规则删除安全、正确性、兼容、错误处理或接口契约要求的代码。
- 发生必要例外时，必须在代码风格契约的“最小新增内容”和“冲突裁决”中说明原因。

用户示例的执行口径：

```go
func Init() {
	changelly.New().Start(true)     // 创建并注册交易所changelly
	swft.New().Start(true)          // 创建并注册交易所swft
	changenow.New().Start(true)     // 创建并注册交易所changenow
	ellipal.New().Start(true)       // 创建并注册交易所ellipal
	swapkit.New().Start(true)       // 创建并注册交易所swapkit
	exolix.New().Start(true)        // 创建并注册交易所exolix
	lets.New().Start(true)          // 创建并注册交易所lets
	simpleswap.New().Start(true)    // 创建并注册交易所simpleswap
	nexchange.New().Start(true)     // 创建并注册交易所nexchange
	nexchangeDefi.New().Start(true) // 创建并注册交易所nexchange defi
	nearIntents.New().Start(true)   // 创建并注册交易所near intents
}
```

新增交易所时，必须保持 `xx.New().Start(true)` 的结构和注册区段排版，只新增对应的一行；不得改成循环、注册表、helper 或其他抽象。

## 接口实现参考规则

实现已有接口时，按以下顺序寻找参考实现：

1. 同一文件中已经实现该接口的类型或方法。
2. 同一目录、同一模块中已经实现该接口的类型或方法。
3. 同一语言项目中职责、依赖和边界最接近的实现。

找到参考实现后，必须比较并尽量保持以下内容：

- 接收者、方法顺序和文件归属。
- 字段组织、构造方式和注册方式。
- 参数校验、错误返回、错误包装和降级方式。
- 日志等级、字段、模板和脱敏方式。
- 中文注释、步骤注释和补丁说明。
- 测试命名、fixture 位置和验证断言。

代码风格契约必须记录参考实现的文件路径、符号和目标实现的必要差异。

## 无参考实现与冲突处理

- 找不到任何既有实现时，必须记录 `N/A + 原因 + 证据`，再采用当前文件和同目录稳定风格；不得静默发明新的接口实现模板。
- 多个实现冲突时，优先采用最近同模块、重复出现次数最多且没有已知正确性问题的写法。
- 如果冲突仍无法收敛，停止编码并记录 `GAP-*`，不得按个人偏好任选一种。
- 局部风格与正确性、安全、兼容、错误处理或接口契约冲突时，保留硬性要求，并在契约中记录例外原因。

## 通过与驳回

通过条件：

- 已记录局部风格证据，或明确记录没有可用局部模板的原因。
- 已记录接口参考实现，或明确记录无参考实现的降级依据。
- 新增代码只做必要替换，没有引入无需求结构。
- 必要例外均有正确性、安全、兼容、错误处理或接口契约证据。

驳回条件：

- 未读取当前文件、同目录或同模块样例。
- 统一代码区段出现个人偏好、外部模板或结构跳变。
- 接口实现没有既有实现对照，也没有无参考实现说明。
- 为几行局部逻辑新增抽象、helper、临时变量、循环、日志或校验。
- 以“风格一致”为理由删除硬性必需代码。
