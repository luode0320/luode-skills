# 仓库协作与交付规则

## 适用范围

用于仓库上下文建立、提交信息、AGENTS.md 编写和交付说明阶段。

## Git 与上下文建立

### 规则 19

每次新建对话时，若具备读取 Git 历史的权限或能力，需至少读取当前分支最近一次提交记录，包含提交信息、修改文件和代码变更内容，以快速熟悉近期开发背景。

## 提交规范

### 规则 16

Git 提交信息默认遵循 `type: 中文描述`；若仓库已有明确提交规范，则优先遵循仓库规范。

## 文档规范

### 规则 18

`AGENTS.md` 必须全部使用中文生成。

## 交付说明

交付说明默认包含以下四项：

1. 改动原因
2. 影响范围
3. 风险点
4. 验证方式

### 规则 24

交付与复盘检查时，必须核对新增或修改的代码块是否满足注释要求；只要代码块存在大于等于 2 个步骤，就应补充中文步骤注释，并用 `1.`、`2.`、`3.` 标识步骤。
