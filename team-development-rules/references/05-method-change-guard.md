# 方法修改保护规则

## 适用范围

用于修改已有方法、函数或核心逻辑前的风险判断。

## 最近修改时间字段

### 规则 22

新增或修改代码时，必须在方法级注释中明确标注该方法的“最近修改时间”字段，格式为北京时间：`yyyy-MM-DD HH:mm:ss`。

示例：

```text
最近修改时间：2026-03-21 14:25:30
```

## 7 天保护约束

日常修改代码过程中，禁止主动改动“最近修改时间”超过 7 天的已有方法。

如果确需修改这类方法，必须先暂停当前实现，并向用户输出以下内容：

1. 修改必要性
2. 风险评估
3. 具体修改方案
4. 验证方式

在得到用户确认之前，不要继续修改该类方法。
