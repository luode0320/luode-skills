# 回归验证规则

只在“这次改动有没有把旧能力带坏”这个问题上使用这个 skill。
如果当前争议是当前需求本身有没有做对，请转交 `functional-validation-rules`；如果当前争议是上下游环境或链路跑不通，请转交 `integration-debugging-rules`。

## Skill 作用与适用场景

- 作为第五波测试链路的收尾规则，负责当前改动后的兼容性与影响面验证。
- 约束回归范围判定、用例选取、验证重点和回归结论留痕方式。
- 聚焦 Bug 修复、原有功能迭代、公共模块修改、共享逻辑调整、接口兼容变化后的影响扩散。
- 防止把“当前功能验证”和“历史能力回归”混成一件事，导致范围失控或遗漏关键旧路径。

## 自动触发信号

- Bug 修复、原有功能迭代、公共模块修改、共享逻辑调整、接口兼容性变化后准备执行测试。
- 当前功能验证已经完成，需要继续确认旧逻辑、已有流程、上下游依赖和兼容行为是否被破坏。
- 变更涉及公共方法、公共组件、共享配置、通用中间件、基础库或复用链路时。
- 用户明确要求确认“修了这个问题，会不会影响别的地方”时。
- 发现回归结论准备记录到 `testing/`、`analysis/` 或其他非 `test/` 任务目录位置时。

## 进入后先做什么

1. 先确认当前需求或当前 Bug 的功能验证已经基本完成，不带着明显未收敛的新问题做回归。
2. 确认当前回归验证已对应到 `test/` 根目录下的测试任务目录；若有多轮独立回归或多个回归程序，应分别建目录。
3. 梳理本次改动直接影响点、间接影响点、共享依赖和上下游关联链路。
4. 判断本轮回归属于局部回归、链路回归还是公共能力回归。
5. 明确哪些旧能力必须验证，哪些风险只需记录待后续补测。

## 默认执行流程

1. 默认先读 `references/regression-scope-selection.md`，确定回归范围和用例选取方式。
2. 如果问题归属不清，可能混入功能验证或联调问题，再读 `references/regression-boundaries.md`。
3. 输出回归结论、风险项和未覆盖说明时，再读 `references/regression-template-and-examples.md`。
4. 给出回归通过、回归驳回或待补测结论，并明确是否需要回流编码域、Bug 域、联调域或测试策略域。
5. 回归范围未稳定前，不直接宣告任务具备交付条件。

## 权责边界与不负责事项

- 只负责旧能力兼容性与影响面验证，不负责当前需求本身是否实现正确，那属于 `functional-validation-rules`。
- 不负责测试优先级总设计和大盘策略制定，那属于 `test-strategy-rules`。
- 不负责联调环境、上下游协议和链路打通问题定位，那属于 `integration-debugging-rules`。
- 不负责测试目录、命名、测试程序和测试文档组织，那属于测试资源管理类 skill。
- 如果回归无法开展，是因为需求、Bug 定义或当前改动本身尚未收敛，应先回流对应上游 skill。

## 需要暂停并确认的条件

- 当前功能验证尚未通过，直接做回归会掩盖主问题。
- 本次改动影响面不清，难以合理界定回归范围。
- 当前回归环境、历史数据或关键链路依赖不具备，结论可信度不足。
- 回归范围已经扩张到全量测试计划，需要升级到 `test-strategy-rules` 重新确定优先级。

## 执行通过 / 驳回标准

- 通过：本次改动涉及的旧能力、关联链路、共享依赖和兼容行为在已判定回归范围内未发现新的破坏性问题，且结论已可追溯。
- 驳回：本次改动引入了旧能力回退、兼容性破坏、公共链路异常，或因关键回归范围未验证而无法证明“没有带坏别的地方”。

## 执行结果归档要求

- 将回归验证结论统一记录到 `test/` 根目录下当前测试任务目录的 `README.md` 或同目录回归说明文件中，不得再写到 `testing/` 或 `analysis/`。
- 归档内容至少包含改动类型、回归范围、选取用例、执行环境、结果结论、未覆盖项和遗留风险。
- 如果结论为待补测，必须写明缺失链路、补测条件和后续责任归属。
- 如果同一需求存在多轮独立回归验证，应分别放在各自测试任务目录中，各自记录回归结论。

## references 读取规则

- 默认先读 `references/regression-scope-selection.md`。
- 只有在功能验证、联调验证、回归验证边界不清时，再读 `references/regression-boundaries.md`。
- 只有在需要回归结论模板和样例时，再读 `references/regression-template-and-examples.md`。
