# Git 协作规则（最小闭环版）

## -1.4 极简硬闸门（新增，强制）
- 识别到 Git 协作意图后，首条中间进度第一行必须输出：
  - `命中检查:通过; Git规则:通过`
- 若未输出该行，仅允许执行盘点命令：
  - `git status --short`
  - `git diff --cached --stat`
- 执行其它 git 命令视为违规，判定 `阻断`。

## -1.5 执行前自检标记（新增，强制）
- 在执行任何 `git add/commit/push/pull/rebase/merge/cherry-pick/stash` 前，必须先在首条中间进度出现：
  - `前置自检:就绪`
- 若没有 `前置自检:就绪`，只允许盘点命令，禁止继续执行其他 git 命令。

## -1.6 违规自恢复（新增，强制）
- 若发生“未完成首条命中检查就执行 git 命令”，必须立即进入恢复模式：
1. 立即停止非盘点 git 命令。
2. 输出 `命中检查:阻断; Git规则:阻断` 与 `自恢复:需要重启`。
3. 重走首条模板 + 证据三要素 + `前置自检:就绪`，完成后才允许继续。

## -1. 触发确认（强制）
当用户请求包含 `git/commit/提交/push/pull/status/diff/log` 任一关键词时，必须命中本 skill；未命中不得执行 git 命令。

## -1.0 新会话首轮保障（强制）
- 新会话第一条消息只要出现 Git 协作意图，必须立刻命中本 skill，不得等待后续轮次确认。
- 即使用户仅输入极短指令（如 `提交git`、`push`、`看下状态`），也必须按命中处理。
- 命中后第一条中间进度先输出证据模板，再执行除盘点外的 Git 命令。

## -1.1 语义触发扩展（强制）
以下表达即使不出现关键词，也必须命中本 skill：
- `提交git`、`帮我提交`、`commit一下`、`把改动提交到分支`
- `推送代码`、`给我推上去`、`同步到远端`
- `看下git状态`、`看下改动`、`看看最近提交`

判定原则：只要用户意图是让代理执行或检查 Git 协作动作，即视为命中，不得以“缺少明确 git 关键词”回避。

## -1.2 联动前置（强制）
- 本 skill 一旦命中，必须同时命中 `skill-hit-check-rules`。
- 若首条中间进度缺少“Skill 命中检查”或缺少 Git 三要素证据，则 `阻断`，除盘点命令外禁止继续执行 Git 命令。

## -1.3 口语与缩写兜底（强制）
以下输入默认视为 Git 协作，不得漏判：
- 单词/短句：`提交`、`提交代码`、`提一下`、`交一下`、`推一下`、`拉一下`、`同步代码`
- 中英混写：`提交git`、`git提交`、`commit下`、`push下`
- 省略主语命令式：`帮我弄到远端`、`把这次改动交了`

## 0. 首条中间进度（强制）
必须先输出并填写模板：
- `templates/evidence-template.md` 中的 **Skill 执行证据** 区块。
- 且必须追加一行：`前置自检:就绪`

## 1. 执行短清单（强制按顺序）
1. 盘点：`git status --short`、`git diff --cached --stat`
2. 业务拆分：可按改动主题拆分为多次提交（不要求一次提交完成）
3. 写 README 日志：根目录 `README.md` 增加 `yyyy-MM-DD HH:mm:ss <提交标题>`
4. 设定标题：`<type>: [中文简要说明] 标题说明`
5. 提交前闸门：优先执行 `scripts/pre_commit_gate.sh "<title>"`，失败即阻断
6. 执行提交：`git commit`（正文必须真实换行，禁止字面量 `\\n`）
7. 提交后闸门：优先执行 `scripts/post_commit_gate.sh`，失败即 `git commit --amend` 修正
8. 若用户请求“提交git”：重复 1-7，直到 `git status --short` 为空
9. 最终回执：按时间顺序列出本次执行产生的全部 commit（hash + 标题），用于复核多次提交是否完整覆盖

## 1.1 脚本缺失回退（强制）
若仓库不存在 `scripts/pre_commit_gate.sh` 或 `scripts/post_commit_gate.sh`，必须执行等价回退检查，且在最终证据中标注 `回退`：

- pre 回退检查：
  - 标题匹配 `<type>: [中文简要说明] 标题说明`
  - `README.md` 包含本次提交标题
  - Go 项目不存在 `test/` 外 `*_test.go`
  - staged 不命中 `internal/service/[^/]+.go`
- post 回退检查：
  - `git log -1 --pretty=%B` 不包含字面量 `\\n`

缺任一回退检查视为未完成，不得给“已完成”。

## 2. 阻断条件（脚本化/回退等价）
以下任一命中必须停止提交并回报阻断：
- 标题不匹配 `<type>: [中文简要说明] 标题说明`
- `README.md` 未包含本次提交标题日志
- Go 项目存在 `test/` 外 `*_test.go`
- staged 命中 `internal/service/[^/]+.go`
- 最新 commit 正文含字面量 `\\n`

## 3. 统一证据输出（强制）
最终回复必须贴出并填写：
- `templates/evidence-template.md` 中的 **Skill 执行证据清单** 区块。
- 且必须追加固定校验行：
  - `命中检查:通过`
  - `Git规则:通过`

## 4. 通过标准
- 闸门脚本存在时均为通过；若脚本缺失，回退检查全部通过 且证据标注 `回退`
- 提交标题与正文格式合规
- 用户请求“提交git”时工作区清空（`git status --short` 为空），且允许多次提交完成清空
- 证据模板已完整填写
- 固定校验行完整输出（`命中检查:通过`、`Git规则:通过`）

## 5. 执行文件
- `scripts/pre_commit_gate.sh`
- `scripts/post_commit_gate.sh`
- `templates/evidence-template.md`
