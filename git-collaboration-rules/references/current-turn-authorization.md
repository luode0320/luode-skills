# 当前轮授权契约

本文件是 Git 动作授权与用户改动放行的唯一详细定义；首条命中格式仍由 `skill-hit-check-rules` 所有。

### 当前轮隔离

- Git 协作授权只来自当前这轮用户消息，不得继承、沿用或默认延续更早轮次的提交、推送、拉取、合并、变基或同步意图。
- 当前轮只要求查看状态、差异或日志时，仅授权只读盘点，不授权提交或其它写历史动作。
- 当前轮明确要求 `commit/push/pull/rebase/merge` 时，只授权该动作及其必要门禁；不能把一个动作扩大解释为另一个动作。
- “上一轮说过提交”“已经进入 Git 模式”“刚提交过一次”“Obsidian 需要沉淀”都不是本轮授权来源。

### 非代理改动保护

- 工作区存在未由当前代理产生的改动时，默认保护并停下确认，不得擅自覆盖、删除、暂存或提交。
- 用户当前轮明确说“这些也要提交”“一起提交”“都提交”时，才视为放行；最终证据必须列出被放行文件。
- 用户要求“提交git”且明确目标为全部当前改动时，按提交域拆分后循环提交，但仍不得覆盖冲突内容或绕过门禁。

### 安全边界

- 未授权不得执行 `git commit`、`git push`、`git pull`、`git rebase`、`git merge`、`git cherry-pick` 等动作。
- 不得用 amend、reset、checkout、clean、stash 或其它命令规避用户改动保护。
- 推送、强推、变基和破坏性历史操作继续受更高优先级安全与审批规则约束。
