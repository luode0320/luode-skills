# 命名模板

本文件定义默认命名模板。模板调整时，应先修改 `path-map.yaml`，这里负责补充说明。

## 需求域

- 需求文档：`ment/YYYY-MM-DD_需求中文说明.md`
- 验收文档：`ment/YYYY-MM-DD_验收中文说明.md`
- 实施计划：`ment/YYYY-MM-DD_实施计划中文说明.md`

补充要求：

- 同一天存在多个近似标题时，在不改变中文语义前提下补充更具体描述，避免重名。
- 同一需求的补齐、边界、拆分、变更应持续更新同一份需求文档，不新建平行入口。

## Bug 域

- Bug 根目录：`bug/YYYY-MM-DD_HHmmss_问题中文简介/`
- Bug 主入口文件：`bug/YYYY-MM-DD_HHmmss_问题中文简介/README.md`

补充要求：

- 同一个 Bug 的后续复现、定位、诊断、方案和验证都沿用同一根目录。
- `README.md` 作为主入口文件，其他附件和细节记录围绕该根目录组织。

## 测试域

- 测试任务根目录：`test/YYYY-MM-DD_HHmmss/`
- 测试中文主说明：`test/YYYY-MM-DD_HHmmss/测试任务中文简介/README.md`
- 真实测试资产：放在同一时间戳根目录下的 ASCII 真实代码路径镜像目录中

补充要求：

- 会被 Go 工具链扫描或编译的路径必须保持 ASCII。
- 中文说明默认只放在中文主说明目录，不进入编译路径。

## doc 域

- 通用项目文档默认放在 `doc/` 下。
- 具体文件名按文档主题命名，但应避免使用 `temp.md`、`result.txt` 这类无意义名称。
