# 根目录职责

本文件定义当前项目研发产物的默认根目录职责边界。目录名如果未来调整，以 `path-map.yaml` 为最终准。

## 当前默认根目录

- `ment/`
  用于需求文档、需求补齐、边界结论、拆分结果、实施计划和验收文档等需求域持久化产物。
- `bug/`
  用于 Bug 录入、复现、根因、运行时诊断、修复建议、风险评估和修复验证等 Bug 域持久化产物。
- `test/`
  用于测试任务目录、测试说明、测试脚本、模拟数据、验证证据、测试截图、测试日志和回归结果。
- `doc/`
  用于不直接归属单一需求、单一 Bug 或单一测试任务的通用项目文档、专题说明和长期说明材料。

## 目录职责边界

- 需求域默认写入 `ment/`，不要把需求文档散落到 `doc/`。
- Bug 域默认写入 `bug/`，不要把 Bug 主记录混写到 `test/` 或 `doc/`。
- 测试域默认写入 `test/`，不要把测试证据散落到业务代码目录、仓库根目录或 `doc/`。
- 通用项目说明才进入 `doc/`；如果文档明显服务于单一需求、单一 Bug 或单一测试任务，应优先归回对应根目录。

## 最近上下文预热默认扫描范围

新会话近期上下文预热默认扫描：

1. `ment/`
2. `test/`
3. `bug/`
4. `doc/`

如果其中某个根目录不存在，允许直接跳过，不报假错误。
