# Prop Pack 合同

Prop pack 用来把一批紧凑的小型静态道具放进一个网格 sheet，再切成独立透明 PNG。
它不是本 skill 的默认场景物件生产路线，而是一个受限例外。

## 强制前提

只有在同时满足以下条件时，才允许使用 prop pack：

- 用户明确要求做“小型道具包”“边缘点缀包”“demo 占位包”或类似小物件集合
- 物件本身不是地图视觉主角
- 物件不承担“用户会盯着看并判断质感”的任务
- 物件不需要独立成立
- 物件不是树、建筑、掩体、门、桥、雕像、祭坛、残骸、关键交互物件或大型植被

只要有一条不满足，就不要用 prop pack，改走独立 hero prop 路线。

## 适合

- 石头
- 草丛
- 花
- 蘑菇
- 箱子
- 木桶
- 小灯
- 路牌
- 小雕像

这些对象也只是在“明确是配角小物件”时才适合进 prop pack。

## 不适合

- 建筑
- 门
- 桥
- 平台
- 墙段
- 梯子
- 大树
- 任何正式树木资产
- 任何正式建筑资产
- 掩体
- 祭坛
- 雕像主物件
- 建筑边缘件
- 路障主物件
- 车辆残骸
- 大型植被
- 任何需要商业级质感验收的场景物件
- 宽长地形块
- 精确碰撞对齐对象
- 关键叙事物件

## 默认判定

除非用户特别要求，否则本项目中的地图场景物件默认按独立 hero prop 生产，而不是 prop pack。

以下信号一旦出现，就必须切换到 hero prop：

- 用户说“树木”“建筑”“掩体”“场景物件”“地图物件”“建筑残骸”“路障”
- 用户强调“高质量”“商业感”“像正式游戏”“像土豆兄弟那种完成度”
- 用户会把这个物件当作主要观感对象
- 结果如果做成符号化小图标会明显显得廉价

## 网格建议

- `2x2`：4 个物件，最稳
- `3x3`：9 个物件，默认首选
- `4x4`：16 个物件，只适合特别简单的小物件

## 宽长结构

地板、平台、桥、长墙、长地刺这类对象不要用方形 pack，改用：

- `1x3`
- `1x4`
- 或单独生成

## 提示词关键点

- top-down 场景道具
- 同风格、同镜头、同尺度
- 每格一个独立物件
- 每个物件完整留在格内
- 四周保留充足纯色留白
- 不要文字、标签、UI、水印、格线

## 禁止误用

不要用 prop pack 去规避高质量资产生产。

如果一个物件需要：

- 明确轮廓
- 体积
- 结构
- 接地阴影
- 材质分层
- 单体成立

那它就不是 prop pack 任务，而是 hero prop 任务。
