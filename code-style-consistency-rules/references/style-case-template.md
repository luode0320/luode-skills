# 用户风格反例条目模板

只在向全局用户风格反例库 `user-style-feedback-library.md` 写入或更新一条条目时使用本模板。
条目借鉴 `execution-failure-learning-rules/references/case-template.md` 的生命周期与去重思路，但只承载“代码风格正反例”，不承载执行失败案例。

## 单条 style case 字段

每条 case 用一个 `###` 小节承载，字段以 bullet 逐行给出，字段名固定为中文或稳定英文键：

```md
### STYLE-CASE-<LANG>-<三位序号>：<一句话标题>
- id: STYLE-CASE-GO-001
- status: candidate            # candidate 仅回显不落盘 | active 已确认落盘 | superseded 已被取代
- 语言/技术栈: Go
- 适用范围: 错误处理
- 去重键: go|错误处理|吞异常忽略error
- 来源: 用户文字反馈 / 用户截图反馈（2026-07-13）
- 反例（禁止这样写）:
  ```go
  data, _ := doSomething()   // 忽略 error
  ```
- 正例（应该这样写）:
  ```go
  data, err := doSomething()
  if err != nil {
      return fmt.Errorf("doSomething 失败: %w", err)
  }
  ```
- 规则一句话: Go 中禁止用 `_` 丢弃 error，必须显式判断并包装返回。
- 首次记录: 2026-07-13
- 确认时间: 2026-07-13
- 出现次数: 1
```

## 字段说明

- id：`STYLE-CASE-<语言大写缩写>-<三位序号>`，序号在同语言内递增，不复用已删除序号。
- status：只有三个取值。写入磁盘的条目必须是 `active`；`candidate` 只在当轮对话回显给用户，不落盘；被更好条目取代时改 `superseded` 并在正文标注取代者 id。
- 语言/技术栈：单一语言或栈，跨语言通用约定用 `通用`。
- 适用范围：一个语义场景（错误处理、命名、结构、排版、并发、日志等），不堆多个场景。
- 去重键：`语言|场景|反例签名` 三元组，签名先规范化大小写与空白后再写；用于合并重复反馈。
- 来源：标明是文字反馈还是截图反馈，并附确认日期。
- 反例 / 正例：优先给可编译或可对照的最小代码块；无法用代码表达时用一句话描述并说明差异。
- 规则一句话：从本条 case 抽象出的可执行约束，写码前加载时直接作为禁用写法。
- 出现次数：同去重键重复反馈时只加计数，不新增正文条目。

## 去重与生命周期

- 命中同一去重键：更新原条目的确认时间与出现次数，不新增条目。
- 正例发生变化：在原条目更新正例并记最新确认时间。
- 条目被取代：旧条目改 `superseded`，保留历史，不物理删除。
- 无法提取正例：向用户追问期望写法，不凭空生成正例，不写入库。
