# Header 透传与转发规则

## 用途

用于约束 Header 在网关、代理、服务之间的透传、覆盖和信任方式。

## 透传原则

- 只透传有明确契约的 Header，不做无差别全量透传。
- 透传前要明确该 Header 是保留原值、网关覆盖还是下游重建。
- 链路追踪头和幂等键通常需要跨服务传递，但要有统一命名和覆盖规则。
- 代理头只在明确代理链可信时消费，不随意直接信任客户端自传值。

## 转发与覆盖

- `X-Forwarded-*` 相关头必须结合反向代理或网关信任边界使用。
- 多层代理存在时，要明确真实来源的优先级解析规则。
- 会被网关重写的 Header，不能假设客户端传入值仍然生效。
- 若 Header 同时影响日志、鉴权或路由决策，必须先固定来源可信度。
