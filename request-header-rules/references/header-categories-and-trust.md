# Header 分类与可信边界

## 用途

用于区分不同 Header 的职责，并明确哪些来源可信。

## 常见分类

- 认证头：承载访问令牌、会话凭证或签名信息。
- 链路追踪头：承载 trace-id、span-id、request-id 等链路上下文。
- 租户上下文头：承载多租户场景下的租户标识。
- 幂等头：承载幂等键、去重键或请求重放控制信息。
- 代理转发头：如 `X-Forwarded-*`、真实 IP、协议和主机信息。
- 客户端上下文头：如客户端版本、设备类型、区域等辅助上下文。

## 可信原则

- 先明确 Header 由谁生成，再决定是否可信。
- 认证头、租户头、代理头通常不能简单信任任意客户端自传值。
- 网关注入和服务端生成的 Header，要明确覆盖策略和下游消费方式。
- 能从请求参数或业务数据推导出的业务语义，不应强塞进 Header。
