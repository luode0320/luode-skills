# 事务、锁与批量操作规则

## 用途

用于约束事务范围、锁粒度和批量操作风险。

## 事务原则

- 事务只包住需要原子性的数据库操作。
- 不把外部调用、慢 I/O、长时间计算随意包进数据库事务。
- 事务范围越大，越要解释一致性收益与并发代价。

## 锁原则

- 先明确为什么需要锁，再决定悲观锁、乐观锁或不加锁。
- 锁范围要尽量小，避免把读多写少场景机械升级成强锁方案。
- 使用锁时必须考虑阻塞、超时、死锁和重试路径。

## 批量操作原则

- 大批量写入优先考虑分批、幂等和失败恢复。
- 批量更新或删除要说明筛选条件和回滚风险。
- 不把一次性数据修复任务伪装成普通访问层逻辑。
