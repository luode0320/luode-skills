# Skill 完善建议输出模板

## 最小输出字段

1. 当前任务
2. 当前命中的原 skill
3. 当前卡住的步骤
4. 判定结果
5. 目标 skill
6. 缺口类型
7. 是否阻断当前任务
8. 最小修改建议
9. 建议修改文件
10. 修改后验收方式
11. 回到原任务的重启点

## 推荐输出格式

```text
当前任务：
- ...

当前命中的原 skill：
- ...

当前卡住的步骤：
- ...

判定结果：
- 这是 / 不是 skill gap

目标 skill：
- 应优先补的 skill 名称

缺口类型：
- description / 流程 / 边界 / references / 归档 / 验收

阻断级别：
- 阻断 / 非阻断

最小修改建议：
- 先补什么
- 再补什么
- 哪些不要动

建议修改文件：
- skill-x/SKILL.md
- skill-x/references/...
- 编码skill.md
- README.md

修改后验收方式：
- 重新触发该 skill
- 观察是否还会误命中 / 漏命中
- 观察当前任务是否可继续稳定推进

回到原任务的重启点：
- 从哪个阶段或哪一步重新开始
```

## 写法要求

- 要具体到文件，不要只写“补一下这个 skill”
- 要具体到缺口类型，不要只写“还不够完善”
- 要明确阻断级别，不要让主流程是否暂停处于模糊状态
