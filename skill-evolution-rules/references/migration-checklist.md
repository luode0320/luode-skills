# Skill 迁移检查清单

## 用途

用于在 skill 内容迁移、references 调整或 skill 退休时，检查是否遗漏同步项。

## 迁移前检查

- 是否已经明确迁移源和迁移目标。
- 是否已经说明迁移原因，而不是单纯为了“看起来更整齐”。
- 是否会影响已有 skill 的自动触发 description。

## 迁移中检查

- 是否同步更新 `SKILL.md` 中的 references 入口。
- 是否同步更新 `agents/openai.yaml` 的展示文案。
- 是否保留了必要的边界说明，避免迁移后职责丢失。

## 迁移后检查

- 是否仍然能说清每个 skill 的职责。
- 是否出现新的重复内容或新的误触发风险。
- 是否需要在规划文档、归档记录或审查清单中同步更新。

## 退休检查

- 是否已经明确替代者或确认无需替代。
- 是否已经清理旧 skill 对应的触发入口。
- 是否已经记录退休原因和生效时间，方便后续追溯。
