# 最终验收模板

## 适用定位

- 本模板只用于后置“最终验收”，发生在测试和审核之后。
- 文档路径统一为 `doc/验收/<需求文档同名主干>最终验收.md`。
- 最终验收文档与需求主文档、验收标准文档、实施文档、测试文档、审核文档分开维护。

## 推荐结构

1. 输入材料清单：需求主文档、验收标准、实施总览/实施周期、测试结果、审核结果。
2. 前置条件检查：测试是否完成、审核是否完成、是否存在已失效文档。
3. 逐条验收判定：按验收标准逐条记录通过 / 不通过 / 阻断。
4. 遗留项：尚未通过但允许记录待办的内容。
5. 阻断项：导致当前不能最终放行的关键问题。
6. 最终结论：通过 / 不通过 / 待重验。
7. 重验触发条件：若后续变更、补测、补审，何时必须重新打开最终验收。

## 归档要求

- 最终验收文档路径统一为 `doc/验收/<需求文档同名主干>最终验收.md`。
- 测试或审核任一未完成时，不得写“通过”；只能写“未满足最终验收前置条件”。
- 同一个需求只保留一份最终验收文档，后续重验继续更新同一份文档。
