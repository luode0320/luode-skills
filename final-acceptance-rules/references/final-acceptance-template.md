# 最终验收模板

## 适用定位

- 本模板只用于后置“最终验收”，发生在测试和审核之后。
- 文档路径统一为 `doc/7-验收/YYYY-MM-DD_HHmmss_<来源对象标识>_最终验收.md`。
- 最终验收文档与来源对象文档（需求或 Bug）、验收标准文档、实施文档、测试文档、审核文档分开维护。

## 推荐结构

1. 输入材料清单：来源对象文档（需求或 Bug）、验收标准、实施总览/实施周期、测试结果、审核结果。
2. 前置条件检查：测试是否完成、审核是否完成、实施周期是否按顺序收口、最小任务是否都有实现 / 真实测试 / 审查 / 验收证据、是否存在已失效文档。
3. 逐条验收判定：按验收标准逐条记录通过 / 不通过 / 阻断。
4. 实施闭环证据：按周期列出周期顺序、周期收口状态和周期内最小任务闭环证据。
5. 遗留项：尚未通过但允许记录待办的内容。
6. 阻断项：导致当前不能最终放行的关键问题。
7. 最终结论：通过 / 不通过 / 待重验。
8. 重验触发条件：若后续变更、补测、补审，何时必须重新打开最终验收。

## 归档要求

- 最终验收文档路径统一为 `doc/7-验收/YYYY-MM-DD_HHmmss_<来源对象标识>_最终验收.md`。
- 测试或审核任一未完成时，不得写“通过”；只能写“未满足最终验收前置条件”。
- 实施周期未按顺序收口，或任一纳入验收范围的最小任务缺少实现、真实测试、审查、验收证据时，不得写“通过”；只能写“未满足最终验收前置条件”。
- 同一个来源对象只保留一份最终验收文档，后续重验继续更新同一份文档。
