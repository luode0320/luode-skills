# 变更影响重算

## 变更 Owner 输出

- 需求：是否推翻原目标、边界、拆分、权限、兼容、异常或数据结论。
- 验收：哪些 `AC-*`、适用性和放行结论失效，需要回开 `acceptance-criteria-rules`。
- 实施：哪些实施总览、周期、切片、任务或文件/符号落点可能失效，需要回开 `implementation-planning-rules`。
- 测试与审查：哪些测试、审查和最终放行证据失效，需要对应 Owner 重验。
- 交付与图片：哪些图形、目录树、链接、图片决策、`IMG-*`、共享引用、敏感/版权、清理或回滚结论失效。

本 Skill 只声明受影响对象、失效原因、优先级和应回开的 Owner；实施文档结构、精确文件/符号、测试命令和任务闭环由 `implementation-planning-rules` 重新冻结。

## 决策输出

- 继续当前需求，但回开受影响结论。
- 当前任务只接一部分，其余拆为独立变更。
- 回流 `requirement-boundary-rules` 或 `requirement-splitting-rules`。
- 原实现不符合原需求时转 Bug 域。
- 所有重算结论都更新到同一份需求主文档，不创建独立长期变更文档。

## `CHG-*` 变更重算契约（强制）

每次变更必须生成唯一 `CHG-*`，并记录：来源、原值/新值、决策人、状态、授权、影响的 `SRC/DEC/REQ/RULE/AC/SLICE/CYCLE/TASK`、受影响文件/符号线索、图表、测试、审查、交付、清理、回滚、证据和复核时间。

| 影响面 | `CHG-*` 必须声明 |
| --- | --- |
| 需求/规则 | 范围、非范围、优先级、权限、兼容、异常、数据和 `unresolved_decisions` 的失效或新结论 |
| 前置验收 | 受影响 `AC-*`、场景、适用性和需回开的验收 Owner |
| 实施规划 | 受影响的实施总览、周期、切片、任务、文件/符号线索及重规划原因；具体结构由实施规划 Owner 重写 |
| 测试/审查 | 失效的主、异常、边界场景、样本、断言和审查结论，以及需重验的 Owner |
| 交付/图片 | 图形、目录树、链接、版本、图片决策、`IMG-*`、引用、版权/敏感、孤儿、清理和回滚影响 |

## 回开与授权闸门

1. 变更影响已确认目标、范围、接口、数据、权限、兼容、验收、计划、测试或审查时，必须保留旧值/新值差异并回开对应 Owner；不得只在聊天中说明。
2. `P0/P1` 变更未获授权前状态为 `blocked`，禁止继续编码或验收；`P2` 也必须登记默认值授权人、有效期和复核证据。
3. 用户明确要求同时补充需求与实施文档时，变更 Owner 先完成需求侧 `CHG-*`，随后必须移交实施规划 Owner 更新全部受影响实施文档，并记录未受影响周期的依据。
4. 原需求已完成后的新增改动不得覆盖旧周期；必须标记“需要新实施周期”，由实施规划 Owner 创建独立闭环。
5. 变更关闭后，对应 Owner 必须重新运行文档机器校验和受影响行为测试；任一证据缺失，原完成状态不得恢复。
6. 图片变更必须重新检查 `doc/data/images/` 文件签名、`../data/images/<file>` 相对引用、`IMG-*` 清单和消费者；绝对、远程、Base64、HTML、反斜杠、越界和历史目录路径均阻断。
