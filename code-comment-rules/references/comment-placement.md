# 注释放置原则

- 放在读者最需要停下来理解的地方。
- 先以“本轮改动位点”为优先注释范围：新增或修改了哪里，就先补哪里。
- 只要一个代码块存在大于等于 2 个步骤，就必须在对应步骤前补中文编号注释，使用 `1.`、`2.`、`3.` 这类顺序标识。
- 步骤编号必须就近放在对应步骤入口上方，禁止只在函数开头集中写 `1/2/3` 总览。
- 如果方法体已写出多个流程型普通注释，也必须改写为 `1.`、`2.`、`3.` 编号步骤注释，不能用普通注释替代。
- 类、结构体、DTO、配置对象等字段定义处必须写字段注释。
- 对象初始化、结构体字面量、字段映射或字段使用位置，也要在对应字段旁重复补注释，避免读者来回跳字段定义处理解含义。
- 每个函数、方法的修改都必须补充最近修改时间，格式为北京时间：`yyyy-MM-DD HH:mm:ss`。
- 最近修改时间行必须追加本次改动原因简述，例如：`// 最近修改时间: 2026-04-02 17:30:14 将查询1小时改成10小时`。
- 每个函数、方法都必须在注释中补充 `[参数]` / `[返回]` 简单说明；无参数或无返回时写 `无`。
- 函数、方法注释顺序固定为：`[参数]`、`[返回]`、`最近修改时间`，且最近修改时间必须放在最后一行。
- 用户要求“补充注释”时，也必须检查方法体是否需要 `1.`、`2.`、`3.` 步骤注释，不能只补方法头注释。
- 优先注释复杂分支入口、兼容逻辑、非直观约束和风险点。
- 块注释说明整体意图；字段列表、对象初始化和步骤拆解场景可以使用贴近代码的逐项注释。
- 未改动区域可不强制补注释，但不能作为跳过改动位点注释补齐的理由。
