# 压缩执行手册

## 目标

在尽量短的输出里保留继续执行必需信息，剔除重复背景和无关噪音。

## 信息分层

### 必须保留

- 当前任务目标、成功标准、阶段和唯一主域 Owner。
- 已确认的范围、接口、环境、安全、用户习惯、停止和回滚约束。
- 当前代码与工作区事实、关键文件路径、验证结果和阻断项。
- 当前平台规则文件及项目当前上下文中仍影响下一步的规则。
- 下一动作、最大推进边界和编码许可 `confirmed` / `not-confirmed` / `unknown`。
- 近期事实状态 `sufficient` / `missing` / `uncertain`，以及是否调用近期预热和原因。

### 可折叠

- 重复背景解释。
- 已完成且不再影响后续决策的过程细节。
- 已有持久化证据覆盖的低相关讨论分支。

### 可剔除

- 与当前任务无关的闲聊、泛问和重复确认语句。
- 已被当前代码、当前用户消息或新结论明确覆盖的旧口径。

## 输出模板

1. 当前目标与阶段
2. 已确认事实、规则和范围
3. 关键路径与验证证据
4. 待确认项、风险和阻断
5. 下一动作与最大推进边界
6. 编码许可状态
7. 规则与项目上下文重载状态
8. 近期事实三态、近期预热调用状态和主域交接

## 质量自检

- 恢复包是否足以在不重新猜测的情况下继续当前任务。
- 用户习惯、安全、授权、停止、回滚、local 环境和输出规则是否完整保留。
- 是否把不确定内容标成待确认，而非写成事实。
- 是否只有 `recent_context_state=missing` 才调用近期预热。
- 编码许可不是 `confirmed` 时，是否明确禁止编码。
- 当前平台规则文件与项目当前上下文是否真实重载；未完成时是否阻断。
