# 压缩执行手册

## 目标

在尽量短的输出里，保留“继续执行必需信息”，剔除重复背景和无关噪音。

## 信息分层

### 必须保留

- 当前任务目标与成功标准。
- 已确认约束（时间、范围、接口、环境、风险）。
- 当前主线上的关键结论与关键文件路径。
- 下一步动作与阻塞项。
- 编码许可状态：`confirmed` / `not-confirmed` / `unknown`（未确认默认 `unknown`）。

### 可折叠

- 重复出现的背景解释。
- 已完成且对后续无直接影响的细节过程。
- 低相关的历史讨论分支。

### 可剔除

- 与当前任务无关的闲聊、泛问和重复确认语句。
- 已被新结论明确覆盖的旧结论。

## 输出模板

建议压缩包保持以下结构：

1. 当前目标
2. 已确认事实与约束
3. 关键路径（文件 / 模块 / 提交线索）
4. 待确认项与风险
5. 下一步动作
6. 编码许可状态

## 质量自检

- 压缩后是否还能直接继续执行当前任务。
- 是否保留了关键约束和风险，不会误导下一步决策。
- 是否标注了仍缺的信息，而不是把不确定内容写成已确认结论。
- 若下一步可能进入编码，编码许可状态是否明确；若不明确，是否标记为 `unknown` 并要求先确认。
