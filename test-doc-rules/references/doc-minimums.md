# 测试文档最小要求

## 用途

用于明确测试 README、验证说明、测试报告、覆盖说明和执行记录至少要写什么。

## 测试 README 最小内容

- 测试对象或任务背景
- 执行目的
- 相关测试资源入口
- 执行前置条件
- 运行方式或定位方式
- 依赖数据
- 覆盖范围
- 验证结论

## 验证说明最小内容

- 验证目标
- 验证步骤
- 输入条件与依赖
- 预期结果

## 测试报告 / 执行记录最小内容

- 执行时间
- 执行人
- 执行环境
- 实际结果
- 失败项或风险项

## 覆盖说明最小内容

- 本次覆盖的功能范围
- 未覆盖部分
- 未覆盖原因
- 后续补测或回归建议

## 归档要求

- 上述文档默认写在 `test/` 根目录下当前任务目录的 `README.md` 中；如章节较多，可拆成同目录下的其他 `.md` 文件。
- 不得将测试文档归档到 `testing/`、`analysis/`、仓库根目录或业务代码目录。
