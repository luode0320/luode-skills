# 测试文档最小要求

## 用途

用于明确主 README 和补充文档至少要写什么，以及哪些内容必须写进中文 README。

## 中文主 README 最小内容

主 README 固定放在 `test/yyyy-MM-DD_HHmmss/<测试任务中文主题>/README.md`，至少包含：

- 测试目的或任务背景
- 对应的真实代码路径或测试对象
- 真实测试资产入口
- 执行前置条件
- 运行方式或定位方式
- 依赖数据与环境
- 覆盖范围
- 验证结论
- 未覆盖项或后续动作

说明：`<测试任务中文主题>` 为动态目录名，按该 README 将要描述的测试主题命名；禁止固定写成“测试任务中文简介”。

## 补充文档最小内容

补充文档放在同一时间戳根目录下的 ASCII 真实代码路径镜像目录中，至少包含：

- 补充文档类型
- 对应的真实代码路径
- 补充目的
- 具体步骤、案例或证据
- 实际结果或观察结论
- 与主 README 的关联说明

## 执行记录 / 报告最小内容

- 执行时间
- 执行人
- 执行环境
- 输入条件
- 实际结果
- 失败项或风险项
- 产物路径

## 归档要求

- 中文说明目录默认只存放 `README.md`，不得继续堆放其他 `.md`、截图或附件。
- 如需更详细的说明、检查单、案例表、截图解读或附录，统一放到 `test/yyyy-MM-DD_HHmmss/<ASCII 真实代码路径镜像>/` 下。
- 主 README 必须写清这些详细文档的路径，保证接手人先读 README，再顺着 README 找到详细证据。
- 不得将测试文档归档到 `testing/`、`analysis/`、仓库根目录或业务代码目录。
