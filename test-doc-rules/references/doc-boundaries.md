# 测试文档边界

## 用途

用于区分“文档组织问题”与“功能验证问题、回归问题、测试程序问题、目录落点问题”。

## 属于测试文档域

- 中文主 README 的章节是否完整
- 执行说明是否足以指导复现
- 测试报告和执行记录是否留痕完整
- 补充文档是否放到了正确的 ASCII 镜像路径
- 主 README 是否正确引用了详细证据

## 不属于测试文档域

- 测试目录怎么落点、时间戳根目录怎么创建
- 测试程序怎么写、断言怎么做、mock 怎么组织
- 功能是否满足需求和验收标准
- 回归范围够不够、是否覆盖上下游链路
- 临时调试日志和运行时定位证据是否应该保留

## 处理原则

- 文档只描述事实、范围、过程和结论，不替代实际测试执行。
- 中文说明目录只保留 `README.md`，不能继续膨胀成文档杂物区。
- 详细说明、案例、补充报告和截图说明放到同一时间戳根目录下的 ASCII 真实代码路径镜像目录。
- 功能验证结论由 `functional-validation-rules` 判定，文档域只负责把结论记录清楚。
- 回归验证结论由 `test-regression-rules` 判定，文档域只负责留痕和归档。
- 如果文档写不清，是因为前置测试资产或执行事实本身没准备好，应回流上游测试资源管理或验证 skill。
