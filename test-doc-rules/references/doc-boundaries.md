# 测试文档边界

## 用途

用于区分“文档组织问题”与“功能验证问题、回归问题、测试程序问题”。

## 属于测试文档域

- 测试 README 的章节是否完整
- 验证说明是否足以指导执行
- 测试报告和执行记录是否留痕完整
- 覆盖说明是否说清本次测了什么、没测什么

## 不属于测试文档域

- 测试程序怎么写、断言怎么做、mock 怎么组织
- 功能是否满足需求和验收标准
- 回归范围够不够、是否覆盖上下游链路
- 临时调试日志和运行时定位证据

## 处理原则

- 文档只描述事实、范围、过程和结论，不替代实际测试执行。
- 功能验证结论由功能验证 skill 判定，文档域只负责把结论记录清楚。
- 回归验证结论由回归 skill 判定，文档域只负责留痕和归档。
- 测试文档必须跟随 `test-location-rules`，统一落在 `test/` 根目录下的当前任务目录中；默认由该目录的 `README.md` 承载主说明。
- 如果文档写不清，是因为前置测试资产本身没准备好，应回流上游测试资源管理 skill。
