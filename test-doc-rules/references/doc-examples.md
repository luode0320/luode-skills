# 测试文档正反例

## 正例

- 测试 README 能让接手人快速找到测试资源、运行入口和前置条件。
- 验证说明写清步骤、输入、预期结果，执行者无需再口头确认。
- 测试报告记录执行环境、实际结果、失败项和风险项，便于复盘。
- 覆盖说明明确指出本次覆盖范围与未覆盖部分，不制造“已经全测过”的错觉。

## 反例

- 只写“已测通过”，没有测试对象、步骤、环境和结论细节。
- 把测试报告写成一堆聊天记录或截图堆砌，缺少结构化结论。
- 测试 README 只放目录列表，没有入口、依赖、执行方式说明。
- 覆盖说明只写“全部覆盖”，但没有边界说明和未覆盖项说明。

## 判定提示

- 如果接手人看完文档仍不知道怎么执行或怎么复现，说明文档不达标。
- 如果文档只能证明“做过事情”，不能证明“做了什么、结果如何”，说明留痕不够。
- 如果文档里充满实现细节，却没有测试目标和结论，说明职责偏到了程序域。
