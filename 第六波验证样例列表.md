# 第六波验证样例列表

## 用途

用于验证第六波增强类 skill 是否能按预期命中，不与前五波的既有技能边界冲突。

## 样例 1：修改环境变量与默认值，应命中配置规则

- 场景：新增 `.env.example` 占位、调整服务端默认值、区分客户端公开配置与服务端私有配置。
- 预期命中：`config-change-rules`
- 预期流转路径：配置修改 -> `config-change-rules`
- 验证重点：不能把危险默认值、真实密钥或客户端公开配置边界问题混入普通代码修改。

## 样例 2：修改表结构与迁移脚本，应命中 schema 规则

- 场景：新增字段、增加索引、调整唯一约束、编写迁移和回滚说明。
- 预期命中：`database-schema-rules`
- 预期流转路径：结构变更 -> `database-schema-rules`
- 验证重点：结构变更、数据修复和查询实现必须拆开，不允许一条迁移包办全部职责。

## 样例 3：修改查询逻辑与事务控制，应命中 query 规则

- 场景：重写 SQL、调整 Repository、补分页、缩事务范围、优化批量写入。
- 预期命中：`database-query-rules`
- 预期流转路径：数据访问修改 -> `database-query-rules`
- 验证重点：query 规则只处理访问实现，不回头接管 schema 设计。

## 样例 4：新增 trace 头或幂等键，应命中请求头规则

- 场景：新增 `trace-id`、`span-id`、幂等键、租户头，或调整 `X-Forwarded-*` 信任逻辑。
- 预期命中：`request-header-rules`
- 预期流转路径：Header 协议调整 -> `request-header-rules`
- 验证重点：Header 规则应与请求参数、日志 trace 记录和鉴权策略保持分离。

## 样例 5：增加认证 / 鉴权 / 敏感信息处理，应命中安全规则

- 场景：新增 JWT 校验、中间件鉴权、对象级权限判断、输入校验、敏感字段保护、上传下载限制。
- 预期命中：`auth-security-rules`
- 预期流转路径：安全相关实现 -> `auth-security-rules` -> 按框架选择中文 seed 细则
- 验证重点：安全基线要集中，不能只靠前端隐藏、只靠 Header 或零散 `if`。

## 样例 6：做缓存或性能优化时，应验证没有被误包装成业务逻辑修改

- 场景：为慢查询加缓存、为接口加耗时预算、为大列表加虚拟滚动、为热点链路改批量策略。
- 预期命中：`performance-caching-rules`
- 预期流转路径：性能 / 缓存优化 -> `performance-caching-rules`
- 验证重点：性能问题要先归因，不能用“优化”掩盖业务逻辑改动或结构问题。

## 样例 7：联调时出现环境差异、链路不通，应命中联调增强规则

- 场景：上下游接口样例不一致、测试环境配置不同、trace 断链、网关透传异常、某一跳调用失败。
- 预期命中：`integration-debugging-rules`
- 预期流转路径：功能单点通过 -> 联调串联失败 -> `integration-debugging-rules`
- 验证重点：联调问题不能直接粗暴判成单点功能失败，也不能和单模块运行时调试混淆。

## 总体验证结论

- 第六波应形成明确增强链：配置边界 -> 数据结构 -> 数据访问 -> Header 协议 -> 安全基线 -> 性能缓存 -> 联调排障。
- 只要上述 7 类场景都能稳定命中正确 skill，第六波核心职责边界就算跑通。
