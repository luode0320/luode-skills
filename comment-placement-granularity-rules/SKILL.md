# 注释放置与颗粒度规则

只在需要判断“这条注释该不该写、该放哪里、该写多细”时使用本 skill。
如果当前争议是改动位点注释是否补齐、函数头 `[参数]` / `[返回]` / `最近修改时间` 是否完整、方法体 `1.` `2.` `3.` 步骤编号是否达标，请转交 `comment-completion-gate-rules`；如果主要争议是中文表达，请转交 `chinese-comment-rules`；如果是 Swagger/OpenAPI 接口文档注解，请转交 `api-swagger-rules`。

## Skill 作用与适用场景

- 判断注释是否必要，避免无效注释泛滥。
- 规范块注释、边界注释、风险注释和字段注释的放置方式。
- 约束注释颗粒度，防止把每行代码都重新写一遍。
- 要求字段定义、初始化、映射和使用位置在必要时重复补字段含义注释。
- 对配置对象、启动参数对象、数据库连接对象、服务实例依赖对象、运行缓存/状态对象、交易/风控参数对象、HTTP 客户端或第三方 SDK 参数对象的结构体字面量初始化，若字段值来自另一层配置、环境变量、请求体、数据库模型、第三方响应，或承载服务依赖注入、运行状态缓存和并发控制上下文，必须逐字段或逐字段组补中文注释，说明字段用途与来源，不能只在结构体上方写一句总览。
- 治理过期注释、错位注释和误导性注释。
- 对于较长的函数调用，如果该调用承担关键补丁逻辑、兜底分支、重试收口或兼容分支，必须在调用点上方补就近中文意图注释，说明“为什么这里要这样调用”，而不是只让调用参数裸露在代码里。
- 这类注释应优先落在调用语句上一到两行，避免把说明写到函数头后面，导致读者在长调用前看不到触发背景。

## 自动触发信号

- 准备新增或修改代码注释，但不确定该不该写。
- 已确定要写注释，但不确定应放在哪一段代码上方。
- 不确定某条注释该写到什么颗粒度，担心太粗或太碎。
- 用户只说“补充注释/加注释”且未给出明确落点时，默认先命中本 skill 做落点与颗粒度判定。
- 新增或修改结构体、DTO、配置对象、字段映射或对象初始化逻辑。
- 新增或修改结构体字面量字段赋值，尤其是 `Config{}`、`LogConfig{}`、服务实例对象、数据库模型、订单模型、请求/响应 DTO、缓存状态对象、第三方 SDK 参数对象等。
- 代码评审指出注释位置不对、字段注释缺失或注释过量。
- 需要清理与当前实现不一致的旧注释。

## 进入后先做什么

1. 先定位本轮需要判断的代码位点。
2. 对每个候选位点判断“是否真的需要注释”，不能默认一律补。
3. 若需要注释，选择最能帮助读者理解的位置，而不是机械写在函数开头。
4. 对字段定义、对象初始化、字段映射和字段使用位置，分别判断是否需要重复补字段含义。
5. 对跨层字段映射的结构体字面量，逐字段检查是否已有就近中文注释；若字段较多且语义连续，可按字段组补注释，但不得让关键字段裸露。
6. 顺手检查是否存在过期、错位或误导性旧注释，避免新旧注释并存。

## 默认执行流程

1. 默认先读 `references/comment-placement.md`，判断注释应放在哪里最有效。
2. 需要控制“写多细、写多少”时，再读 `references/comment-granularity.md`。
3. 需要对照正反例或排查旧注释偏移时，再读 `references/comment-examples.md`。
4. 输出 注释必要性判断、推荐落点、推荐颗粒度、字段注释方案和待清理旧注释清单。
5. 如果已经识别出“这是改动位点补齐检查、函数头元信息补齐或步骤编号闸门问题”，立即转交 `comment-completion-gate-rules`。
6. 注释结构确定后，若还需统一中文表达，再交给 `chinese-comment-rules` 继续处理。

## 权责边界与不负责事项

- 只负责注释结构与颗粒度，不代替 `chinese-comment-rules` 处理语言表述。
- 不代替 `comment-completion-gate-rules` 处理改动位点补齐闸门、函数头元信息或步骤编号完整性。
- 不代替 `api-swagger-rules` 处理 Swagger/OpenAPI 接口文档注解。
- 不把糟糕命名、复杂结构和错误抽象都靠注释硬解释。
- 不保留已经与实现脱节的旧注释。

## 需要暂停并确认的条件

- 代码本身可以通过重命名或小重构解决理解问题。
- 注释打算解释一大段明显应拆函数的流程。
- 现有注释和实现冲突，但代码行为还未确认。
- 为了通过评审而临时补大量无效注释。

## 执行通过 / 驳回标准

- 通过：注释只出现在真正需要解释的位置；字段定义、初始化、映射和使用位置的注释方案清楚；配置/启动/数据库/服务依赖/运行缓存/交易/风控/第三方 SDK 结构体字面量的关键字段已有逐字段或逐字段组中文注释；长调用若承载关键补丁逻辑时已在调用点上方补就近意图注释；注释颗粒度不过粗也不过碎；过期、错位和误导性注释已清理或明确列入清理清单。
- 驳回：注释位置错乱、逐行复述代码、字段关键含义缺失、跨层映射的结构体字面量关键字段裸露、旧注释已过期却仍保留，或本应转交 `comment-completion-gate-rules` 的闸门问题被混在本 skill 中处理。
