# 注释放置与颗粒度规则

只在需要判断“这条注释该不该写、该放哪里、该写多细”时使用本 skill。
如果当前争议是改动位点注释是否补齐、函数头 `[参数]` / `[返回]` / `最近修改时间` 是否完整、方法体 `1.` `2.` `3.` 步骤编号是否达标，请转交 `comment-completion-gate-rules`；如果主要争议是中文表达，请转交 `chinese-comment-rules`；如果是 Swagger/OpenAPI 接口文档注解，请转交 `api-swagger-rules`。

## 注释模式（新增）

- `default`（默认最小注释模式）：仅在“读者无法直接从代码看懂意图/边界/风险”处补注释，不追求注释覆盖率。
- `strict`（严格注释模式）：仅在用户明确提出“补充注释/只补注释/注释完善/严格注释”时启用，允许提高注释覆盖。
- 若用户未显式要求严格注释，必须使用 `default` 模式，避免与“注释应简洁、少而有效”原则冲突。

## Skill 作用与适用场景

- 判断注释是否必要，避免无效注释泛滥。
- 规范块注释、边界注释、风险注释和字段注释的放置方式。
- 约束注释颗粒度，防止把每行代码都重新写一遍。
- 要求字段定义、初始化、映射和使用位置在必要时重复补字段含义注释。
- 治理过期注释、错位注释和误导性注释。

## 自动触发信号

- 准备新增或修改代码注释，但不确定该不该写。
- 已确定要写注释，但不确定应放在哪一段代码上方。
- 不确定某条注释该写到什么颗粒度，担心太粗或太碎。
- 新增或修改结构体、DTO、配置对象、字段映射或对象初始化逻辑。
- 代码评审指出注释位置不对、字段注释缺失或注释过量。
- 需要清理与当前实现不一致的旧注释。

## 进入后先做什么

1. 先定位本轮需要判断的代码位点。
2. 先判定当前模式：`default` 或 `strict`。
3. 对每个候选位点判断“是否真的需要注释”，不能默认一律补。
4. 若需要注释，选择最能帮助读者理解的位置，而不是机械写在函数开头。
5. 对字段定义、对象初始化、字段映射和字段使用位置，分别判断是否需要重复补字段含义。
6. 顺手检查是否存在过期、错位或误导性旧注释，避免新旧注释并存。

## 默认执行流程

1. 默认先读 `references/comment-placement.md`，判断注释应放在哪里最有效。
2. 需要控制“写多细、写多少”时，再读 `references/comment-granularity.md`。
3. 需要对照正反例或排查旧注释偏移时，再读 `references/comment-examples.md`。
4. 输出注释必要性判断、推荐落点、推荐颗粒度、字段注释方案和待清理旧注释清单。
5. 如果已经识别出“这是改动位点补齐检查、函数头元信息补齐或步骤编号闸门问题”，立即转交 `comment-completion-gate-rules`。
6. 注释结构确定后，若还需统一中文表达，再交给 `chinese-comment-rules` 继续处理。

## default 模式最低要求（新增）

- 复杂分支、并发控制、风控校验、外部系统兼容分支、金额/精度处理必须有意图注释。
- 业务常识即可读代码（简单赋值、直观判断、基础循环）不强制补注释。
- 禁止逐行复述代码。

## strict 模式增强要求（新增）

- 在 `default` 基础上，允许提高函数头与步骤注释覆盖率。
- strict 不得与 `comment-completion-gate-rules` 冲突；冲突时以 `comment-completion-gate-rules` 的 strict 规则为准。

## 权责边界与不负责事项

- 只负责注释结构与颗粒度，不代替 `chinese-comment-rules` 处理语言表述。
- 不代替 `comment-completion-gate-rules` 处理改动位点补齐闸门、函数头元信息或步骤编号完整性。
- 不代替 `api-swagger-rules` 处理 Swagger/OpenAPI 接口文档注解。
- 不把糟糕命名、复杂结构和错误抽象都靠注释硬解释。
- 不保留已经与实现脱节的旧注释。

## 需要暂停并确认的条件

- 代码本身可以通过重命名或小重构解决理解问题。
- 注释打算解释一大段明显应拆函数的流程。
- 现有注释和实现冲突，但代码行为还未确认。
- 为了通过评审而临时补大量无效注释。

## 执行通过 / 驳回标准

- 通过：注释只出现在真正需要解释的位置；字段定义、初始化、映射和使用位置的注释方案清楚；注释颗粒度不过粗也不过碎；过期、错位和误导性注释已清理或明确列入清理清单。
- 驳回：注释位置错乱、逐行复述代码、字段关键含义缺失、旧注释已过期却仍保留，或本应转交 `comment-completion-gate-rules` 的闸门问题被混在本 skill 中处理。

## references 读取规则

- 默认先读 `references/comment-placement.md`。
- 只有在控制注释颗粒度和必要性时，再读 `references/comment-granularity.md`。
- 只有在对照注释样例或排查旧注释偏移时，再读 `references/comment-examples.md`。
