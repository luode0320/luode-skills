# 注释放置原则

- 注释应放在读者最需要停下来理解的地方，而不是机械堆在函数开头。
- 优先关注复杂分支入口、兼容逻辑、风险点和非直观约束。
- 字段定义处必须写字段含义；当初始化、字段映射或字段使用位置会让读者频繁回跳定义时，也要在对应字段旁重复补字段注释。
- 块注释用于说明整体意图；贴近代码的短注释用于说明局部边界、风险和字段含义。
- 函数/方法体、闭包体和连续控制流代码块超过 5 行非空代码/注释行时，步骤注释必须进入对应代码块内部，并贴近步骤或控制流入口；只在函数头、块外或开头集中写总览不算满足。
- 每个超过 5 行的代码块独立判断；嵌套块超过阈值时，不能只借用外层编号，必要时使用 `1.1`、`1.2` 等层级编号。
- 当判断结果已经进入“改动位点必须补齐哪些注释、方法体步骤编号是否完整、函数头元信息是否缺失”时，应转交 `comment-completion-gate-rules`。
- 旧注释若已与实现脱节，应优先清理或更新，不能让新注释叠在旧误导信息上。
