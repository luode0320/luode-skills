# 注释颗粒度

- 能靠好命名解决的，不靠注释重复解释。
- 注释优先解释为什么、边界是什么、风险在哪里，不逐行复述代码动作。
- 字段清单、对象初始化、字段映射和关键字段使用位置的逐项注释属于明确保留的例外场景。
- 一条注释只解决一个理解问题，避免把多层背景、流程和术语塞进同一段大注释。
- 超过 5 行的代码块至少要有一条块内顶层步骤注释；多个步骤按 `1.`、`2.`、`3.` 拆开说明意图，不要求每一行都写注释。
- 步骤注释应解释该段逻辑的意图、边界或原因，不逐行翻译代码；代码块长度门槛约束的是“块内落点”，不是把注释堆成逐行复述。
- 若某段逻辑复杂到需要成片注释才能看懂，应先评估是否该回到命名或结构调整。
- 若读者已经能从上下文自然推断，不额外补“显而易见”的重复注释。
