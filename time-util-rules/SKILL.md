# 时间处理规则

## 作用

- 统一北京时间、UTC、时间窗、日/月边界、时间戳与字符串转换。
- 统一业务代码中对当前时间、开始结束时间、自然日/月统计区间的写法。

## 自动触发

- 出现 `time.Now()`、`time.Now().In(...)`、`time.Date(...)`、`time.Unix(...)`、`Format("2006-01-02")`、`Format("2006-01-02 15:04:05")` 等写法。
- 新增或修改日期筛选、时间范围查询、日报/月报、快照统计、定时任务。
- 新增或修改时间字符串解析、时区转换、秒级/毫秒级时间戳处理。

## 强制规则

1. 业务时间处理优先复用项目内 `timeUtil`。
2. 当前时间优先使用 `timeUtil.GetNow()`。
3. 日/月边界优先使用 `timeUtil` 中的边界工具或统一封装。
4. 时间字符串与时间戳转换优先使用 `timeUtil.ParseTime`、`TimeToString`、`SecondsToTime`、`MillisecondsToTime`、`FormatYYYYToTimestamp` 等方法。
5. 禁止在业务主逻辑里散写 `time.Now().In(...)`、`time.Date(...)`、裸日期格式串。
6. 如果 `timeUtil` 不够用，先补 `timeUtil`，再改业务调用。

## 执行流程

1. 先判断时间语义：当前时间、日开始/结束、月开始/结束、时间戳、字符串解析、时区转换。
2. 再确认是否已有可复用 `timeUtil` 方法。
3. 需要新增能力时，优先补充 `timeUtil`。
4. 最后再修改业务代码，保证同一时间口径一致。

## 边界

- 只负责时间处理方式和时区/边界一致性。
- 不负责数据库 schema。
- 不负责报表指标定义本身。

## 通过标准

- 通过：时间相关逻辑优先复用 `timeUtil`，没有重复造轮子。
- 通过：自然日、自然月、时区转换口径一致。
- 驳回：业务代码直接硬编码日期格式串作为核心逻辑。
- 驳回：同一个功能里混用多套时间处理方式。
