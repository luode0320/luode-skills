# 组件边界规则

## 先判断组件类型

- 展示组件：主要负责结构和展示，不承担复杂流程判断
- 容器组件：负责取数、状态桥接、调度子组件
- 复合组件：负责一组强相关子结构的统一组合
- 页面组件：负责页面级布局和流程编排，不应塞进细粒度展示细节

## 拆分原则

- 一个组件只保留一个主要职责
- 如果 JSX / template 里同时堆了取数、状态判断、事件编排、样式细节和多段条件渲染，应考虑拆分
- 如果拆完以后每个子组件都离不开父组件的上下文，说明拆分过细

## 常见反模式

- 页面和组件不分，所有逻辑都堆在页面文件
- 为了“拆分”把一段很短的结构拆成大量只用一次的小组件
- 容器组件直接承担大量视觉细节，导致职责混杂
