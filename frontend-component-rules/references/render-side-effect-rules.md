# 渲染与副作用规则

## 副作用边界

- 请求、副作用订阅、事件绑定、定时器要有明确进入和清理时机
- effect / watch / lifecycle 的依赖要完整，不要靠侥幸运行
- 派生展示优先直接计算，不要再额外存状态然后同步

## 渲染稳定性

- 不要在渲染阶段做有副作用的事情
- 避免因为对象、函数和数组引用频繁变化导致下游无意义重渲染
- 列表项组件要保证 key 稳定

## 条件渲染

- 优先让条件分支表达清楚，不要把多个条件塞成难读的三元链
- 加载态、空态、错误态和正常态要彼此独立，不要互相覆盖

## 常见反模式

- effect 里再 setState 同步另一个可直接算出的值
- watch / effect 同时承担数据同步和业务修复
- 渲染和副作用耦合，导致难以排查组件行为
