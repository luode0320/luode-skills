# 组合复用规则

## 什么时候抽复用

- 至少在两个以上真实场景稳定成立
- 脱离当前业务上下文后仍然语义完整
- 抽出来后调用方式比复制粘贴更清楚，而不是更绕

## hooks / composables

- 负责抽状态逻辑、派生逻辑和副作用编排
- 不要把 UI 结构细节和业务上下文塞进 hook / composable
- 返回值要稳定、可理解，避免返回一大坨杂乱对象

## 包装组件

- 包装组件要么统一行为，要么统一视觉壳层
- 如果包装组件只是多包了一层、没有明确收益，就不要强加

## 常见反模式

- 复制了两次就立即抽“万能组件”
- 一个 hook 同时管请求、状态、导航、埋点和 UI 展示
- 组件复用名义下制造更多隐式依赖
