# 配置来源与分层规则

## 用途

用于确定配置从哪里来、谁覆盖谁、哪些层可以存在。

## 常见配置来源

- 仓库内非敏感基础配置
- 环境差异配置
- 运行时环境变量
- 外部密钥或配置注入系统
- 特性开关与运行时可调参数

## 分层原则

- 先复用现有配置层级，不额外制造平行配置体系。
- 非敏感基础配置与敏感配置分离存放。
- 环境差异通过明确层级覆盖，不靠散落 if/else 拼接。
- 配置优先级必须可解释，避免同一配置项在多个位置互相覆盖但无人知晓。

## 检查清单

- 当前配置项属于哪一层
- 当前配置由谁注入、谁消费
- 当前优先级链是否清晰
- 当前改动是否会引入新的环境差异或覆盖歧义
