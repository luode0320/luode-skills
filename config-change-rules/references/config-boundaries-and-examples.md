# 配置边界与样例

## 用途

用于区分“配置问题”与“业务逻辑问题、安全策略问题、发布问题”。

## 属于配置域

- 配置来源选择
- 配置分层与优先级
- 默认值是否合理
- 密钥是否外置、占位是否安全
- 客户端公开配置与服务端私有配置的边界

## 不属于配置域

- 鉴权流程、权限模型、输入校验策略本身
- 请求头约定和 trace 透传协议
- 发布、灰度、回滚执行步骤
- 通过配置偷偷改变核心业务规则但未同步业务确认

## 正例

- 在 `.env.example` 中增加 `PAYMENT_TIMEOUT_MS=` 这种占位，不提交真实值。
- 在服务端通过环境变量注入数据库密码，而不是写进 `application.yml`。
- 对客户端构建变量只暴露公开域名、公开开关，不暴露私有凭证。

## 反例

- 把真实 `SECRET_KEY`、数据库连接串、第三方 Token 提交到仓库。
- 为了“方便上线”给关键安全开关一个默认关闭值。
- 把业务规则阈值、权限绕过开关伪装成普通配置，却没有边界说明和风险记录。
