# 默认值、密钥与敏感配置规则

## 用途

用于判断哪些配置可以给默认值，哪些配置必须外置，哪些内容绝不能进入仓库或客户端。

## 默认值策略

- 只有安全、可解释、不会放大事故面的配置才允许默认值。
- 关键开关、密钥、数据库凭证、签名参数、外部服务凭证默认不提供真实值。
- 配置缺失时优先快速失败，不要默默退回到危险默认值。
- `.env.example` 只放占位和说明，不放可直接使用的真实秘密。

## 密钥与敏感配置

- API Key、数据库密码、私钥、JWT 签名密钥、第三方令牌必须外置注入。
- 不要把敏感配置写入日志、报错、调试输出或客户端 bundle。
- 客户端环境变量默认可被看到，只能放可公开信息。
- 对需要轮换的密钥，要保留轮换和失效策略，不把生命周期写死在代码里。

## 检查清单

- 当前配置是否属于敏感信息
- 当前默认值是否会掩盖错误配置
- 当前示例文件是否只包含占位
- 当前客户端配置是否只暴露公开信息
