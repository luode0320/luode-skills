# Bug 描述模板

结论：{用一句普通中文说明当前已知的问题结果}；影响：{说明受影响的用户、角色或业务环节}；范围：{说明本记录会确认的现象和场景}；非范围：{说明本记录暂不判断的内容}；变化：{说明问题对用户或业务造成的可感知变化}；完成标准：{说明问题记录达到可复现和可定位的条件}；术语说明：无技术术语需要解释，或 {术语：它在本次问题中的作用}；验证状态：{说明当前证据和复现进展及依据}。

## 白话正文与附录

正文先说明发生了什么、影响谁、范围和非范围、期望结果、当前能否继续、术语含义和怎样算解决；H1 后单段必须完整包含八个固定字段。环境、账号、步骤、日志、截图来源和证据放入文末连续附录；完整规则见 [白话文档与附录分层契约](../../artifact-delivery-gate-rules/references/plain-language-document-contract.md)。

## 用途

用于把用户口头描述整理成可继续复现和定位的问题记录。

## 标准模板

> 当前 Bug 根目录应落在 `doc/4-bugs/YYYY-MM-DD_HHmmss_问题中文简介/` 下，主入口文件为该目录中的 `README.md`。

```markdown
# Bug：{问题中文简介}

结论：{用一句普通中文说明当前已知的问题结果}；影响：{说明受影响的用户、角色或业务环节}；范围：{说明本记录会确认的现象和场景}；非范围：{说明本记录暂不判断的内容}；变化：{说明问题对用户或业务造成的可感知变化}；完成标准：{说明问题记录达到可复现和可定位的条件}；术语说明：无技术术语需要解释，或 {术语：它在本次问题中的作用}；验证状态：{说明当前证据和复现进展及依据}。

## 问题现象
- 出现了什么异常、报错、错误结果或异常行为

## 影响范围
- 影响哪个页面、接口、业务流程、用户范围或数据范围

## 出现环境
- 线上 / 测试 / 本地
- 账号、租户、版本、配置差异

## 触发条件
- 什么输入、什么步骤、什么时机会触发

## 期望结果
- 正常应该发生什么

## 实际结果
- 实际发生了什么

## 当前证据
- 日志、报错、截图、trace、复现片段

## 当前缺失项
- 还缺什么信息

## 图示产物（必填，直接写入本文档正文）
- 流程图：Mermaid `flowchart` 代码块
- 时序图：Mermaid `sequenceDiagram` 代码块

## 执行附录
- local 环境、账号、数据、精确复现步骤、日志和截图原始信息：

## 追踪附录
- 来源、稳定标识、证据定位、图示关联和问题变更记录：
```

## 原则

- 先写现象，再写猜测。
- 猜测原因不能替代问题事实。
- Bug 根目录记录只有 `README.md` 一个主入口文件，流程图与时序图以 Mermaid 语法直接嵌入正文，不另建独立 SVG 等图文件。

## 执行附录

- 生成问题记录时，把 local 环境、账号、数据、精确复现步骤、日志和截图原始信息放在这里。

## 追踪附录

- 生成问题记录时，把来源、稳定标识、证据定位、图示关联和问题变更记录放在这里；本节必须紧随执行附录并位于文末。
