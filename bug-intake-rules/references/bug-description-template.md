# Bug 描述模板

## 用途

用于把用户口头描述整理成可继续复现和定位的问题记录。

## 标准模板

```text
问题现象
- 出现了什么异常、报错、错误结果或异常行为

影响范围
- 影响哪个页面、接口、业务流程、用户范围或数据范围

出现环境
- 线上 / 测试 / 本地
- 账号、租户、版本、配置差异

触发条件
- 什么输入、什么步骤、什么时机会触发

期望结果
- 正常应该发生什么

实际结果
- 实际发生了什么

当前证据
- 日志、报错、截图、trace、复现片段

当前缺失项
- 还缺什么信息

图示产物（必填）
- 流程图：`flow.svg`
- 时序图：`sequence.svg`
```

## 原则

- 先写现象，再写猜测。
- 猜测原因不能替代问题事实。
- Bug 根目录记录必须包含 `README.md` + `flow.svg` + `sequence.svg` 三类主入口产物。
