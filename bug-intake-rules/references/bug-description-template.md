# Bug 描述模板

## 用途

用于把用户口头描述整理成可继续复现和定位的问题记录。

## 标准模板

> 当前 Bug 根目录应落在 `doc/4-bugs/YYYY-MM-DD_HHmmss_问题中文简介/` 下，主入口文件为该目录中的 `README.md`。

```text
问题现象
- 出现了什么异常、报错、错误结果或异常行为

影响范围
- 影响哪个页面、接口、业务流程、用户范围或数据范围

出现环境
- 线上 / 测试 / 本地
- 账号、租户、版本、配置差异

触发条件
- 什么输入、什么步骤、什么时机会触发

期望结果
- 正常应该发生什么

实际结果
- 实际发生了什么

当前证据
- 日志、报错、截图、trace、复现片段

当前缺失项
- 还缺什么信息

图示产物（必填，直接写入本文档正文）
- 流程图：Mermaid `flowchart` 代码块
- 时序图：Mermaid `sequenceDiagram` 代码块
```

## 原则

- 先写现象，再写猜测。
- 猜测原因不能替代问题事实。
- Bug 根目录记录只有 `README.md` 一个主入口文件，流程图与时序图以 Mermaid 语法直接嵌入正文，不另建独立 SVG 等图文件。
