# 证据分级、截图取证与连库只读约束

## 证据分级

- 强证据：当前项目代码位点、真实命令输出、本地库只读查询结果、数据库 schema、配置文件、真实日志 / trace 行、用户明确确认。
- 中证据：项目设计文档、历史 Bug 记录、同仓库相似实现、审查记录、测试记录。
- 弱证据：命名相似、注释暗示、用户主观判断、截图文案未经代码 / 数据印证、第三方公开文档。

输出根因候选时必须标注证据等级。弱证据只能支撑“可疑点 / 待验证假设”，不能写成已证实根因。

## 截图取证

- 截图先提取客观信息：报错文案、异常堆栈行号、HTTP 状态码、页面状态、关键数据值、时间戳。
- 截图信息必须回到代码或数据二次印证，不能仅凭截图下根因结论。
- 截图里的数据值可作为连库只读查询的对比基准（例如页面显示金额与库中金额对比）。
- 截图涉及敏感信息（手机号、身份证、密钥、token）时，记录结论时脱敏。

## 连接本地数据库的只读安全约束（强制）

- 连接对象只能是 `local` 本地配置声明的库（判定依据是配置归属，可能指向远程共享开发库，不要求地址一定是本机；详见 `test-strategy-rules`）；**测试库与生产库一律禁连**，无例外。
- 只允许只读语句：`SELECT`、`SHOW`、`EXPLAIN`、`DESCRIBE` 等；**严禁任何写操作**（`INSERT` / `UPDATE` / `DELETE` / `TRUNCATE` / DDL）。
- 需要增删改数据才能验证或修复时，只产出 SQL 脚本（目的、影响范围、回滚方式）交用户手动执行，agent 不得代执行。
- 连接信息从本地配置文件读取（见 `test-strategy-rules` 的「本地环境配置发现与连接」），不得外泄明文密钥 / 连接串。
- 大表查询加条件或 `LIMIT`；高成本查询先评估再执行，避免拖垮本地库。
- 记录可追溯：连了哪个本地库、执行了哪些只读 SQL、对比了哪些数据与结论，写入 Bug 根目录（脱敏）。

## 必须回写长期记忆的线索

- 已验证相关的代码入口、可疑模块、关键函数与调用链入口。
- 已验证相关的数据库、表、字段、索引或 schema 位置。
- 已验证可复用的本地配置位置、查询关键词、定位经验。
- 被证明无效但容易重复误查的路径或假设，可记录为废弃 / 不适用线索。

## 不得回写的内容

- 未经代码 / 数据印证的截图猜测。
- 未读取真实文件的路径描述、未确认的字段含义。
- 只服务本轮一次性输出、后续不可复用的临时推理。
- 敏感凭据、token、密码、连接串原文。

## 回写格式建议

交给 `project-memory-rules` 时，按以下信息组织：

- `别名`：便于后续搜索的中文名和代码名。
- `类型`：代码入口、调用链线索、数据库线索、本地配置位置、查询经验、废弃线索。
- `定义`：说明这个线索能帮助定位哪类 Bug。
- `来源`：文件路径、命令、只读 SQL 或用户确认来源。
- `适用范围`：当前项目、某业务域、某类问题或跨项目。
- `状态`：启用、待验证、废弃。
