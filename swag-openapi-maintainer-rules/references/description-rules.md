# 中文说明规则

本文件定义 `swag-openapi-maintainer-rules` 在生成 OpenAPI / Swagger YAML 时，如何补齐接口头部、请求参数、响应字段的中文说明，以及如何生成单接口文件名中的中文简要说明。

## 强制范围

以下对象必须有中文说明：

- header 参数，包括鉴权、签名、语言、版本、设备、trace 等头部字段。
- 请求参数，包括 path、query、cookie、body、form、multipart 字段。
- 响应字段，包括统一响应包装字段、业务 `data` 字段、嵌套对象字段和数组元素对象字段。
- 安全方案说明，包括 `securitySchemes` / `securityDefinitions` 中对鉴权头或签名头的描述。

单接口 YAML 和总 YAML 都必须满足这一要求。

## 说明来源优先级

按以下顺序取值：

1. 真实中文注释：结构体字段注释、DTO 注释、wrapper 注释、鉴权中间件注释、错误码注释。
2. 现有业务文档：项目内接口文档、README、统一响应说明、请求模型说明。
3. 受控推导：字段名、DTO 名、wrapper 名、route 上下文、枚举语义足以支撑稳定中文含义时，允许补齐。

禁止跳过前两类来源直接使用推导。

## 接口中文简要说明来源

单接口文件名中的中文简要说明按以下顺序取值：

1. 接口显式中文 `summary`。
2. controller、route 注释、项目内接口文档中的稳定中文接口名。
3. 受控推导：根据 path 动词、controller 名、DTO 名、route 上下文生成短中文接口名。

该中文简要说明只用于“接口名 / 文件名后缀”，不用于补业务规则。
若 `summary` 或推导结果带有 `1.`、`11.`、`（1）`、`【1】` 等序号前缀，进入文件名之前必须剥离，只保留接口中文简介本体。

## 受控推导边界

允许推导：

- 字段字面含义：如 `country` -> `国家代码`。
- 统一包装语义：如 `status` -> `请求状态，成功为 true，失败为 false`。
- DTO / response 名对应的对象语义：如 `GetOrdersByWalletIdRequest` -> `查询订单历史请求`。
- 鉴权头的直译与业务已知含义：如 `Authorization` -> `签名鉴权值`。
- 接口中文简要说明：如 `getHistory` + `SellController` -> `卖出历史`。

禁止推导：

- 业务规则、校验条件、默认值、范围限制、时序要求、失败语义。
- 代码里不存在的上游约束、跨系统约定、环境假设。
- 仅靠字段缩写或模糊命名无法稳定得出的业务含义。

如果无法在允许边界内推导，必须列为缺失并阻断通过。

接口中文简要说明是例外：若无法稳定推导，可回退纯路径文件名，但必须把 `summary_source` 标记为 `unresolved`，不能伪造中文接口名。

## 具体要求

### 头部参数

- 每个 header 参数必须有中文 `description`。
- 若头部通过安全方案表达，安全方案自身也必须有中文说明。
- `Authorization`、`Trace-ID`、`gLang`、`gVersion`、`gDeviceID`、`gApiKey`、`gPlatform` 等常见头部不得只保留英文名和类型。

### 请求参数

- 每个请求字段必须有中文 `description`。
- body 根对象建议同时有中文对象说明，例如 `查询订单历史请求`。
- 对数组字段，数组字段本身必须有中文说明；若数组元素是对象，元素对象内部字段也必须继续补齐中文说明。

### 响应字段

- 响应包装字段如 `status`、`msg`、`data` 必须有中文说明。
- 若 `data` 引用对象 schema，该对象本身和其内部字段都必须有中文说明。
- 不能只给第一层字段补中文，嵌套对象继续留空。

## 阻断条件

- 任一 header 参数缺中文说明。
- 任一请求字段缺中文说明。
- 任一响应字段缺中文说明。
- 说明只能依赖猜测，无法说明来源。
- 为了补字段说明而编造业务规则或失败语义。
