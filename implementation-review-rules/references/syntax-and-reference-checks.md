# 语法与引用检查

## 用途

用于定义 `implementation-review-rules` 内置的语法、类型、引用与明显构建失败风险检查。

## 应拦下的问题

- 语法错误
- 类型不匹配
- 依赖或模块引用缺失
- 明显的构建入口错误
- 前端 import/export 缺失、重复或路径别名错误
- `.vue` 模板绑定、组件注册、script setup 引用、样式引用中的明显基础错误
- `.tsx`、`.jsx`、`.ts`、`.js` 中 ESLint、TypeScript、vue-tsc、构建命令可稳定发现的基础错误

## 常见风险

- 导入路径错误
- 模块名或符号名拼错
- 类型签名不一致
- 调用参数数量或类型不匹配
- 构建入口找不到依赖文件

## 不在本轮主范围

- 未使用导入或变量清理
- 多余空行和纯格式问题
- 命名、注释和实现结构问题
- 功能逻辑是否正确
- 纯格式化排序问题

## 处理原则

- 先拦基础失败，再讨论更高层质量。
- 只要能确定会失败，就不应继续推进到测试阶段。

## 正反例

### 正例 1：基础通过

- 语法正常。
- 类型签名一致。
- 引用路径和符号都存在。
- 结论：可进入下一步。

### 反例 1：导入丢失

- 新增调用了某个模块，但没有正确导入或依赖路径写错。
- 结论：驳回。

### 反例 2：类型签名不匹配

- 调用方和被调方参数不一致。
- 结论：驳回。
