# 颜色、字体、图标与 token 规则

## 颜色

- 组件里优先使用语义 token，不直接散写十几个原始十六进制颜色。
- 主色、辅助色、危险色、成功色、边框色、背景色要职责清楚。
- 暗黑模式单独校准对比度，不直接拿亮色模式值硬套。

## 字体

- 标题和正文的气质要一致，不要标题很个性、正文很随意。
- 没有明确项目规范时，避免默认使用过度常见的通用字体组合作为整页主气质。
- 如果页面需要更强识别度，可采用“个性标题字体 + 克制正文字体”的搭配，但正文仍应优先保证阅读稳定性。
- 正文行高保持可读，不要压得太紧。
- 数据列、价格、计时器等场景优先考虑等宽数字。

## 图标

- 不要把 emoji 当结构化图标。
- 同一个界面层级尽量统一同一套图标语言。
- 图标大小、描边粗细和圆角风格保持一致。
- 图标如果很小，点击热区也要放大。

## token 约束

- 颜色、字体、间距、圆角、阴影、层级和动效时长最好有统一 token。
- 不要为了赶进度就在组件里临时发明一堆局部值。
- 主题色和强调色应有主次，不要把所有颜色都做得同样响亮。
