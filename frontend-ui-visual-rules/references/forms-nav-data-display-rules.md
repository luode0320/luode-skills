# 表单、导航与数据展示规则

## 表单

- 标签清楚，必填项标识清楚。
- 错误文案要告诉用户哪里错了、怎么改。
- 长表单优先分组或分步，不要一次铺满。
- 复杂输入需要辅助说明，不要全靠猜。

## 导航

- 一个页面只保留一个最主要的导航层级。
- 导航当前位置要清楚，返回路径要稳定。
- 不要同时把顶部导航、底部导航、侧边栏都当一级主导航。

## 表格与列表

- 表格优先保证可扫描性，再讨论装饰。
- 列表项层级要清楚，主信息、次信息、操作区不要挤在一起。
- 大量数据优先考虑筛选、分组、分页或虚拟列表。

## 图表与数据

- 图表类型要匹配数据结构，不要为了酷随便选。
- 不要只靠颜色区分数据。
- 小屏幕下图表需要简化，不要把桌面端整张图硬缩小。
- 无数据、加载中、加载失败都要有明确状态。
