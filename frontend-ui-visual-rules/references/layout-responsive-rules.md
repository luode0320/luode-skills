# 布局与响应式规则

## 基础规则

- 默认移动端优先，再向平板和桌面扩展。
- 不能出现手机端横向滚动。
- 正文最小字号通常不低于 `16px`。
- 长文本桌面端不要无限拉宽，控制阅读宽度。
- 固定头部、底部工具栏、吸底按钮必须给正文预留空间。

## 间距与容器

- 统一使用可预期的间距梯度，不要一处 `14` 一处 `22` 一处 `37`。
- 卡片、分组、段落之间要靠间距表达层级，而不是只靠颜色。
- 大屏幕应增加左右留白，小屏幕优先保证内容先出现。

## 空间构图

- 允许有意识地使用不对称、错位、重叠、局部破格和密度对比，但必须服务信息层级。
- 一个页面最多保留 1 到 2 处强构图冲突点，不要处处都在破网格。
- 大面积留白和高密度信息都可以成立，关键是整页节奏要统一。
- 装饰层、背景层和内容层必须区分清楚，不能让背景抢走正文可读性。

## 移动端体验

- 底部主操作区要便于单手点击。
- 刘海、状态栏、底部手势区附近不要放关键点击目标。
- 不要把多个滚动区叠在一起，优先保持单一主滚动。

## 常见反模式

- 为了“满屏”强行塞满内容，结果层级混乱。
- 只在桌面端看起来合理，手机端断裂。
- 顶部一大块装饰把主内容压到首屏之外。
- 为了做出“设计感”而引入大量重叠，结果点击区、阅读顺序和响应式全部失控。
