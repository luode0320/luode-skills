# 布局与响应式规则

## 基础规则

- 默认移动端优先，再向平板和桌面扩展。
- 不能出现手机端横向滚动。
- 正文最小字号通常不低于 `16px`。
- 长文本桌面端不要无限拉宽，控制阅读宽度。
- 固定头部、底部工具栏、吸底按钮必须给正文预留空间。

## 间距与容器

- 统一使用可预期的间距梯度，不要一处 `14` 一处 `22` 一处 `37`。
- 卡片、分组、段落之间要靠间距表达层级，而不是只靠颜色。
- 大屏幕应增加左右留白，小屏幕优先保证内容先出现。

## 移动端体验

- 底部主操作区要便于单手点击。
- 刘海、状态栏、底部手势区附近不要放关键点击目标。
- 不要把多个滚动区叠在一起，优先保持单一主滚动。

## 常见反模式

- 为了“满屏”强行塞满内容，结果层级混乱。
- 只在桌面端看起来合理，手机端断裂。
- 顶部一大块装饰把主内容压到首屏之外。
