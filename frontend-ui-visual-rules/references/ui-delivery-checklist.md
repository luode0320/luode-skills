# 交付前 UI 自审清单

- 主 CTA 是否唯一且足够突出
- 首屏是否先出现最重要的信息
- 页面语气和风格方向是否明确，而不是像通用模板
- 是否存在一个清楚但不过度喧宾夺主的视觉记忆点
- 手机端是否无横向滚动、无遮挡、无误触
- 颜色对比度是否达标
- 文本字号和行高是否可读
- 图标、按钮、输入框是否有一致的状态表现
- loading、empty、error、success 状态是否齐全
- 焦点、键盘、屏幕阅读器基础可用性是否没有被破坏
- 动效是否克制且不影响操作
- 动效强度是否与页面语气、产品类型和实现复杂度匹配
- 是否仍然存在原始颜色值、原始阴影值、原始间距值到处散落
- 是否为了追求“高级感”破坏了清晰度、性能或可维护性
- 是否仍残留明显的“AI 味”特征，例如无场景的流行渐变、默认字体堆叠、模板化区块排列
- 若改动 `.vue`、`.tsx`、`.jsx`、`.ts`、`.js`、`.html`、`.css`、`.scss`、`.less`，是否已检查 import 排序/分组、未使用 import、缩进、空行和模板/script/style 区块排版
- 若项目存在 Prettier、ESLint、vue-tsc、tsc 或 lint 命令，是否已执行可用的前端基础校验
- 新增或修改的前端函数、事件处理、异步加载、条件渲染、复杂样式是否已完成注释必要性判断与补齐
