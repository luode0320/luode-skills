# UI 优先级模型

前端界面改动不要先堆效果，先按下面的顺序判断：

## 进入前先定方向

- 先明确页面用途、受众、语气和主目标。
- 先定义一个视觉记忆点，再决定是否需要强表达。
- 如果方向都没定，不要直接进入配色和动画。

1. 可访问性
- 对比度是否够
- 文字是否可读
- 键盘和焦点是否可用
- 图标按钮是否有可理解标签

2. 触达与交互
- 点击区域是否足够大
- 交互是否有明确反馈
- 是否只依赖 hover
- 错误、加载、成功状态是否明确

3. 布局与响应式
- 手机端是否优先成立
- 是否出现横向滚动
- 固定头部、底栏是否遮挡内容
- 内容层级是否清楚

4. 视觉系统
- 是否已经确定统一风格
- 颜色、字体、圆角、阴影是否属于同一语言
- 是否存在“每块都像不同产品”的问题

5. 动效与装饰
- 动效是否表达状态变化
- 是否尊重 reduced motion
- 是否为了炫而影响阅读和操作

## 使用提醒

- 先把 1 到 3 做对，再去优化 4 和 5。
- 如果基础可用性还没成立，不要先讨论渐变、玻璃感和高级效果。
- 如果产品是新页面或需要重新定视觉方向，先配合 `aesthetic-direction-rules.md` 收敛审美方向。
