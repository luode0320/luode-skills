# 交互、动效与可访问性规则

## 触达与反馈

- 点击区域应足够大，避免细小图标直接裸露成点击点。
- 所有可点击元素都要有清晰反馈，如颜色、透明度、阴影或位移变化。
- 异步按钮要有 loading 状态，避免重复提交。
- 错误提示要贴近问题位置，不要只在页面顶部丢一句失败。

## 可访问性

- 文本与背景要有足够对比度。
- 图标按钮、纯图形按钮要能被理解，必要时补文本或可访问标签。
- 焦点状态不能被移除。
- 不要只依赖颜色传达状态，错误和成功最好同时有文案或图标。

## 动效

- 微交互通常控制在 `150ms` 到 `300ms`。
- 动画优先使用 `transform` 和 `opacity`。
- 动效是为了说明状态变化，不是为了制造噪音。
- 要尊重 `prefers-reduced-motion`。

## 表单与状态

- 输入框要有真实标签，不要只靠 placeholder。
- 提交后必须有成功、失败或处理中反馈。
- 空状态要说明当前为什么为空，以及下一步能做什么。
