# 前端界面设计种子 Skill

当任务结果的好坏主要取决于视觉方向、信息层级、构图节奏、图片叙事和动效质感，而不是组件数量时，使用这个 skill。

目标：产出有明确主张、整体克制、质感稳定、首屏有记忆点的前端界面。

## 工作模型

开始实现前，先写清楚三件事：

- 视觉主张：一句话说明页面气质、材质感和情绪方向
- 内容结构：首屏、支撑信息、细节说明、最终行动
- 交互主张：2 到 3 个能明显影响体验的动效点

每个区块只承担一个主要职责，只保留一个主视觉重点和一个主要动作。

## 默认做法

- 先做构图，再做组件。
- 优先让首屏拥有完整视觉锚点，而不是先堆导航、卡片和按钮。
- 品牌名或产品名通常应是首屏最醒目的文字。
- 文案要足够短，用户几秒内能扫完。
- 优先通过留白、对齐、比例、裁切和对比建立层次，不要先加装饰性容器。
- 默认限制系统复杂度：最多两种字体、默认一个强调色。
- 默认不要做满屏卡片布局，优先用分栏、段落、媒体块和间隔建立结构。
- 首屏更像一张海报，而不是一份说明书。

## 落地页

默认顺序：

1. 首屏：品牌或产品、核心承诺、主按钮、一个主视觉
2. 支撑区：一个具体卖点、功能点或可信证明
3. 细节区：氛围、流程、产品深度或故事
4. 结尾行动区：注册、开始、联系、购买或访问

首屏规则：

- 只保留一个构图中心。
- 优先使用全幅图片或占主导地位的视觉平面。
- 做品牌型落地页时，首屏本身应支持全宽展开，不要被通用页面容器压缩。
- 信息优先级通常是：品牌、标题、正文、按钮。
- 默认不要使用首屏卡片、数据条、徽章云、悬浮仪表盘。
- 桌面端标题尽量控制在 2 到 3 行内，移动端一眼能读完。
- 文字区保持窄而稳，落在图片相对平静的区域。
- 图片上的文字必须保证对比度和点击区域清晰。

判断标准：

- 去掉图片后首屏仍然成立，说明图片太弱。
- 隐藏导航后品牌感明显消失，说明层级不够强。

## 应用界面

做业务系统、后台或工具型页面时，默认采用克制、安静、可扫描的布局：

- 表面层级清楚
- 字体和间距稳定
- 颜色少但明确
- 信息可以密但不能乱
- 装饰性外壳最小化
- 只有当卡片本身就是交互单元时才使用卡片

应用界面优先围绕以下区域组织：

- 主工作区
- 导航区
- 次级上下文或侧栏
- 单一明确的强调色用于动作或状态

避免：

- 首页就是卡片拼图
- 每个区域都加粗边框
- 常规业务界面滥用装饰性渐变
- 多个强调色互相竞争
- 只起装饰作用、不能提升扫描效率的图标

如果一个区域去掉卡片外壳后不影响理解，就优先去掉卡片。

## 图片使用

图片必须承担叙事职责，而不是只做背景。

- 品牌页、场馆页、内容页、生活方式产品页，至少使用一张有真实感的强图片。
- 优先使用真实场景图片，不优先抽象渐变或假 3D 元素。
- 选择适合承载文字的稳定明暗区域。
- 避免带有杂乱标识、路牌、文字噪声的图片。
- 避免生成自带 UI 面板、分屏和卡片结构的图片。
- 如果需要多个情境，用多张图，不用拼贴图。

## 文案

- 使用产品语言，不要把设计说明写进界面。
- 让标题承担主要信息。
- 辅助文案通常控制在一句短句。
- 删除重复表达。
- 每个区块只解释一件事：解释、证明、补充或转化。

如果删掉 30% 文案后页面更好读，继续删。

## 产品型页面文案

当页面是后台、仪表盘、管理端或运营工作台时，默认用“功能性文案”而不是“营销型文案”：

- 优先说明状态、数据和动作，不优先渲染情绪口号。
- 先展示工作区本身，例如指标、图表、筛选、表格、状态和任务上下文。
- 除非用户明确要求，否则不要给业务系统硬加营销式首屏。
- 标题要说明这个区域是什么、用户能做什么。
- 如果一句话像首页广告语，就把它改写成产品界面语言。

## 动效

动效用于建立存在感和层级，不用于制造噪音。

对于偏视觉导向的页面，至少设计 2 到 3 个明确动效：

- 一个首屏进入动效
- 一个滚动联动、粘性或景深效果
- 一个 hover、揭示或布局切换动效

推荐原则：

- 录屏里能明显感知
- 移动端也要流畅
- 节奏快但克制
- 全页动效语言统一
- 纯装饰性动效应删除

## 硬规则

- 默认不要用卡片。
- 默认不要做首屏卡片。
- 用户要求全幅首屏时，不要退化成居中窄栏布局。
- 每个区块不要同时塞多个主概念。
- 不要依赖很多微小 UI 装置来解释一个区块。
- 品牌型页面中，标题不要压过品牌本身。
- 不要写填充型废话文案。
- 无充分理由时，不要同时使用两种以上字体或一种以上强调色。

## 常见失败

- 首页第一眼就是普通 SaaS 卡片九宫格
- 图片不错，但品牌感很弱
- 标题很强，但没有清晰动作
- 图片太忙，压住文字
- 多个区块重复同一种情绪表达
- 轮播没有叙事意义
- 业务系统页面完全由堆叠卡片组成

## 自检问题

- 首屏是否能一眼看出品牌或产品是谁
- 是否存在一个明确的视觉锚点
- 只看标题能否理解页面结构
- 每个区块是否只有一个职责
- 卡片是否真的必要
- 动效是否提升了层级或氛围
- 去掉装饰阴影后，页面是否仍然成立
