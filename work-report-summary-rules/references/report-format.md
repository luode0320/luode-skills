# 报告格式模板

## 白话正文与附录

报告正文先说明工作结果、业务影响、完成情况和风险。提交明细、时间口径、统计来源和证据放入附录。

## 周报/月报/年报模板（含总体归纳统计）

```
周报（2026-03-30 星期一 - 2026-04-05 星期日）

总体归纳统计:
当前开发待发布版本总体进度(98%):
买卖币项目: 旧版数据迁移未完成 -> 项目完成度: 98%
市场活动项目: 金沙不够起兑门槛可用于兑换可叠加的优惠劵未完成 -> 项目完成度: 95%

各项目明细:

买卖币项目:
报告内容点:
- 功能: 新增订单风控校验
- 修复: 下单金额精度问题
- 进行中: 完善订单风控校验联调
```

## 日报模板（不含总体归纳统计）

```
日报（2026-04-03 星期五）

各项目明细:

买卖币项目:
报告内容点:
- 功能: 新增订单风控校验
- 修复: 下单金额精度问题
- 进行中: 补充周报 skill 的未提交任务描述
```

## 全周期无提交记录示例

```
周报（2026-03-30 星期一 - 2026-04-05 星期日）

总体归纳统计:
本周期无提交记录

各项目明细:

本周期无提交记录
```

- 顶部必须包含 `日期 + 星期`。
- 周报/月报/年报必须先输出 `总体归纳统计`（按项目大纲），再输出 `各项目明细`。
- 日报不输出 `总体归纳统计`，直接输出 `各项目明细`。
- `总体归纳统计` 必须按待发布版本进度口径输出，推荐格式：
  - `当前开发待发布版本总体进度(98%):`
  - `项目名: 未完成事项 -> 项目完成度: 98%`
- `总体归纳统计` 不写“完成2项/修复1项”这类数量统计。
- 未配置项目可输出：`项目名: 待补充未完成事项 -> 项目完成度: 未配置`。
- 无提交记录项目不展示在 `总体归纳统计` 和 `各项目明细` 里。
- 若本周期所有项目都无提交记录，统一输出 `本周期无提交记录`。
- 项目块必须保留 `报告内容点`。
- `报告内容点` 仅展示提交内容摘要，不再附带提交时间。
- 未提交事项必须使用 `进行中:` 前缀，并与已提交事项一起放在 `报告内容点` 下。
- 未提交事项优先根据代码 diff 内容和代码模块输出中文任务摘要。
- 设计文件、设计目录、需求/实施/审查文档和纯文档改动不作为任务依据；若只有这些改动，不输出进行中事项。
- 只有无法从代码 diff 可靠归纳时，才回退到代码路径摘要。
- 单条事项建议来源于 commit subject，避免泛化描述。
- 重命名、回滚、构建、文档、测试类低价值提交默认不进入报告。
- 低价值未提交改动（如纯文档、纯测试、构建产物、纯重命名）默认不进入报告。
- 报告输出语言必须为中文，编码必须为 UTF-8。
- 报告只能统计本人提交，不得混入其他作者提交。
- 未提交事项只能来自真实 Git 工作区证据，且任务摘要必须可回溯到代码 diff 或代码路径。
- 不使用设计文件、设计稿或文档名称生成报告任务点。
- 报告生成后自动保存到 `/home/luode/code`，文件名格式为 `?报-YYYYMMDDHHMMSS`（示例：`日报-20260403114521`）。
- `总体归纳统计` 可从 `references/projects.json` 读取配置：
  - 顶层：`release_overall_progress_percent`
  - 项目级：`release_pending_summary`、`release_progress_percent`
