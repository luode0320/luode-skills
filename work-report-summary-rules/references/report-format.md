# 报告格式模板

```
日报（2026-04-03 星期五）

买卖币项目:
1. 报告内容点:
- 2026-04-03 11:12:54: 功能: 新增订单风控校验
- 2026-04-03 14:38:21: 修复: 下单金额精度问题

市场活动项目:
1. 报告内容点:
- 本周期无提交记录
```

- 顶部必须包含 `日期 + 星期`。
- 项目块必须保留 `1. 报告内容点`。
- 单条任务点时间精度要求到秒：`YYYY-MM-DD HH:mm:ss`。
- 时间统一使用北京时间（`Asia/Shanghai`）。
- `完成/修复` 仅作为示例语义，不要求强制拆分为两个小节。
- 单条事项建议来源于 commit subject，避免泛化描述。
- 重命名、回滚、构建、文档、测试类低价值提交默认不进入报告。
- 报告输出语言必须为中文，编码必须为 UTF-8。
- 报告只能统计本人提交，不得混入其他作者提交。
