# 报告格式模板

## 周报/月报/年报模板（含总体归纳统计）

```
周报（2026-03-30 星期一 - 2026-04-05 星期日）

总体归纳统计:
买卖币项目: 本周主要完成订单风控校验、下单金额精度问题修复等任务。

各项目明细:

买卖币项目:
1. 报告内容点:
- 2026-04-03 11:12:54: 功能: 新增订单风控校验
- 2026-04-03 14:38:21: 修复: 下单金额精度问题
```

## 日报模板（不含总体归纳统计）

```
日报（2026-04-03 星期五）

各项目明细:

买卖币项目:
1. 报告内容点:
- 2026-04-03 11:12:54: 功能: 新增订单风控校验
- 2026-04-03 14:38:21: 修复: 下单金额精度问题
```

## 全周期无提交记录示例

```
周报（2026-03-30 星期一 - 2026-04-05 星期日）

总体归纳统计:
本周期无提交记录

各项目明细:

本周期无提交记录
```

- 顶部必须包含 `日期 + 星期`。
- 周报/月报/年报必须先输出 `总体归纳统计`（按项目大纲），再输出 `各项目明细`。
- 日报不输出 `总体归纳统计`，直接输出 `各项目明细`。
- `总体归纳统计` 只写任务大纲，不写数量统计（例如不要写“完成2项/修复1项”）。
- 无提交记录项目不展示在 `总体归纳统计` 和 `各项目明细` 里。
- 若本周期所有项目都无提交记录，统一输出 `本周期无提交记录`。
- 项目块必须保留 `1. 报告内容点`。
- 单条任务点时间精度要求到秒：`YYYY-MM-DD HH:mm:ss`。
- 时间统一使用北京时间（`Asia/Shanghai`）。
- `完成/修复` 仅作为示例语义，不要求强制拆分为两个小节。
- 单条事项建议来源于 commit subject，避免泛化描述。
- 重命名、回滚、构建、文档、测试类低价值提交默认不进入报告。
- 报告输出语言必须为中文，编码必须为 UTF-8。
- 报告只能统计本人提交，不得混入其他作者提交。
- 报告生成后自动保存到 `/home/luode/code`，文件名格式为 `?报-YYYYMMDDHHMMSS`（示例：`日报-20260403114521`）。
