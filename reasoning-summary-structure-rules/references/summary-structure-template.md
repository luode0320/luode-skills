# 推理总结结构模板

```text
Skill 命中检查：
- 命中 skill：...

Skill 执行证据：
- ...

当前要解决的问题：
- 用户原始需求：...
- 模型理解的需求：...
- 两者是否一致：...

当前问题的解决方案、问题根因：
- 解决方案：...
- 根因：...

当前解决结果、结论：
- ...

本次改动点：
- ...（仅在本轮有改动时出现）

验证结果：
- ...（仅在本轮有验证时出现）

函数注释核对：
- ...（仅在本轮有代码改动时出现；无函数改动需写“函数位点 0 个”）

后续内容：
1. ...（仅当存在原执行计划内未完成必需项、阻断项，或用户明确要求后续建议时出现；默认不出现）
```

## 结构要求

- 固定顺序输出，不要随意重排。
- 每一节至少 1 条可复核信息。
- 默认不输出“下一步状态 / 下一步建议”区块，直接在“当前解决结果、结论”处收口。
- 禁止用“等待用户新指令”“无需继续动作”“当前无需补充动作”一类占位文案替代下一步区块；这类文案在循环 loop 会话中可能被误触发为新任务。
- 只有存在原执行计划内未完成必需项、阻断项，或用户明确要求提供后续建议时，才输出 1-3 条后续内容；其中可选优化只能在用户明确要求建议/backlog 时出现。
