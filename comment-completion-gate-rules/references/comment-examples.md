# 注释补齐样例

## 更好

```go
// CreateOrder 创建订单
// [参数] ctx: 请求上下文；req: 创建订单请求
// [返回] *OrderResponse: 订单创建结果；error: 失败原因
// 最近修改时间: 2026-04-03 09:18:00 增加库存预占失败重试提示
func CreateOrder(ctx context.Context, req *OrderRequest) (*OrderResponse, error) {
    // 1. 校验请求参数并提取订单 ID
    // ...

    // 2. 查询库存并校验是否允许预占
    // ...

    // 3. 创建订单并回写预占状态
    // ...
}
```

```go
// Warmup 预热缓存
// [参数] 无
// [返回] error: 预热失败时返回错误
// 最近修改时间: 2026-04-03 09:20:00 增加空缓存保护
func Warmup() error {
    // 1. 加载核心配置并校验缓存开关
    // ...

    // 2. 逐批预热热点数据
    // ...
}
```

## 较差

- 新增或修改函数时，只补了方法体注释，没有 `[参数]` / `[返回]` / `最近修改时间`。
- 已经写了“查询订单”“更新状态”两个普通注释，但没有改成 `1.` `2.` 编号步骤注释。
- 只写 `1.1`、`1.2` 子步骤编号，却没有顶层 `1.`。
- `最近修改时间` 放在 `[参数]` 之前，或缺少本次改动原因简述。
