# 步骤编号闸门

- 当函数、方法存在 1 个及以上步骤时，必须至少使用顶层 `1.` 编号步骤注释；若存在多个连续步骤，继续使用 `1.` `2.` `3.`。
- 当顶层步骤内部仍有 2 个及以上子步骤时，允许并推荐继续使用 `1.1`、`1.2`、`2.1` 这类分层编号。
- 每个编号注释必须紧贴其对应步骤的第一行代码上方。
- 当步骤代码是 `if/for/switch/select` 时，编号注释放在该结构入口上方。
- 禁止把多个步骤编号集中写在函数开头作为总览后不就近落位。
- 禁止只出现子步骤编号而缺失对应顶层编号。
- 当方法体已经写出流程型普通注释时，必须改写为编号步骤注释；即使只有一个步骤也至少补 `1.`。
- 函数头元信息完整，也不能抵消方法体步骤编号缺失。
- 在原有方法中新增或修改补丁分支时，除编号步骤注释外，还必须就近补“做了什么 + 为什么要加”的原因注释；函数头元信息完整也不能抵消该要求。
