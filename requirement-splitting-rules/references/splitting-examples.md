# 拆分样例与反例

## 合理拆分示例

- 新页面 + 新接口 + 新配置：先后端接口闭环，再前端接入，再补配置与回归，并把拆分顺序更新到同一份 `ment/` 需求文档。
- 新导入流程：先文件校验，再入库处理，再结果展示与通知，并在同一份需求文档中持续维护拆分结果。
- 平台型需求同时包含聊天、文件、计费、分析：先拆成独立子项目，先细化并推进第一闭环子项目，其余只保留顶层边界和依赖说明。

## 不合理拆分反例

- 把同一接口的字段新增、校验和返回处理拆成三个不能独立验收的任务。
- 在目标和边界还没稳定前就先切波次，导致后续不断返工。
- 明明包含多个独立子系统，却不先做顶层拆分，直接把所有子系统细节写进一轮实现。
- 同一个需求单独再开一份“拆分文档”，导致入口、缺口、边界和拆分信息分散。
