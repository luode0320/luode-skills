# 设计预览确认闸门

正式 2D 游戏素材默认不允许“跳过参考筛选和设计确认直接生产”。

## 强制规则

- 只要用户还没选定参考方向，默认先进入“候选素材检索与筛选”阶段，先给用户看候选网站、截图或候选清单。
- 候选素材可以来自免费站、付费站、需登录站、商店页、作品页或宣传图；如果当前目的只是参考筛选，就不要因为不能自动下载而排除。
- 用户未选定要推进的参考截图前，不得默认假设某个候选已经被确认。
- 只要任务属于正式 2D 素材，不论是角色、怪物、地图、props、FX、图标还是掉落物，默认先展示设计图给用户确认。
- 用户未明确确认前，当前结果只能视为设计预览，不得直接进入最终生产。
- 用户如果继续提意见，默认继续改设计图，而不是直接改最终素材。
- 只有用户明确表达“可以按这个做 / 方向确认 / 开始生产 / 这个版本就做最终稿”时，才允许进入实现层。

## 设计阶段最低要求

设计预览前的参考阶段至少要满足以下之一：

- 网站候选清单
- 候选截图板
- 候选链接加风格提炼说明
- 用户自己提供的参考截图

设计预览阶段至少要满足以下之一：

- 单体设计图
- 场景方向图
- 地图底图方向图
- props 方向图
- projectile / impact / spell bundle 方向图
- UI / 图标方向图

默认不要只输出文字描述就直接开始生产。

## 为什么必须先确认

如果没有这一步，最常见的失败是：

- agent 参考了错误网站或错误候选，但用户根本没选过
- agent 把错误世界观一路做到底
- 图像方向不对，但后处理已经投入很多时间
- 用户真正想要的是审美方向调整，不是技术生产
- 低质量几何结果被错误包装成“已完成素材”

## 失败信号

出现以下任一项时，不得进入生产：

- 用户还没选定参考截图
- 用户说“先给我看几个素材方向”
- 用户说“先给我看设计”
- 用户说“先出几张方案”
- 用户说“我想先确认风格 / 方向”
- 当前项目风格还不稳定
- 当前参考差距很大
- 当前结果仍像图标、贴纸、占位图、几何拼图、幼稚简笔画或廉价 prop pack

## 进入生产前必须说明

在从设计阶段切到生产阶段前，至少说明：

- 当前用户已选定哪个参考候选或参考截图
- 当前用户已确认哪个设计版本
- 生产对象是什么
- 产出格式是什么
- 是否继续沿用当前视觉基线
