# 角色动画生产闸门

当 2D 素材任务从“静态设计”进入“角色动画 / 怪物动画 / Boss 动画 / sprite sheet 生产”时，必须联动共享根目录 `character-sprite-animation-production`。

## 自动触发条件

出现以下任一项时触发：

- idle
- walk
- run
- attack
- cast
- hit
- death
- 4向
- 8向
- sprite sheet
- frame set
- 动作帧

## 强制规则

- 先锁定角色 identity，再做动作。
- 先做 base pose，再扩动作。
- 动作和方向逐项生产，不要一锅生成。
- 生产后必须做动作 QA。

## 默认失败信号

出现以下任一项时，默认回炉：

- 帧间角色身份飘了
- 角色缩放漂移太大
- 脚底锚点飘动
- 左右方向不对称
- 攻击帧把本体挤小
- 大范围 FX 应拆未拆
- 动作节奏读不出来

## 与设计确认闭环的关系

- 如果用户还没确认静态设计方向，先停在设计阶段。
- 只有静态设计方向被确认后，才进入动画生产。
