# 架构专题示例片段

以下片段用于约束文档形态。使用时必须替换为当前项目的真实目录、模块和业务链路，不得原样保留占位内容。

## `2-目录树.md`

````markdown
# 目录树

## 文档说明

- 用途: 记录当前有效目录结构及目录职责入口。
- 事实来源: 当前仓库目录、构建配置、模块入口。
- 最近同步时间: YYYY-MM-DD

## 当前目录

```text
.
├── cmd/                 # 进程入口
├── internal/
│   ├── order/           # 订单业务
│   └── payment/         # 支付业务
├── migrations/          # 数据库迁移
└── doc/                 # 研发文档
```

## 目录职责

| 路径 | 职责 | 允许内容 | 禁止内容 |
| ---- | ---- | -------- | -------- |
| `cmd/` | 组装并启动进程 | 启动入口、依赖装配 | 业务规则 |
| `internal/order/` | 承载订单领域能力 | 订单用例、领域模型 | 支付渠道实现 |

## 变更同步

- 新增顶层目录时，同步更新 `1-总架构.md`。
- 目录职责变化时，同步更新 `3-模块职责.md`。
````

## `3-模块职责.md`

```markdown
# 模块职责

## 模块清单

| 模块 | 核心职责 | 主要输入 | 主要输出 | 不负责 | 依赖模块 |
| ---- | -------- | -------- | -------- | ------ | -------- |
| 订单模块 | 创建订单并维护订单状态 | 下单请求 | 订单、状态事件 | 扣款 | 支付模块 |
| 支付模块 | 发起扣款并回传支付结果 | 支付指令 | 支付结果 | 创建订单 | 渠道适配模块 |

## 协作边界

- 订单模块只能通过支付接口发起扣款，不直接调用具体支付渠道。
- 支付模块发布结果后，由订单模块决定订单状态迁移。

## 变更同步

- 模块新增、拆分、合并或职责迁移时，更新本表。
- 依赖方向变化时，同步检查 `1-总架构.md` 和相关业务链路文档。
```

## `6-业务链路-订单支付.md`

```markdown
# 业务链路-订单支付

## 文档说明

- 业务目标: 完成订单扣款并形成可追踪的最终结果。
- 入口: 支付接口。
- 责任模块: 订单模块、支付模块。
- 事实来源: 路由、服务调用、状态定义、事件处理代码。
- 状态: 启用。

## 前置条件

- 订单已创建且允许支付。
- 支付渠道配置有效。

## 主链路

1. 订单模块校验订单状态。
2. 支付模块创建支付请求。
3. 渠道适配模块调用外部支付渠道。
4. 支付模块记录支付结果。
5. 订单模块根据结果迁移订单状态。

## 异常分支

| 异常场景 | 处理方式 | 最终状态 |
| -------- | -------- | -------- |
| 渠道超时 | 保持处理中并等待查询或回调 | 支付处理中 |
| 明确失败 | 记录失败原因并允许按规则重试 | 支付失败 |

## 输入输出

| 类型 | 内容 |
| ---- | ---- |
| 输入 | 订单标识、支付方式、金额 |
| 输出 | 支付结果、订单状态、审计记录 |

## 变更影响

- 模块职责变化时同步更新 `3-模块职责.md`。
- 新增目录或入口时同步更新 `2-目录树.md`。
- 成为项目核心链路或改变顶层理解时同步更新 `1-总架构.md` 与根目录 `项目设计.md`。
```
