# 与项目设计文档的分层关系

## 根目录 `项目设计.md`

- 定位为项目级总览主入口。
- 重点说明项目目标、整体技术栈、顶层目录与模块总览、核心链路和当前约束。
- 内容应保持可快速阅读，不承载过长专题细节。

## `doc/1-架构/`

- 定位为长期架构专题目录。
- 适合沉淀目录树、目录职责、模块关系、业务功能设计、运行时读取、配置规范和专题架构说明。
- 可以被 `项目设计.md` 引用，但不与 `项目设计.md` 并行维护同一层级总览。

## 分层判断

- 如果内容回答“这个项目整体是什么、当前总览是否可信”，优先进入 `项目设计.md`。
- 如果内容回答“某个长期专题如何设计、某组目录/模块职责如何划分”，优先进入 `doc/1-架构/`。
- 如果内容只服务于当前一次需求交付，优先进入 `doc/3-实施/`，不要写入长期架构目录。
- 如果专题变化改变了项目顶层理解，应先更新 `doc/1-架构/`，再同步摘要到 `项目设计.md`。
