# 架构文档模板

建议文件名：

```text
doc/1-架构/YYYY-MM-DD_<架构中文主题>.md
```

建议结构：

```markdown
# <架构中文主题>

## 文档说明

- 用途:
- 事实来源:
- 适用范围:
- 最近同步时间:

## 当前结论

- 长期成立的架构结论。
- 当前不覆盖的范围。

## 目录树或模块结构

```text
.
└── example/
```

## 目录职责或模块职责

| 路径/模块 | 职责 | 边界 |
| --------- | ---- | ---- |

## 主要业务或功能设计

- 主链路:
- 关键输入输出:
- 上下游关系:

## 与项目设计.md 的关系

- `项目设计.md` 应保留的摘要:
- 本文承接的专题细节:

## 待同步项

- 待确认:
- 待更新:
```

写作要求：

- 事实优先，不写无来源猜测。
- 长期专题优先，不混入一次性实施步骤。
- 目录树使用当前真实目录；路径以 `artifact-storage-rules` 当前口径为准。
