# 架构主入口模板

## 标准目录形态

```text
doc/1-架构/
├── 1-总架构.md
├── 2-目录树.md
├── 3-模块职责.md
├── 4-主要业务链路.md
├── 5-业务链路-用户登录.md
├── 6-业务链路-订单支付.md
└── 7-业务链路-内容发布.md
```

四个主入口固定占用序号 `1-4`。单条业务链路从 `5` 开始按当前最大业务链路编号加一，同一链路持续更新原编号文档。

## `1-总架构.md` 模板

```markdown
# 总架构

## 文档说明

- 用途:
- 事实来源:
- 适用范围:
- 最近同步时间:

## 架构总览

- 长期成立的架构结论。
- 当前不覆盖的范围。

## 系统边界

- 系统负责:
- 系统不负责:
- 外部依赖:

## 架构分层

| 层级 | 核心职责 | 主要模块 | 依赖方向 |
| ---- | -------- | -------- | -------- |

## 专题入口

- [目录树](2-目录树.md)
- [模块职责](3-模块职责.md)
- [主要业务链路](4-主要业务链路.md)

## 与项目设计.md 的关系

- `项目设计.md` 应保留的摘要:
- 本文承接的专题细节:

## 待同步项

- 待确认:
- 待更新:
```

写作要求：

- 事实优先，不写无来源猜测。
- 长期专题优先，不混入一次性实施步骤。
- 目录树使用当前真实目录；路径以 `artifact-storage-rules` 当前口径为准。
- `1-总架构.md` 只保留架构总览和专题导航，目录、模块、链路细节分别进入对应编号中文文档。
