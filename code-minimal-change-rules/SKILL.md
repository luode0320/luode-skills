# 最小改动编码规则

只在控制“本次到底该改多少、该改到哪里”为核心问题时使用这个 skill。
如果当前争议已经是可读性、注释风格、代码存放位置或测试规范，请转交对应的小 skill。

## Skill 作用与适用场景

- 收缩本次改动范围，避免无关修改。
- 阻断顺手重构、顺手统一风格、顺手清理大段旧代码的越界行为。
- 约束新增代码、修复 Bug、补测试支撑代码时只做最小必要改动。
- 降低回归风险、审查成本和问题排查难度。
- 即使要做表达层简化，也只限最近改动和当前必要位置，不扩散到历史代码。
- 对高复用通用代码执行保守策略：最近修改超过7天时，默认新增能力，不直接改旧行为。
- 明确“最小改动”不等于“跳过注释”：本轮改动位点仍需联动 `comment-placement-granularity-rules` 与 `comment-completion-gate-rules` 补齐注释。
- 编码前必须写清假设、边界和成功标准；不确定项先澄清，不得带着模糊假设直接实现。
- 实现时坚持简单优先：不做猜测性扩展，不为一次性改动引入过度抽象。
- 结果应可验证：每个关键步骤都要有对应检查项，避免“看起来能用”式弱收口。

## 自动触发信号

- 新增业务代码。
- 修改已有逻辑。
- 修复 Bug。
- 为测试补最小支撑代码。
- 为联调补最小必要适配。

## 进入后先做什么

1. 先确认本次任务唯一目标和成功标准（改到什么算完成）。
2. 显式写出当前实现依赖的关键假设；若存在不确定项，先澄清再编码。
3. 列出为了完成目标必须改动的文件、方法和模块。
4. 标出不属于当前目标的潜在顺手改动。
5. 如果包含“顺手简化代码”，先限定只能处理最近改动中真正影响理解的部分。
6. 如果发现改动目标已经扩散，先暂停并收缩范围。
7. 标记本轮改动位点，预留 `comment-placement-granularity-rules` 与 `comment-completion-gate-rules` 注释补齐动作，不把“未改动周边”误解为“改动位点可不注释”。

## 默认执行流程

1. 先读取 `references/minimal-change-general.md`，明确本次允许的改动范围以及“假设-目标-验证”最小闭环。
2. 如果与可读性、风格一致性、代码归位、测试补充发生边界争议，再读取 `references/minimal-change-boundaries.md`。
3. 如果需要判断某个改动是否越界，再读取 `references/minimal-change-examples.md` 对照正反例。
4. 先收缩范围，再用最少代码实现目标，不做猜测性扩展。
5. 完成后复查是否混入无关重构、无关格式化、无关命名调整、无关清理。
6. 复核“每一行改动都能追溯到当前目标”，并按成功标准执行最小验证闭环。
7. 最后联动注释双 skill：用 `comment-placement-granularity-rules` 判断必要性 / 位置 / 颗粒度，再用 `comment-completion-gate-rules` 完成改动位点补齐闸门，不扩散到无关历史代码。

## 权责边界与不负责事项

- 只负责约束改动范围，不替代 `code-readability-rules` 处理可读性细节。
- 不替代 `code-style-consistency-rules` 处理项目风格一致性。
- 不替代 `code-placement-review-rules` 处理代码归位和目录落点问题。
- 不替代测试类 skill 决定如何组织测试资源和验证范围。

## 需要暂停并确认的条件

- 当前需求想顺手修历史问题。
- 当前 Bug 修复想顺手重构整个模块。
- 本次改动已经扩散到多个互不相干的目标。
- 用户明确要求扩大改动范围，但尚未确认额外影响和回归风险。
- 计划直接修改“最近修改超过7天”的高复用通用代码旧行为，但尚未确认兼容与影响范围。

## 执行通过 / 驳回标准

- 通过：改动与当前目标直接相关，范围可解释，无明显无关修改；关键假设与成功标准明确；本轮改动位点注释补齐已完成。
- 驳回：出现顺手优化、顺手重构、顺手统一风格、顺手清理整片旧代码等越界行为，或把本该独立立项的问题混进本次修改；或以“最小改动”为由跳过改动位点注释补齐；或直接修改“最近修改超过7天”的高复用通用代码旧行为。

## 执行结果归档要求

- 如果明确排除了某些不应在本次处理的事项，将结论记录到 `analysis/` 或 `review/`。
- 归档内容至少包含本次目标、刻意不处理项、排除原因和建议后续动作。
- 如果本次范围没有争议且无额外排除项，可以不单独归档。

## references 读取规则

- 默认先读 `references/minimal-change-general.md`。
- 只有在边界不清时，再读 `references/minimal-change-boundaries.md`。
- 只有在需要做越界判断时，再读 `references/minimal-change-examples.md`。
