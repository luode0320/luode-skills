# 验收不通过归因样例

## 用途

用于在验收不通过时，判断应回需求域还是 Bug 域。

## 正例

### 正例 1：需求理解偏差

- 现象：功能实现和用户预期不一致。
- 原因：需求描述本身存在歧义，开发按另一种合理理解实现。
- 结论：回需求域。

### 正例 2：代码缺陷

- 现象：接口返回明显错误或运行异常。
- 原因：实现错误、状态异常或历史逻辑缺陷。
- 结论：转 Bug 域。

## 反例

### 反例 1：验收不通过就直接判成 Bug

- 未做归因就直接进入代码修复。
- 结论：驳回。

### 反例 2：明显是历史缺陷却继续留在需求域

- 问题本身与本次需求目标无关，只是开发过程中暴露出来。
- 结论：应拆出并转 Bug 域。
