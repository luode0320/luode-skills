# 历史回忆规则

只在需要补回“过去发生过什么、以前怎么做过、以前怎么修过、以前怎么决策过”时使用这个 skill。
如果当前问题本质上还是需求不清、Bug 未定位、代码未写完或测试未完成，不要把历史回忆当成主流程本身。

## Skill 作用与适用场景

- 检索历史需求、历史修复、历史决策、历史实现方案。
- 帮当前任务快速判断“是不是以前做过、以前是怎么处理的”。
- 降低重复分析、重复试错和重复讨论。
- 在没有真实记忆后端时，退化为基于本地材料的历史回溯，而不是假装具备永久记忆。

## 自动触发信号

- 用户问“上次怎么做的”“之前是不是修过这个”“以前讨论过这个方案吗”。
- 当前任务需要知道类似接口、类似表结构、类似配置、类似 Bug 的历史处理方式。
- 用户要求“先回忆一下以前怎么做的，再决定现在怎么做”。
- 当前结论明显依赖过去会话，而不是只靠本轮上下文就能回答。

## 进入后先做什么

1. 先确认用户问的是历史信息，不是当前对话里刚说过的内容。
2. 如果只是新会话刚开始、当前缺少最近上下文，应让给 `recent-context-bootstrap-rules`。
3. 明确回忆目标：需求、Bug、接口、表结构、配置、交付结论中的哪一类。
4. 先判断是否存在真实持久记忆后端。
5. 如果没有后端，明确说明将降级为本地材料回溯。
6. 先缩小检索范围，再决定是否深读详情。

## 默认执行流程

1. 默认先读 `references/backend-boundary.md`，判断当前能否使用真实记忆后端。
2. 如果存在真实记忆后端，再读 `references/persistent-search-workflow.md`。
3. 如果没有真实记忆后端，先读 `../artifact-storage-rules/references/path-map.yaml` 与 `../artifact-storage-rules/references/update-policy.md`，确认本地回溯扫描根目录，以及命中后应回写到当前主任务文档还是当前 Bug / 测试主记录。
4. 如果没有真实记忆后端，再读 `references/local-fallback-recall.md`。
5. 如果需要了解来源和吸收边界，再读 `references/source-notes.md`。
6. 输出时明确区分“检索到的历史记录”与“基于历史记录得出的结论”。

## 权责边界与不负责事项

- 只负责补回历史上下文，不负责代替当前需求分析、当前 Bug 定位和当前测试验证。
- 新会话刚开始且只需要最近 3 天上下文预热时，应让给 `recent-context-bootstrap-rules`。
- 不把本地文档回溯结果伪装成真实跨会话记忆命中结果。
- 不因为“以前做过”就跳过当前代码和当前需求核对。
- 不把历史旧方案直接机械复制到当前实现。

## 需要暂停并确认的条件

- 用户要查的是当前会话刚刚说过的内容，其实不需要历史检索。
- 当前只是新会话缺少上下文，但用户并没有明确要求历史深回忆。
- 没有记忆后端，也没有足够本地材料支持可靠回溯。
- 历史记录之间明显冲突，无法判断哪条才是最终结论。
- 历史记录与当前仓库状态明显不一致，继续引用风险很高。

## 执行通过 / 驳回标准

- 通过：能明确给出历史来源、时间顺序、相关结论，并标清是真实记忆检索还是本地材料回溯。
- 驳回：把猜测当成历史事实，或把历史信息错误当成当前实现已经正确的证据。

## 执行结果归档要求

- 如果历史回忆结论会影响当前需求、Bug、交付或实现决策，应写回当前主任务文档，而不是单独散落。
- 优先更新 `artifact-storage-rules` 约定的当前需求文档、Bug 文档、交付说明或评审说明。
- 本地回溯扫描根目录和回写复用策略统一遵循 `../artifact-storage-rules/references/path-map.yaml` 与 `../artifact-storage-rules/references/update-policy.md`。
- 归档内容至少包含：回忆范围、数据来源、核心历史结论、对当前任务的影响。

## references 读取规则

- 默认先读 `references/backend-boundary.md`。
- 只有在真实记忆后端可用时，再读 `references/persistent-search-workflow.md`。
- 只有在需要本地降级回溯或决定历史结论回写位置时，再读 `../artifact-storage-rules/references/path-map.yaml` 与 `../artifact-storage-rules/references/update-policy.md`。
- 只有在需要本地降级回溯时，再读 `references/local-fallback-recall.md`。
- 只有在需要了解来源和吸收边界时，再读 `references/source-notes.md`。
