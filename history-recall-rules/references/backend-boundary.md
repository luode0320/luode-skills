# 记忆后端边界

## 先判断有没有真实记忆后端

只有在存在真实持久记忆后端时，这个 skill 才能执行真正的跨会话历史检索。

典型等价能力包括：

- 根据关键词搜索历史记录
- 获取某条历史记录前后的时间线
- 批量获取历史记录完整详情

如果这些能力不存在，就不能声称自己具备永久记忆检索。

## 没有后端时如何处理

没有真实记忆后端时，只能做“本地材料历史回溯”：

1. 先查 `ment/`
2. 再查需求、验收、评审、交付和测试文档
3. 再查 Git 提交历史和变更说明
4. 再查仓库里的分析文档和已有规划

## 输出要求

- 必须明确写出“真实记忆检索”还是“本地材料回溯”。
- 如果只是本地回溯，不要写成“已从永久记忆确认”。
- 若来源不充分，应直接说明结论可信度有限。

## 边界提醒

- 历史检索只补上下文，不替代当前事实核对。
- 如果历史结论与当前代码不一致，应优先指出冲突，而不是盲从旧记录。
