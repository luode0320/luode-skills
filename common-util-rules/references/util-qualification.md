# 通用工具判定

## 用途

用于判断一段代码到底是不是通用工具。

## 可以算通用工具

- 脱离当前业务上下文仍然成立。
- 输入输出清晰、职责稳定、复用范围明确。
- 不依赖当前业务状态、流程阶段或领域术语。

## 不应算通用工具

- 只服务于单一业务流程。
- 强依赖当前业务上下文或特定模块。
- 需要带大量业务参数才能勉强复用。

## 原则

- 真正通用才抽，不为未来幻想抽象。
- 通用代码优先保持稳定，默认新增能力而不是修改旧行为。

## 7天冻结判定

- 目标是降低高复用通用代码的回归风险。
- 默认判定口径：以目标文件最近一次 Git 修改时间为准。
- 判定命令示例（Unix 时间戳）：
  - `git log -1 --format=%ct -- <file>`
- 若“当前时间 - 最近修改时间”大于 `7 * 24 * 3600` 秒：
  - 视为“7天冻结”。
  - 默认策略：可新增，不改旧行为。
- 若最近修改时间在 7 天内：
  - 视为“开发迭代期”。
  - 允许在充分评估后修改，但仍应遵循最小改动原则。
- 无法获取历史（新文件、未纳入 Git）时，按“7天内”处理。
