# 主动侦察清单

## 侦察入口

- 当前项目长期记忆：`PROJECT_MEMORY.md`
- 当前项目风格记忆：`PROJECT_STYLE.md`
- 需求与阶段材料：`doc/1-架构/`、`doc/2-需求/`、`doc/3-实施/`、`doc/7-验收/`
- 近期记录：`doc/4-bugs/`、`doc/5-tests/`、`doc/6-审查/`
- 根目录说明：`README.md`、`项目设计.md`、同类项目说明文档
- 代码入口：路由、controller、service、repository、model、frontend 页面、任务脚本、配置文件
- 数据入口：数据库连接配置、迁移脚本、模型定义、SQL、ORM schema、表名和字段名
- 上下游入口：HTTP client、SDK、第三方服务配置、webhook、队列、定时任务、RPC 调用
- 用户补充入口：本地项目路径、URL、数据库名、服务名、文档路径、截图、资料包、GitHub 仓库或官方 API 文档入口

## 侦察顺序

1. 先查长期记忆，避免重复搜索已经沉淀过的项目经验。
2. 再查当前项目文档和最近需求，确认是否有同类历史方案。
3. 再查代码结构，找已有模块、页面、接口、表模型和上下游调用。
4. 再查配置和依赖，确认外部服务、数据库、队列、SDK 或第三方能力。
5. 最后查用户补充路径、URL、GitHub 仓库和官方 API 文档入口，作为扩展证据来源。

## 最小成果

- 原始 idea 和 agent 当前理解。
- 已查资料清单。
- 当前项目已有能力。
- 可用数据、表、字段、接口或代码模块。
- 缺失能力和阻断点。
- 2-3 个可行方案或唯一推荐方案。
- 推荐方案的证据来源。
- 需要用户拍板的最少问题。
