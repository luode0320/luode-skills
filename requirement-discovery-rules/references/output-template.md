# 需求侦察输出模板

## 推荐结构

```markdown
## 原始想法

- 用户原始 idea:
- agent 当前理解:

## 已侦察资料

| 来源 | 类型 | 结论 | 证据等级 |
| ---- | ---- | ---- | -------- |

## 当前项目现状

- 已有能力:
- 可复用代码:
- 可用数据 / 表 / 字段:
- 上下游 / 第三方:
- 用户补充线索处理结果:

## 方案对比

| 方案 | 做法 | 依赖 | 优点 | 风险 | 证据 |
| ---- | ---- | ---- | ---- | ---- | ---- |

## 推荐方案

- 推荐:
- 推荐原因:
- 不推荐方案及原因:

## 风险与边界

- 业务风险:
- 技术风险:
- 数据风险:
- 权限 / 安全 / 成本风险:

## 仍需用户拍板

1. ...

## 记忆回写建议

- 需要写入 `PROJECT_MEMORY.md` 的线索:
- 不写入原因:

## 下一步

- 转交 skill:
- 进入条件:
```

## 输出要求

- 待确认问题不超过 3 个，除非当前 idea 本身包含多个互不相关目标。
- 每个推荐方案必须至少有一个证据来源。
- 如果没有查到可用资料，要明确写出查过哪里、为什么没有结论。
- 若已形成可开发需求，下一步转 `requirement-intake-rules`。
- 若仍缺少关键事实，下一步转 `requirement-gap-rules`，但只保留侦察后仍无法解决的问题。
